# 第 1 章：为什么我们需要 3DGS？——从问题出发的第一性推导

### 🎯 本章核心问题链

```
你有几十张照片 → 想看没拍过的角度 → 计算机怎么"想象"新视角？
     ↓
体素太占内存 → 点云有采样空洞 → NeRF 慢到不能用
     ↓
3DGS：换表示，从隐式 MLP 跳到显式高斯
```

---

## 一、视图合成问题的本质

**场景**：你用手机绕着房间转了一圈，拍了 20 张照片。现在你想看**蹲下来仰视天花板**的角度——但你没拍过这个视角。

问题可以形式化为：

$$
\text{输入: } \{(I_m, C_m)\}_{m=1}^M \quad \text{输出：任意新视角 } C_\text{new} \text{ 对应的图像 } I_\text{new}
$$

其中 $C_m$ 是第 m 张照片的**相机参数**。

### 🔍 **关键概念细化：什么是相机参数？**

<details>
<summary>点击展开详细解释</summary>

每张已知照片需要两个信息才能重建 3D 射线：

#### 1. 内参矩阵 $K$（相机自己的属性）

$$
K = \begin{bmatrix} f_x & 0   & c_x \\ 
                    0   & f_y & c_y \\ 
                    0   & 0   & 1 \end{bmatrix}
$$

- **fₓ, fᵧ**：焦距（像素单位），典型值 ~1000-3000
- **cₓ, cᵧ**：主点（principal point），图像中心的像素坐标

#### 2. 外参 $[R|t]$（相机在世界中的位置和朝向）

- **$R \in SO(3)$**: 旋转矩阵，描述相机朝向
- **$\mathbf{t} \in \mathbb{R}^3$**: 平移向量
- 等价地用**相机中心 $\mathbf{C} = -R^\top\mathbf{t}$**

#### 射线重建公式：从像素到 3D 射线

$$
(u,v) \rightarrow \text{归一化方向 } \mathbf{d} = R^\top K^{-1}[u, v, 1]^\top \\
r(t) = \mathbf{C} + t \cdot \mathbf{d}, \quad t > 0
$$

</details>

### ⚠️ **问题的关键约束**（这是本质，不是细节）：

| 约束 | 含义 | 对算法的影响 |
|------|------|--------------|
| **稀疏性** | 只有几十到几百张照片 | 不能假设输入是密集的 |
| **稠密输出** | 要渲染连续、无锯齿的图像 | 表示必须是连续的或可插值的 |
| **实时性需求**（可选） | AR/VR 需要 60fps+ | 推理速度必须快 |

## 二、第一个想法：把世界切成方块？

### 体素（Voxel）——3D 版本的像素

**直觉**：2D 图像由像素组成，那 3D 空间是不是可以由**体素**（体积像素）组成？

$$
\begin{aligned}
&\text{每个体素存储:} \\
&\quad \bullet \; \text{颜色 } c \in \mathbb{R}^3 \text{ (RGB)} \\
&\quad \bullet \; \text{密度 } \sigma > 0 \text{（表示这个位置有没有东西）}
\end{aligned}
$$

### 渲染公式：体积渲染方程

从相机发射射线 $r(t)$，沿射线积分每个点的颜色贡献：

$$C(r) = \int_{t_n}^{t_f} T(t) \cdot \sigma(r(t)) \cdot c(r(t)) \, dt$$

其中：
- $T(t) = \exp\left(-\int_{t_n}^t \sigma(r(s)) \, ds\right)$ 是**透射率**（transmittance），表示光线从起点到 t 没有被遮挡的概率
- $\sigma(r(t))$ 是该点的密度
- $c(r(t))$ 是该点的颜色

#### 🔍 **关键概念细化：这个公式的物理来源？**

<details>
<summary>点击展开</summary>

**1. 密度的物理含义**
- $\sigma=0$: 完全透明（空气）
- $\sigma \to \infty$: 完全不透明（实体表面）  
- $\sigma > 0$: 半透明介质（烟雾、毛玻璃）

**2. 透射率的推导**
考虑长度为 $dt$ 的薄层：
- 被吸收的概率 = $\sigma \cdot dt$
- **穿过的概率** = $1 - \sigma \cdot dt$

从起点到 t，要穿过无数薄层，累积穿透概率就是连乘：
$$T(t) = \prod_{s=t_n}^t (1 - \sigma(s)ds) \approx \exp\left(-\int_{t_n}^t \sigma(s) \, ds\right)$$

**3. 数值近似（Ray Marching）**
实际计算时离散成 N 个采样点：

```python
C = 0.0
T_acc = 1.0  # 累积透射率

for t in [t_1, t_2, ..., t_N]:
    alpha = 1 - exp(-sigma * dt)  # 该段的"不透明度"
    C += T_acc * alpha * color    # 当前段对最终颜色的贡献
    T_acc *= (1 - alpha)          # 更新透射率（前面物体的遮挡）
```

</details>

### ❌ **体素的致命缺陷：内存立方增长**

**公理**：如果空间分辨率从 $V$ 翻倍到 $2V$，体素数量变成 $(2V)^3 = 8V^3$ ——**立方增长**！

算一笔账：
- **128³网格** = 2.09×10⁶个体素 → ✅ 可接受
- **512³网格** = **1.34×10⁸个体素** → ❌ 每个存 RGB+σ = 512MB（还只是存储）
- **1024³网格** = **1.07×10⁹个体素** → ❌ **需要 4GB 只存密度和颜色！**

而且大部分空间是空的（桌子后面、墙里面），我们付了钱却没用到。

## 三、第二个想法：既然空，为什么不存？→ 点云

### 点云的直觉

如果 90% 的空间都是空气，**只存储有东西的地方**：

$$
p_i = (\mathbf{x}_i, c_i), \quad \mathbf{x}_i \in \mathbb{R}^3, \; c_i \in [0,1]^3
$$

**优势**：内存从 $V^3$ 降到 $N$（$N$是点的数量），而且通常 $N \ll V^3$。

### ❌ **新问题：点是零维的——采样间隙问题**

想象渲染过程：相机发射射线，怎么和点云交互？

- **理想情况**：射线刚好穿过某个点 → 获取颜色
- **实际情况**：**99.9% 的射线从点的间隙中穿过** → 什么也看不到！

点在 3D 空间是零维的，投影到 2D 也是零维的——它们没有"面积"来贡献像素颜色。

### 🔍 **可视化理解：为什么点会漏？**

```
2D 示意图（降低维度）:

相机射线 →  ·      ← 点在间隙中
          ↓
       [空]  
       [空] 
       [空]
       
实际渲染时，每个像素应该从某条射线采样，但射线"错过"了所有点！
```

### 尝试修复：给点加体积？

**方案 A：Point Sprites（点精灵）**
- 每个点渲染成一个小圆圈/平面
- **问题**：近大远小关系不对，透视错误明显

**方案 B：2D Gaussian Splatting（屏幕空间高斯）**
- 直接在屏幕空间画 2D 高斯椭圆
- **问题**：**没有 3D 几何约束**，换个角度就穿帮！

### ❌ **核心矛盾**

> 我们需要"有体积的东西"来贡献颜色，但点云只有位置。给点加体积的尝试都失败了——要么几何错误，要么视角不一致。

## 四、NeRF：另辟蹊径，用神经网络表示场景

### NeRF 的关键跳跃（2020）

> "为什么不把整个场景存成一个**连续函数**呢？"

$$
\text{场景} = \text{MLP}_\theta(\mathbf{x}, \mathbf{d}) \rightarrow (\sigma, c)
$$

- **输入**: 3D位置 $\mathbf{x} \in \mathbb{R}^3$ + 视角方向 $\mathbf{d} \in S^2$
- **输出**: 密度 $\sigma > 0$（这个位置有多"实"）+ 颜色 $c \in [0,1]^3$

#### 🔍 **关键概念细化：MLP 到底长什么样？**

<details>
<summary>点击展开</summary>

**原始 NeRF (Mildenhall et al. 2020) 的网络结构**:

**步骤 1: 位置编码（Positional Encoding）**
```python
# 输入 x = [x_1, x_2, x_3] ∈ R^3 → γ(x) ∈ R^{18} (L=6 层频率)
γ(x) = [sin(2⁰*x), cos(2⁰*x), sin(2¹*x), cos(2¹*x), ..., sin(2⁵*x), cos(2⁵*x)]
```

**为什么？** 高频正弦编码让 MLP 能学习细节（否则 MLP 倾向于低频平滑函数）。

**步骤 2: 主网络（密度 + 粗略颜色）**
```
γ(x) [18 维] → FC[256]→ ReLU × 8 层 → [σ, p]
                                      ↑
                                   [1+3 维]
# σ: 密度（sigmoid+exp 保证>0）  
# p: 3 维向量，用于球谐函数计算颜色
```

**步骤 3: 颜色子网络（视角相关）**
```
[γ(x)[6 维], γ(d)[4 维], p[3 维]] → FC[128]→ ReLU × 2 层 → c ∈ RGB

# d: 视线方向 (view direction)  
# γ(d): 只有低频编码（颜色变化是平滑的）
```

**总参数量**: ~0.6M-1M，听起来不大？但每查询一次都要前向传播整个网络！

</details>

### ✅ **为什么 NeRF 质量好？**

**公理：MLP 是通用函数逼近器（Universal Function Approximator）** ——理论上可以表示任何连续函数。

这意味着：
- ✅ **无锯齿**: 场景是连续的，不是离散的像素/体素
- ✅ **视角相关颜色**: 输入 $\mathbf{d}$ 可以建模镜面反射、各向异性材质
- ✅ **无限分辨率**: 想渲染多高分辨率都行（只要计算跟得上）

### ❌ **NeRF 的原罪：慢到无法使用**

#### 第一重灾难：每像素要采样上百次

为什么？Volume Rendering Equation 是积分，没有解析解，必须用数值方法（Ray Marching）离散近似。

**计算量估算**（1920×1080 图像）:
- **像素数**: $1920 \times 1080 \approx 2\times10^6$
- **每像素采样**: 100 次（保守估计）
- **总采样点**: $2\times10^8$（**2 亿个**）
- **每个采样点**: 查询 MLP（8 层，256→512 神经元）≈ **4-10 万次 FLOPs**
- **总算力需求**: $2\times10^8 \times 5\times10^4 \approx 10^{13} \text{ FLOPs} = \mathbf{10\ TeraFLOPs/帧}$

对比 RTX 4090（~80 TFLOPS FP32）:
- **理论极限**: $10 / 80 = 0.125$ 秒 = 125ms/帧
- **实际 NeRF**: **30 秒/帧**
- **差距**: 约 240 倍！

#### 🔍 **关键概念细化：为什么理论极限和实际差距这么大？**

<details>
<summary>点击展开</summary>

**1. FLOPs vs 实际运行时间**
- **FLOPs（浮点运算数）**: 只是计算量的理论值
- **实际速度**还取决于：
  - 内存访问模式（GPU 显存带宽瓶颈）
  - 指令级并行度（能否充分利用 GPU Core）  
  - 分支预测失败（条件跳转导致流水线停顿）

**2. NeRF 的内存访问问题**
```python
# 每射线采样点的位置是完全不规则的！
for pixel in image:
    for sample_t in [t_1, t_2, ..., t_100]:
        x = ray_origin + sample_t * ray_direction  # 随机位置
        color = MLP(x)  # 每次访问不同的权重缓存行 → Cache Miss!
```

**GPU 利用率分析**:
- **理想情况**: 80 TFLOPS / (2 T FLOPs/帧) ≈ 25ms/帧  
- **NeRF 实际 GPU 利用率**: <10%（因为随机内存访问）
- **瓶颈在显存带宽，不是计算能力！**

**3. 对比：为什么 3DGS 能跑满 GPU？**
```python
# 3DGS: 批量处理，规则内存访问
for gaussian in all_gaussians:
    # 连续读取高斯参数（GPU 友好）
    mu, Sigma, color = gaussian_buffer[i]  
    project_to_2d(mu, Sigma)  # 批量矩阵运算，SIMD 友好
```

**这就是为什么架构设计（隐式 vs 显式）对性能的影响远超理论 FLOPs。**

</details>

#### 第二重灾难：训练时间以天计

$$
\begin{aligned}
\text{每张图片}: & \quad 2\times10^8 \text{次 MLP 查询} \\
\text{每个 epoch}: & \quad \text{假设 10 张图} \times 200\text{s} = 2000\text{s} \\
30k\text{训练步} \approx 1500\text{ epochs} & \rightarrow \mathbf{83\ \text{小时}}
\end{aligned}
$$

**原因**: 每步训练都要做完整的 Ray Marching + MLP 前向。

## 五、第一性分析：NeRF 的结构性瓶颈

回到公理层面：

| 公理 | 含义 |
|------|------|
| A1 | 场景 = MLP 表示的连续函数（隐式）|
| A2 | 渲染需要沿射线积分（Volume Rendering）|
| A3 | **MLP 是黑盒**——给定 $\mathbf{x}$，必须前向传播才能知道 $\sigma(\mathbf{x})$ |

### 从这些公理能推出什么？

> **推论**：每帧的所有采样点都要重新计算 MLP → **结构性慢，无法避免**

这不是工程问题。NeRF 的表示方式（隐式函数）决定了查询必须是 $O(\text{采样点数})$。

## 六、3DGS 的关键跳跃：换表示！

### "啊哈！"时刻

到这一步你看到两个方向：
1. **优化 NeRF** (Instant-NGP 等): 用 Hash Grid 缓存 MLP，加速查询
2. **换个思路**: 如果不用 MLP 呢？

**3D Gaussian Splatting (2023) 选择了后者**。核心想法：

> "既然点云的问题是'点是零维的'，那为什么不直接给每个点一个**3D 椭球体积**（高斯分布），然后利用高斯的数学性质**直接投影到 2D**？"

### ✅ **为什么是高斯？**——数学上的必然性

假设你在 3D 空间有一个高斯分布：

$$G_{3D}(\mathbf{x}) = \exp\left(-\frac{1}{2} (\mathbf{x} - \boldsymbol{\mu})^\top \boldsymbol{\Sigma}^{-1} (\mathbf{x} - \boldsymbol{\mu})\right)$$

其中：
- $\boldsymbol{\mu}$ 是中心（3D 位置）
- $\boldsymbol{\Sigma}$ 是协方差矩阵（决定椭球形状，对称正定）

### 🔍 **关键概念细化：为什么高斯能解决采样问题？**

<details>
<summary>点击展开</summary>

**核心性质**：3D 高斯经过透视投影后，在 2D 平面上**仍然是高斯**！这是概率论中的经典结论。

**直观理解**:
- 3D 高斯是一个连续的"云团"，有体积、可积分
- 投影到 2D 时，它变成一个椭圆形的 2D 高斯，可以精确计算覆盖的像素区域
- **解析解**：不用采样！直接算参数。

**对比点云**:
- 点云：点是零维 → 投影后还是点 → 无法贡献颜色（除非刚好命中）
- 3D 高斯：有体积 → 投影后有面积 → 可以用 Alpha Blending 混合多个半透明高斯 ✅

</details>

### **2D 投影的闭式解**（简化版）:

$$
\begin{aligned}
&\text{1. 世界→相机坐标: } \mathbf{\mu}_c = R \cdot (\boldsymbol{\mu}_w - \mathbf{C}) \\
&\text{2. 透视投影到像素: } \boldsymbol{\mu}_{2D} = K \cdot [\mu_{c,x}/\mu_{c,z}, \mu_{c,y}/\mu_{c,z}, 1]^\top \\
&\text{3. 协方差变换（误差传播）: } \boldsymbol{\Sigma}_{2D} \approx (KJ) \cdot \boldsymbol{\Sigma}_c \cdot (KJ)^\top
\end{aligned}
$$

其中：
- $K$ = 相机内参矩阵
- $R, \mathbf{C}$ = 外参（旋转 + 平移）  
- $J = \frac{\partial(x,y)}{\partial(X,Y,Z)}$ 是透视投影的**雅可比矩阵**，在高斯中心 $\boldsymbol{\mu}_c$ 处计算

#### 🔍 **关键概念细化：雅可比矩阵 J 是怎么来的？**

<details>
<summary>点击展开</summary>

透视投影公式将 3D 点 $(X,Y,Z)$ 映射到 2D 像素坐标 $(x,y)$：

$$\Pi(X,Y,Z) = \left(\frac{X}{Z}, \frac{Y}{Z}\right)$$

求偏导得到雅可比矩阵 $J = \frac{\partial(x,y)}{\partial(X,Y,Z)}$：

$$
J = \begin{bmatrix}
\frac{\partial x}{\partial X} & \frac{\partial x}{\partial Y} & \frac{\partial x}{\partial Z} \\
\frac{\partial y}{\partial X} & \frac{\partial y}{\partial Y} & \frac{\partial y}{\partial Z}
\end{bmatrix} = 
\begin{bmatrix}
1/Z & 0   & -X/Z^2 \\
0   & 1/Z & -Y/Z^2
\end{bmatrix}
$$

**物理意义**: 描述 3D 空间在 $\boldsymbol{\mu}_c$ 附近如何被拉伸/压缩到 2D。

**为什么用 $J \Sigma J^\top$？**——这是概率论中的**误差传播定律**（一阶泰勒近似）。当高斯足够小（$\sigma \ll z$）时，线性化近似效果很好。

</details>

### ✅ **这个性质意味着什么？**

- ✅ **有解析解**: 不用采样，直接算！
- ✅ $O(1)$ 复杂度：每个高斯一次投影就搞定
- ✅ **保持连续性**: 高斯本身就是连续函数

## 七、3DGS vs NeRF：根本区别

| 维度 | NeRF | 3DGS |
|------|------|------|
| **表示形式** | MLP 权重（隐式，黑盒）| 高斯参数（显式，可读）|
| **查询方式** | 输入坐标→前向传播（慢）| 直接读取参数（快）|
| **渲染管线** | Ray Marching 采样 → 积分 | 投影 → 排序 → Alpha Blending |
| **每像素计算量** | ~100 次 MLP 查询 | $O(\text{覆盖该像素的高斯数})$ |

### **速度为什么提升这么多？**（第一性分析）

回到我们的 1920×1080 场景：

```math
\begin{aligned}
&\text{NeRF: } \\
&\quad \bullet \; 2\times10^6 \text{ 像素} \times 100 \text{ 采样} = 2\times10^8 \text{次 MLP 查询} \\
\\
&\text{3DGS（典型场景）:} \\
&\quad \bullet \; \approx 10^6 \text{ 个高斯} \\
&\quad \bullet \; \text{每个高斯投影一次} \rightarrow O(1)\text{计算} \\
&\quad \bullet \; \text{总操作数: } \approx 10^6 \text{次投影} \\
\\
&\text{对比：理论加速比} \approx \frac{2\times10^8}{10^6} = \mathbf{200\text{ 倍}} \text{（2 亿 vs 1 百万）}
\end{aligned}
```

而且这还没算上 GPU 利用率的差距（NeRF <10%，3DGS >70%）。

### 🔍 **关键概念细化：3000 倍加速是怎么算的？**

<details>
<summary>点击展开</summary>

**从两个维度对比**:

**1. FLOPs 减少 (理论计算量)**:
```math
\begin{aligned}
&\text{NeRF: } O(\text{像素数} \times \text{采样数}) = 2\times10^6 \times 100 = 2\times10^8 \text{次 MLP 查询} \\
&\text{3DGS: } O(\text{高斯数}) = 10^6 \text{次投影} \\
\\
&\text{理论加速比：}\frac{2\times10^8}{10^6} = \mathbf{200\text{ 倍}}
\end{aligned}
```

**2. GPU 利用率提升（实际运行效率）**:
```
NeRF:
- 随机内存访问 → Cache miss 率高  
- 每射线独立计算 → 难以向量化
- GPU 利用率 < 10%
- 30 秒/帧 @ RTX 4090

3DGS:
- 连续读取高斯参数 → Cache friendly
- 批量矩阵运算 → SIMD 友好  
- GPU 利用率 > 70%
- <10ms/帧 @ RTX 4090

实际加速比：30000ms / 10ms = **3000 倍**
```

**为什么差距这么大？**
- NeRF 的瓶颈在**显存带宽**（random access）
- 3DGS 的瓶颈在**计算能力**（compute bound，可以跑满 GPU）
- 从 memory-bound → compute-bound = **10-20 倍额外加速**

```math
\text{总加速比} = (\text{FLOPs 减少}) \times (\text{GPU 利用率提升})
               \approx 200 \times 15 
               \approx 3000 \text{ 倍 ✅}
```

</details>

### **实际性能对比**（来自官方论文）

| 方法 | 训练时间 | 推理速度 | PSNR（质量） | 显存占用 |
|------|----------|----------|--------------|----------|
| NeRF（原始） | 1-2 天 | **30 秒/帧** | ~32 dB | <1GB |
| Instant-NGP | 30 分钟 | 10-50ms/帧 | ~32 dB | ~2GB |
| **3DGS** | **~30 分钟** | **<10ms（60+fps）** | **~34 dB** | **~150MB** |

**直观理解**: 3DGS 比原始 NeRF 快**3000 倍**，质量还更好！

```
体素 (Voxel)
```

1. 接受点云作为起点（稀疏性已验证有效）✅

第二点才是关键——不是所有人都能从"需要体积"跳到"用高斯表示体积"。这需要知道：
- 高斯的投影封闭性（概率论知识）

### Q2: "3DGS 的 150MB 内存是怎么来的？"

假设 1M 个高斯，每个存储哪些参数？试着算一下...

<details>
<summary>提示</summary>
- 位置 $\boldsymbol{\mu}$: 3 个 float (x, y, z)  
- 颜色 $c$: 3 或 5 个 SH 系数
- 协方差 $\Sigma$: 用 3 个缩放 +4 个四元数（避免对称矩阵的冗余）
- 不透明度 $\alpha$: 1 个值  
→ 大约 52 字节/高斯，1M ≈ 52MB。实际 150MB 是因为...
<details>
<summary>答案</summary>
**单高斯参数估算**:
- 位置 $oldsymbol{\mu}$: 3 float (x, y, z)  
- 颜色 $c$: 5 SH 系数（order=1）
- 协方差 $\Sigma$: 3 scale + 4 quaternion（共 7 个值，避免对称矩阵冗余）
- 不透明度 $lpha$: 1 float

→ **约 28 floats × 4 bytes = 112 bytes/高斯**  
→ 1M 高斯 ≈ **112 MB**

实际 3DGS 实现中还会存储：
- 更多 SH 系数（如果 order>1）
- 梯度缓冲、临时变量等 overhead

所以最终达到 ~150MB/M，合理！
</details>

### Q3: "3DGS 牺牲了什么来换取速度？"

NeRF 可以表达视角相关的颜色（镜面反射）。3DGS 能吗？为什么？

<details>
<summary>思考方向</summary>
- NeRF：输入 $(\mathbf{x}, \mathbf{d}) \rightarrow c(\mathbf{x}, \mathbf{d})$，显式建模视角依赖  
- 3DGS：每个高斯的颜色是固定的...但有**View-Dependent Appearance**！下一章会讲到如何加入视角相关项。
<details>
<summary>答案</summary>
**牺牲的代价**:

1. **视角相关能力受限**: 
   - NeRF: $c(\mathbf{x}, \mathbf{d})$ 显式依赖视角 → 可以表达复杂镜面反射
   - 3DGS: 用 SH 系数近似 → order=1 只能表达简单高光，order=3 才有较好效果

2. **几何精度**: 
   - 高斯是"软"表示 → 表面不如 NeRF 的等值面精确
   - 需要更多高斯才能逼近锐利边缘

3. **训练稳定性**:
   - densify/split/prune 引入离散操作 → 梯度不连续
   - NeRF 是全连续优化

**但换来的是**: 
- 推理速度：**实时渲染**（60+ fps）vs NeRF 的秒级/帧
- 内存效率：显式表示 vs 隐式 MLP 每帧重算
