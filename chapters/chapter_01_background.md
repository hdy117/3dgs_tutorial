# 第1章：为什么我们需要3DGS？——从NeRF的瓶颈到突破

**学习框架**：`问题 → 现有方案 → 矛盾 → 新突破`

**核心目标**：理解NeRF为什么慢，以及3DGS是如何在**不牺牲质量**的前提下实现**100倍加速**的

---

## 一、你要解决什么问题？

### 1.1 视图合成：从照片到新视角

**想象一下**：
- 你拍了10张房间的照片（不同角度）
- 你想知道：如果站在房间那个未拍摄的位置，会看到什么？

**这就是视图合成（Novel View Synthesis）**：
- 输入：M张已知视角的照片 + 每张的相机参数
- 输出：任意新视角的照片

**关键挑战**：
- 怎么用有限照片"理解"3D场景？
- 怎么从新相机位置"渲染"出合理图像？

---

### 1.2 两个核心需求（看似矛盾）

**需求A：高质量**
- 渲染图要接近真实照片
- 边缘锐利，无锯齿
- 细节丰富（纹理、几何）

**需求B：实时性**
- 游戏/AR需要60帧/秒（16ms/帧）
- 自动驾驶仿真需要快速生成新视角
- 不能等30秒才出一张图

**矛盾**：早期方法要么质量好但慢，要么快但质量差

---

## 二、NeRF：质量标杆，但慢得无法忍受

### 2.1 NeRF的核心思想（为什么它质量好）

**核心洞见**：
> 把3D场景看作一个**连续函数**：输入(位置, 视角) → 输出(颜色, 密度)

**数学表达**：
```
场景 = MLP神经网络 θ
输入: x ∈ ℝ³ (3D位置), d ∈ ℝ³ (视角方向)
输出: σ(x) > 0 (密度), c(x,d) ∈ [0,1]³ (颜色)

渲染: 沿射线积分 C = ∫ T(t)·σ·c dt, T(t)=exp(-∫₀ᵗσds)
```

**为什么质量好？**
- MLP是通用函数逼近器，可以表示任意复杂场景
- 连续函数 → 渲染无锯齿
- 视角相关颜色 → 可以表达镜面反射

---

### 2.2 NeRF的"三宗罪"：为什么慢

**罪1：每像素都要采样100-200次**

```
1920×1080 = 207万像素
每像素采样100次 = 2亿次采样点
每次采样：MLP前向 ≈ 10,000次浮点运算
总计算量：2亿 × 1万 = 2×10¹² FLOPs = 2 TeraFLOPs!
```

**理论极限**（RTX 4090, 80 TFLOPS）：
- 2 / 80 = 0.025秒 = **25ms**

**实际NeRF**：30秒/帧
**差距**：约**1200倍**

**问题在哪？**
- MLP查询有内存访问开销
- 不能完全峰值算力
- 但核心是：**2 TeraFLOPs/帧就是太大**

---

**罪2：ray marching本质离散**

**采样-质量权衡**（无解）：
- 采样64次：快但噪点多
- 采样256次：质量好但慢2倍
- **为什么必须采样？**
  - 连续函数的积分无解析解
  - 必须数值近似
  - **数学必然，无法避免**

---

**罪3：训练时间以天计**

```
每帧2亿次MLP查询
每epoch ≈ 200秒（2M像素 × 100采样 × MLP时间）
30k步 ≈ 1500 epochs → 300,000秒 ≈ 83小时
（实际优化后仍需数小时-天）
```

**原因**：训练时每张图都要跑完整ray marching管线

---

### 2.3 NeRF的瓶颈根源（第一性分析）

**公理**：
1. 场景是连续函数（MLP编码）
2. 渲染需要沿射线积分
3. MLP是黑盒，无法直接查询3D结构

**矛盾**：
- 我们需要快速知道"某个3D位置的颜色和密度"
- 但MLP必须通过输入坐标计算，无法缓存3D信息
- 每帧所有采样点都要重算 → **结构性慢**

**解决方向**：
```
方向1：预计算3D结构 → 缓存（Instant-NGP: hash grid）
方向2：改变表示 → 可直接投影的primitive（3DGS: 高斯）
```

---

## 三、3DGS的突破：全都要

### 3.1 核心思想（一句话）

> 不用MLP，改用**显式高斯**作为primitive，直接投影到2D，避免ray marching

**关键区别**：

| 维度 | NeRF | 3DGS |
|------|------|------|
| 表示 | MLP权重（隐式）| 高斯参数（显式）|
| 渲染 | ray marching采样 | 直接投影+混合 |
| 查询 | 每点MLP（慢）| 直接读取参数（快）|

---

### 3.2 为什么高斯能直接投影？

**核心公式**（第3章详细推导）：
```
3D高斯 G(x) = exp(-½(x-μ)ᵀΣ⁻¹(x-μ))
投影到2D：μ_2d = K·(Rμ+T), Σ_2d = J·(RΣRᵀ)·Jᵀ

其中J是透视投影的雅可比矩阵
```

**关键洞察**：
- 3D高斯投影到2D**仍是高斯**（投影封闭性）
- 有**闭式解**（不用采样）
- 每个高斯O(1)计算 → 总复杂度O(N)，N=高斯数量

**对比NeRF**：
- NeRF：每像素100次MLP查询 = 2亿次
- 3DGS：每个高斯1次投影 = 100万次（N=1M）
- **加速100倍**（1M vs 100M）

---

### 3.3 性能对比（真实数据）

| 方法 | 训练时间 | 推理时间/帧 | PSNR | 内存 |
|------|----------|-------------|------|------|
| NeRF（原始） | 1-2天 | **30秒** | 32-33 dB | <1GB |
| Instant-NGP | 30分钟 | 10-50ms | 32-33 dB | ~2GB |
| **3DGS** | **30分钟** | **<10ms** | **33-35 dB** | **~150MB** |

**直观对比**（对数尺度）：
```
速度 (秒/帧)
   ^
 30 |                    NeRF
    |
  1 |          NeRF++
    |
0.05|     Instant-NGP  3DGS
    |
0.01|___________________
     体素  点云  NeRF  3DGS
```

**3DGS的成就**：
- 推理速度：比NeRF快**3000倍**（30秒 → 0.01秒）
- 质量：PSNR 33-35 dB，**不输NeRF**
- 内存：150MB（比Instant-NGP还小）

## 四、从问题到方案：完整的推理链

### 4.1 约束传递：每一代解决了什么，遗留什么

```
第1代：体素
  问题：内存V³立方增长（512³=134M体素 → 512MB）
  解决：→ 只存表面（点云）
  遗留：点零维度，无体积

第2代：点云
  问题：点没有面积，投影到2D有空隙
  解决：→ 给点加体积（尝试点精灵、2D高斯）
  遗留：2D高斯无3D几何约束，透视错误

第3代：NeRF
  问题：需要连续质量
  解决：→ MLP表示连续函数
  遗留：MLP查询慢，ray marching离散

第4代：3DGS
  问题：怎么避免MLP + 保持连续质量？
  解决：→ 显式高斯 + 直接投影（投影封闭性）
  结果：✅ 稀疏（O(N)）+ 连续（高斯平滑）+ 快（O(1)投影）
```

**洞察**：每一代的"遗留问题"就是下一代的起点。3DGS的突破是**识别并解决**了NeRF的结构性瓶颈。

---

### 4.2 为什么之前没想到？

**认知障碍**：
1. **从NeRF出发**：认为MLP是必须的（连续表示）
2. **从点云出发**：不知道怎么给点加体积且保持几何正确

**3DGS的关键跳跃**：
> "如果我在3D空间给每个点定义一个**椭球**（高斯），它投影到2D仍是高斯，可以直接算贡献"

**这个想法简单，但需要意识到**：
- 点云是起点（稀疏性已证明）
- 需要体积（连续质量）
- 高斯是唯一投影解析的形状（数学事实）

**"啊哈！"时刻**：不是从NeRF优化，而是**换表示**。

## 六、思考题

**1. 如果NeRF这么快，为什么大家还说它慢？**
- 30秒/帧在离线渲染可以接受，但实时应用（AR/游戏）不可接受
- 训练时间太长（几天），迭代成本高
- 思考：你的应用场景需要多快？

**2. 3DGS的150MB内存合理吗？**
- 1M高斯 × 52字节 = 52MB
- 为什么实际150MB？还有哪些开销？
- 如果场景更大（10M高斯），内存会怎样？
