## 一、Problem：3D重建的困境

### 1.1 3D重建的定义与目标

**定义**：从多视角2D图像恢复3D场景表示，使得可以从任意新视角渲染图像

**核心公式**（视图合成问题）：
```
给定：{ (I_i, C_i) }，i=1..M  # M张图像 + 相机参数
求：  场景表示 S
使得：Render(S, C_new) ≈ I_new  # 对新相机C_new的渲染接近真实图像
```

### 1.2 痛点：NeRF的渲染瓶颈（量化分析）

**NeRF性能数据**（基于原始论文及后续优化）：

| 方法 | 训练时间 | 推理时间/帧 | PSNR | 内存 |
|------|----------|-------------|------|------|
| NeRF（原始） | 1-2天 | 30秒 | 32-33 dB | <1GB |
| NeRF++ | 6-12小时 | 10秒 | 33-34 dB | <1GB |
| Instant-NGP | 30分钟 | 10-50ms | 32-33 dB | ~2GB |
| **3DGS** | **30分钟** | **<10ms** | **33-35 dB** | **~150MB** |

---

#### 1.2.1 核心概念：Ray（射线）

**定义**：
在计算机图形学中，**ray**（射线）是从相机光心出发，穿过成像平面每个像素的无限长直线。

**数学表示**：
```
相机位置: O ∈ ℝ³
像素坐标: (u, v) ∈ ℝ²（归一化设备坐标）
射线方向: d = (R · K⁻¹ · [u, v, 1]ᵀ)  # 世界坐标系方向
射线参数化: r(t) = O + t·d, t ≥ 0
```

**可视化**：

```
    相机位置 O
        *
        |\
        | \
        |  \ 射线方向 d
        |   \
        |    \
        |     \
        |      * 像素(u,v)
        |     /
        |    /
        |   /
        |  /
        | /
        |/
        * 3D场景
```

**为什么需要ray marching？**
- 射线与3D场景相交 → 需要知道沿途有哪些物体
- 对于体素/NeRF这种"连续场"，需要采样多个点来近似积分
- 采样点越多，积分越准确，但越慢

**Ray marching算法**（体素/NeRF通用）：
```
for each pixel (u,v):
    d = compute_ray_direction(u,v)
    t = t_near
    while t < t_far:
        x = O + t·d  # 采样点
        density = query_scene(x)  # 查询体素值或MLP
        color += accumulate_color(density, t)
        t += step_size
```

**复杂度**：
- 每像素采样数：N_s ≈ 100-200
- 总采样数：H·W·N_s（如1920×1080×100 ≈ 2亿次）
- 这是NeRF慢的**根本原因之一**

---

#### 1.2.2 核心概念：Voxel（体素）

**定义**：
**Voxel**（体素像素）是3D空间中的最小单元，类比2D像素。体素网格将3D空间离散化为规则的网格。

**数学表示**：
```
网格分辨率: V × V × V
体素坐标: (i, j, k) ∈ [0, V-1]³
世界坐标: x = (i/V)·size + offset
体素值: V[i,j,k] ∈ ℝ 或 RGB
```

**内存计算**：

```
+------------------+-------------------+-----------------+
| 分辨率 V         | 体素数 = V³       | 内存 (float32)  |
+------------------+-------------------+-----------------+
| 256              | 16.7M             | 64 MB           |
| 512              | 134M              | 512 MB          |
| 1024             | 1.07B             | 4 GB            |
| 2048             | 8.59B             | 32 GB           |
+------------------+-------------------+-----------------+
```

**内存增长规律**：
```
内存 ∝ V³
V提高2倍  →  内存增加 2³ = 8倍
V提高4倍  →  内存增加 4³ = 64倍
```

**体素渲染流程**：

```
Ray → 沿ray采样t → 查询体素网格(tri-linear) → 获取density/color → 累加颜色
          ↑
       [采样点]
```

**体素的"内存墙"问题**：
- 要渲染高质量图像 → 需要高分辨率V
- V=1024时内存4GB → 单卡尚可
- V=2048时内存32GB → 不可能
- 因此体素**不可扩展**

---

#### 1.2.3 核心概念：PSNR（峰值信噪比）

**定义**：
**PSNR**（Peak Signal-to-Noise Ratio）是衡量图像质量的客观指标，表示信号（原始图像）与噪声（重建误差）的功率比。

**公式**：
```
MSE = (1/(H·W·C)) · Σ(I_pred - I_gt)²
PSNR = 10 · log₁₀(MAX² / MSE)

其中:
- MAX: 像素最大值（通常为1.0或255）
- MSE: 均方误差
- H,W,C: 图像高、宽、通道数
```

**单位**：分贝（dB）

**PSNR解释表**：

```
+----------------+------------------+----------------------+
| PSNR (dB)      | 质量描述         | 视觉感受             |
+----------------+------------------+----------------------+
| > 40           | 优秀             | 几乎无差异           |
| 35-40          | 良好             | 细微差异，不易察觉   |
| 30-35          | 中等             | 明显差异，但可接受   |
| 25-30          | 较差             | 明显伪影             |
| < 25           | 差               | 严重失真             |
+----------------+------------------+----------------------+
```

**NeRF vs 3DGS的PSNR对比**：
- NeRF: 32-33 dB（良好）
- 3DGS: 33-35 dB（良好到优秀）
- **差距：1-2 dB**（人眼几乎无法区分）

**PSNR的局限性**：
- ❌ 不考虑人类视觉系统（HVS）
- ❌ 对结构信息不敏感（只看像素差异）
- ✅ 计算简单，广泛使用

---

#### 1.2.4 其他质量指标

**SSIM（结构相似性指数）**：
```
SSIM(x,y) = (2μ_xμ_y + C1)(2σ_xy + C2) / (μ_x²+μ_y²+C1)(σ_x²+σ_y²+C2)
```
- 范围：[0, 1]
- 越接近1越好
- 考虑局部结构，比PSNR更符合人眼

**LPIPS（学习感知图像块相似度）**：
- 使用深度网络特征计算感知距离
- 越低越好
- 最符合人眼主观感受

### 1.3 问题本质提炼

从NeRF的缺陷中，提取**不可约的事实（Axioms）**：

| Axiom（公理） | 解释 | 对设计的影响 |
|---------------|------|--------------|
| **渲染需要投影** | 3D→2D必须有几何变换 | 表示必须支持快速投影（O(1)查询） |
| **表面需要覆盖** | 每个像素由3D元素负责 | 表示必须有"面积"（不是点） |
| **质量要求连续** | 不能有锯齿/点状伪影 | 需要连续概率分布（不是离散体素） |
| **速度要求可缓存** | 3D表示应能直接投影 | 避免逐样本复杂计算（MLP） |

## 二、Starting Point：现有方案的局限性

### 2.1 方案对比矩阵

| 维度 | 体素 | 点云 | NeRF | **3DGS目标** |
|------|------|------|------|--------------|
| **查询速度** | ✅ O(1)索引 | ✅ O(1) | ❌ O(网络) | ✅ O(1) |
| **渲染质量** | ❌ 块状 | ❌ 点状 | ✅ 连续 | ✅ 连续 |
| **内存效率** | ❌ V³立方 | ✅ 稀疏O(N) | ✅ 稀疏O(N) | ✅ 稀疏O(N) |
| **可微性** | ✅ | ✅ | ✅ | ✅ |
| **表面表示** | ❌ 离散边界 | ❌ 零维度 | ✅ 连续场 | ✅ 椭球叠加 |
| **投影方式** | ray marching | 直接投影（2D贴纸） | ray marching | **直接投影（3D→2D）** |
| **速度瓶颈** | ray采样数 | 无"体积" | MLP查询 | 无 |
| **典型N** | 10⁸-10⁹ | 10⁵-10⁶ | 10⁵-10⁶ | 10⁵-10⁶ |

### 2.2 体素的"死胡同"

**内存需求计算示例**：

```
假设场景分辨率需求:
V = 512 → 512³ = 134,217,728个体素
内存 = 134M × 4字节 = 536,870,912 字节 ≈ 512 MB

V = 1024 → 1024³ = 1,073,741,824个体素
内存 = 1.07G × 4字节 = 4,294,967,296 字节 ≈ 4 GB

V = 2048 → 2048³ = 8,589,934,592个体素
内存 = 8.59G × 4字节 = 34,359,738,368 字节 ≈ 32 GB
```

**结论**：体素内存 O(V³) → 分辨率提高2倍，内存8倍 → **不可扩展**

**边界锯齿问题**：

```
理想光滑表面:  ~~~~~~~~~~~~~~~~~~~~~~
                (连续曲线)

体素表示表面:  `````````````````````
                (离散锯齿)
```

**原因**：离散网格 + 最近邻/三线性插值 → 边界必然不光滑

---

### 2.3 点云的"最后一公里"

**关键洞察**：**大部分3D空间是空的！**

```
3D空间占用率估算:
+----------------------+--------+--------+
| 区域类型             | 占比   | 说明   |
+----------------------+--------+--------+
| 空气/空区域          | 99.9%  | 无物体 |
| 物体表面             | 0.1%   | 有物体 |
+----------------------+--------+--------+
```

**内存对比**：
- 体素：V³（如512³ = 134M）
- 点云：N（表面点数，通常10⁵-10⁶）
- **内存减少：100-1000倍** ✅

**点云渲染的"三次尝试"**：

```
点云: 位置 + 颜色
    |
    +---> 尝试1: 点精灵（面向相机的方块）
    |       问题: 点间有空隙，不连续
    |
    +---> 尝试2: Splatting（2D高斯圆）
    |       问题: 只是2D贴纸，无3D几何约束
    |             不同视角下尺度无法正确变化
    |
    +---> 尝试3: Poisson重建 → Mesh
            问题: 过程复杂，丢失细节
```

**关键问题**：
> 点云是"贴纸"，不是"物体"。如何给点赋予"体积"？

**突破口的思考**：

```
如果在3D空间给每个点定义一个"影响范围"（椭球），
让它自然投影到2D，会怎样？

3D点 + 3D椭球体积  →  投影 → 2D高斯
        ↑
    这就是3DGS的核心思想
```

---

### 2.4 NeRF的"结构性瓶颈"

**NeRF渲染管线**：

```
相机 → 生成ray（原点,方向）
      → 沿ray采样t₁...tₙ
      → 每个采样点查询MLP：query(x(t), d) → (color, density)
      → volume rendering累加
      → 输出像素颜色
```

**瓶颈定位**：
- **MLP查询**：每个样本都要跑网络，无法避免
- **ray marching**：采样点数量与质量/速度正相关

**理论下限计算**：

```
假设:
  每ray采样数 N_s = 100
  每样本MLP FLOPs ≈ 10,000
  图像分辨率 1920×1080 → ray数 N_p ≈ 2,000,000

总FLOPs/帧 = N_p × N_s × 10⁴
            = 2M × 100 × 10⁴
            = 2×10¹² FLOPs = 2 TeraFLOPs!

GPU算力参考:
  RTX 4090 ≈ 80 TFLOPS
  理论最快时间 = 2 / 80 = 0.025秒 = 25ms
  
实际:
  MLP不是峰值算力，有内存访问开销
  实际MLP查询 ~100ns/样本
  单帧时间 = 2M × 100 × 100ns = 20,000秒？不对
  
  实际NeRF原始论文: 30秒/帧
  优化后(Instant-NGP): 10-50ms
```

**NeRF的"三宗罪"**：

1. **MLP查询慢**
   - 无法通过硬件线性加速
   - 每帧必须重新计算（无法缓存）

2. **ray marching本质离散**
   ```
   采样点N_s ↑ → 质量↑ 但 速度↓
   N_s=64: 快但噪点多
   N_s=128: 平衡
   N_s=256: 质量好但慢2倍
   ```

3. **训练时间长**
   ```
   每ray 1ms (含MLP) → 每epoch 200秒
   30k步 ≈ 1500 epochs → 300,000秒 ≈ 83小时
   (实际有优化，但仍需数小时-数天)
   ```

## 三、核心矛盾：我们要突破什么？

### 3.1 需求对比表

```
+----------------+------+-------+------+----------------+
| 需求           | 体素 | 点云  | NeRF | 理想方案        |
+----------------+------+-------+------+----------------+
| 快速投影       | ✅   | ✅    | ❌   | ✅ 直接投影      |
| 连续质量       | ❌   | ❌    | ✅   | ✅ 连续          |
| 稀疏内存       | ❌   | ✅    | ✅   | ✅ 稀疏          |
| 可微分         | ✅   | ✅    | ✅   | ✅              |
| 表面面积       | ❌   | ❌    | ✅   | ?              |
+----------------+------+-------+------+----------------+
```

**缺失的一环**：**有体积的稀疏表示**

---

### 3.2 突破方向决策树

```
需要稀疏+快速+连续
        |
        +---> 给点加"体积"
                |
                +---> 3D立方体?  → 问题: 投影复杂，边界不连续 → ✗
                |
                +---> 3D球体?    → 问题: 只能各向同性，无法拉长 → ✗
                |
                +---> 3D高斯?    → ✅ 投影解析，协方差完整描述 → ✓
```

**为什么其他形状不行？**
- **立方体**：投影是六边形，计算复杂，边界不连续
- **球体**：只能表示各向同性，无法表示拉长的椭球
- **高斯**：投影仍是高斯，解析公式，协方差完整描述形状

---

### 3.3 关键概念总结

| 概念 | 定义 | 为什么重要？ |
|------|------|--------------|
| **Ray** | 从相机出发的射线，用于采样3D场景 | NeRF慢的根源：每ray需多次MLP查询 |
| **Voxel** | 3D空间离散化的最小单元 | 内存O(V³)不可扩展，质量锯齿 |
| **PSNR** | 峰值信噪比，衡量图像重建质量 | NeRF:32-33dB, 3DGS:33-35dB，差距<2dB |
| **稀疏性** | 只存可见表面，N << 空间体积 | 内存效率100-1000倍 |
| **直接投影** | 3D→2D有解析公式，无需采样 | 速度O(1) per primitive |
