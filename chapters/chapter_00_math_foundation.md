## 二、线性代数——为什么它是 3D 表示的"母语"？

### 2.1 问题起点："怎么用数字描述一个椭球？"

想象你要在代码里存储一个**3D 高斯椭球**。你需要记录什么信息？

#### 直觉尝试：用长轴、短轴、旋转角度
```python
gaussian = {
    'center': (x, y, z),           # 位置，好理解
    'axes_length': (a, b, c),      # 三个轴的长度
    'rotation': (roll, pitch, yaw) # 欧拉角旋转
}
```

**听起来很合理**——但马上遇到问题：

> **Q: 如果我要把这个椭球绕 Y 轴转 30°，怎么更新这些参数？**

你尝试推导：新的 `(a', b', c')`和`(roll', pitch', yaw')` 该怎么算？

很快发现：**欧拉角的组合非常复杂**（ gimbal lock、非交换性）。而且，"旋转后的半轴长度"根本不是一个简单公式能表达的。

---

### 2.2 被迫发明：协方差矩阵

你意识到：**与其分别存"形状 + 旋转"，不如直接存"旋转后的形状"**。

#### 核心概念：协方差矩阵 Σ

对于一个 3D 高斯分布，它的**形状和方向**可以用一个 3×3 的对称正定矩阵完全描述：

```math
Σ = [σ_xx  σ_xy  σ_xz]
    [σ_yx  σ_yy  σ_yz]
    [σ_zx  σ_zy  σ_zz]
```

**为什么是矩阵？**——因为它天然支持线性变换。

#### 关键性质：旋转的优雅表达

如果要把椭球用旋转矩阵 `R` 转一下，新协方差是：
```math
Σ' = R · Σ · R^T
```

就这么简单。**没有欧拉角的复杂转换，直接矩阵乘法**。

---

### 🔍 概念细化：什么是"对称正定矩阵"？

<details>
<summary>点击展开（可以跳过，先理解用法）</summary>

**对称性**：Σ = Σ^T，即 σ_ij = σ_ji

- 物理意义：x 对 y 的影响 = y 对 x 的影响
- 好处：9 个参数 → 6 个独立参数（节省存储）

**正定性**：对所有非零向量 v，都有 v^T Σ v > 0

- 几何意义：椭球的半轴长度都是正的（不能是虚数）
- 代数检验：所有特征值 λ_i > 0
- 代码检查：`torch.linalg.eigvalsh(Sigma).min() > 0`

**为什么必须正定？**
如果 Σ 不正定，高斯分布的归一化系数 `1/√|Σ|` 会变成虚数——概率密度函数失去物理意义。

</details>

---

### 2.3 从特征分解理解"形状 + 方向"

协方差矩阵虽然好用，但不够直观。**怎么看出椭球的三个半轴长度和方向？**

答案是：**特征值分解（Eigen Decomposition）**。

```math
Σ = V · Λ · V^T
```

其中：
- `Λ = diag(λ₁, λ₂, λ₃)` —— 对角矩阵，特征值是半轴长度的平方
- `V = [v₁ | v₂ | v₃]` —— 正交矩阵，特征向量是三个主轴方向

#### 代码示例：从 Σ 提取几何信息

```python
def sigma_to_geometry(Sigma):
    """协方差矩阵 → (半轴长度，旋转矩阵)"""
    # 特征分解
    eigenvalues, eigenvectors = torch.linalg.eigh(Sigma)
    
    # 半轴长度 = √特征值（高斯的标准差）
    axes_length = torch.sqrt(eigenvalues)  # [3]
    
    # 旋转矩阵：特征向量作为列向量
    R = eigenvectors  # [3, 3]，每一列是一个主轴方向
    
    return axes_length, R

# 反向操作：从几何信息重建 Σ
def geometry_to_sigma(axes_length, R):
    """(半轴长度，旋转) → 协方差矩阵"""
    Lambda = torch.diag(axes_length ** 2)  # 特征值 = 半轴²
    Sigma = R @ Lambda @ R.T
    return Sigma
```

**关键理解**：Σ 编码了"形状 + 方向"的混合信息，特征分解把它们拆开。

### 2.4 矩阵乘法的几何直觉

你在代码里看到的 `R @ mu`、`J @ Sigma @ J.T`——这些矩阵乘法到底在干嘛？

#### 核心直觉：矩阵 = 线性变换

```python
# 示例：把点从世界坐标系转到相机坐标系
mu_world = torch.tensor([1.0, 2.0, 3.0])  # 世界坐标
R = rotation_matrix()                      # 3×3 旋转
T = translation_vector()                   # 3×1 平移

mu_camera = R @ mu_world + T               # 相机坐标
```

**几何意义**：
- `R @ v`：把向量 v 旋转（保持长度不变）
- `+ T`：整体平移（改变位置）

#### 为什么矩阵乘法对应"复合变换"？

如果先转 R₁，再转 R₂：
```python
v' = R2 @ (R1 @ v) = (R2 @ R1) @ v
```

**结合律**让多次变换可以合并成一个矩阵——这是线性代数的核心优势。

---

### 🔍 概念细化：@ 符号和矩阵乘法规则

<details>
<summary>点击展开</summary>

Python/PyTorch 中 `A @ B` 表示矩阵乘法（不是元素相乘）。

**维度规则**：
```
A: [m, n]   B: [n, p]   →   A @ B: [m, p]
       ↓        ↓                ↓
    (行，列)  (行，列)         (行，列)
```

**关键**：A 的列数必须等于 B 的行数。

**常见错误**：
```python
v = torch.randn(3)        # shape: [3]
R = torch.randn(3, 3)     # shape: [3, 3]

R @ v      # ✅ OK → [3]
v @ R      # ❌ Error! [3] @ [3,3] 维度不匹配
v.T @ R    # ✅ OK → [1,3] @ [3,3] = [1,3]
```

**记忆技巧**：把 `@` 读作"作用在"，`(R @ v)` = "旋转 R 作用于向量 v"。

</details>

## 三、微积分——为什么梯度是"最优调整方向"？

### 3.1 问题起点："怎么让渲染图更接近真值？"

训练 3DGS 时，你有一个损失函数：
```python
loss = L1(render(mu, Sigma), ground_truth)
```

现在 `loss = 5.2`（很大），你想调整参数`(mu, Sigma)`让它变小。

**朴素想法**：随机改一点 mu，看看 loss 变没变小？

```python
for _ in range(1000000):
    delta = torch.randn_like(mu) * 0.01
    mu_new = mu + delta
    
    if compute_loss(mu_new) < compute_loss(mu):
        mu = mu_new  # 接受这个改动
```

**问题**：这要试多久？而且你根本不知道"改多少"合适。

---

### 3.2 被迫发明：导数与梯度

你观察到：**如果知道 loss 随 mu 的变化率，就能直接往下降方向走**。

#### 一维直觉：斜率告诉你该往哪走

```
loss(μ)
  ^
  |     /| 
5.2|    / |  ← 斜率为正（+0.5）
  |   /  |
  |  /   |
  | /    ↓
  |/_____|_______→ μ
        μ_current
```

如果 `d(loss)/dμ = +0.5`，说明：
- μ增加 → loss 增加（斜率正）
- μ减少 → loss 减少（下降方向是负方向）

**更新规则**：`μ_new = μ - learning_rate × gradient`

---

### 3.3 多维推广：梯度向量

但 `mu` 是三维的 `(x, y, z)`。这时候需要**偏导数**：

```math
∇loss = [∂loss/∂x, ∂loss/∂y, ∂loss/∂z]
```

**梯度的几何意义**：损失函数上升最快的方向（梯度下降就是往反方向走）。

#### 代码示例：PyTorch 自动求导

```python
mu = torch.tensor([1.0, 2.0, 3.0], requires_grad=True)

# 前向传播
rendered = render(mu, Sigma)
loss = L1(rendered, ground_truth)

# 反向传播（自动计算梯度）
loss.backward()

# mu.grad 就是 ∇loss
print(mu.grad)  # tensor([0.23, -0.45, 0.12])

# 更新参数
mu = mu - lr * mu.grad
```

---

### 🔍 概念细化：chain rule（链式法则）

<details>
<summary>点击展开</summary>

**问题**：`loss` 不直接依赖 `mu`，而是通过多层计算：
```
mu → projected_mu_2d → rendered_image → loss
```

怎么求 `∂loss/∂mu`？

**链式法则**：
```math
∂loss/∂mu = ∂loss/∂rendered × ∂rendered/∂projected × ∂projected/∂mu
```

**PyTorch 的自动微分**就是帮你做这件事——构建计算图，然后反向传播逐层应用链式法则。

**手动推导示例**（简化版）：
```python
# 假设 rendered = W @ mu （线性投影）
# loss = ||rendered - gt||²

# chain rule:
∂loss/∂mu = ∂loss/∂rendered @ ∂rendered/∂mu
          = 2*(rendered-gt) @ W
```

这就是为什么反向传播是 `error_signal @ weight_matrix` 的形式。

</details>

---

### 3.4 二阶导数：什么时候需要 Hessian？

你学到梯度下降后，可能会问："用二阶导数（曲率信息）会不会更快？"

#### Newton's Method

```math
μ_new = μ - H⁻¹ · ∇loss
```

其中 `H` 是 Hessian 矩阵（二阶偏导数）。

**听起来很美好**——考虑了曲率，应该收敛更快。但实际有致命问题：

| 方法 | 每步成本 | 3DGS 场景的可行性 |
|------|---------|------------------|
| Gradient Descent | O(N) | ✅ N=10⁶个高斯参数，可行 |
| Newton's Method | O(N²) 存储 + O(N³) 求逆 | ❌ N=10⁶ → Hessian 是 10¹²×10¹²的矩阵 |

**结论**：大规模优化只能用一阶方法（Adam、SGD）。二阶信息太贵。

## 四、概率论——为什么是高斯分布？

### 4.1 问题起点："点云渲染有漏洞，怎么办？"

第 2 章你发现：**点是零维的**，投影到屏幕大部分像素没被覆盖。

```
●    ●   ●      ← 点在 3D
↓ 投影
·    ·   ·      ← 屏幕上的洞！
```

**解决方案**：给每个点加"体积"——但它应该是什么形状？

---

### 4.2 候选方案对比

| 形状 | 优点 | 缺点 |
|------|------|------|
| 立方体 | 简单，容易渲染 | 边缘硬，旋转后锯齿严重 |
| 球体 | 各向同性，无方向偏好 | 无法表达拉伸/压扁的几何 |
| **高斯椭球** | ✅ 光滑衰减、可微分、解析投影 | ⚠️ 需要协方差矩阵（稍复杂） |

**为什么高斯赢了？**——因为它的数学性质完美匹配渲染需求。

---

### 4.3 高斯分布的核心性质

#### 1D 高斯公式

```math
𝒩(x; μ, σ²) = (1/√(2πσ²)) · exp(-(x-μ)²/(2σ²))
```

**直观理解**：
- `μ`：峰值位置（中心）
- `σ`：宽度参数（标准差，σ越大越扁平）
- 68% 的概率质量在 `[μ-σ, μ+σ]` 内
- 95% 在 `[μ-2σ, μ+2σ]` 内

#### 3D 高斯推广

```math
𝒩(x; μ, Σ) = (1/√((2π)³|Σ|)) · exp(-½(x-μ)^T Σ⁻¹ (x-μ))
```

**关键变化**：
- `σ²` → `Σ`（标方方差 → 协方差矩阵）
- `(x-μ)²` → `(x-μ)^T Σ⁻¹ (x-μ)`（马氏距离平方）

---

### 🔍 概念细化："为什么 exp(-x²)？"

<details>
<summary>点击展开（最大熵原理）</summary>

**问题**：概率分布有无穷多种，为什么选高斯？

**答案**：**在给定均值和方差的约束下，高斯是熵最大的分布**。

**熵的含义**：不确定性度量。熵越大 = 越"不预设额外信息"。

**最大熵原理**：
```
最大化 H(p) = -∫ p(x) log p(x) dx
满足约束:
  1. ∫ p(x) dx = 1        (归一化)
  2. ∫ x·p(x) dx = μ      (固定均值)
  3. ∫ (x-μ)²·p(x) dx = σ² (固定方差)
```

用拉格朗日乘子法求解，得到唯一解：`p(x) ∝ exp(-ax² + bx)` → 就是高斯形式。

**直觉理解**：高斯是"最少假设"的分布——只知道均值和方差时，用它最稳妥。

</details>

---

### 4.4 高斯的"闭合性质"（Closed-form Property）

这才是 3DGS 选择高斯的**真正原因**。

#### 问题：3D 高斯投影到 2D 是什么？

如果你用立方体：
```
3D 立方体 → 透视投影 → 2D 多边形（形状复杂，依赖视角）
```

如果你用高斯：
```
3D 高斯 𝒩(μ, Σ) → 线性投影 → 2D 高斯 𝒩(μ_2d, Σ_2d)
```

**关键**：**高斯投影后还是高斯**，而且有解析公式：
```math
Σ_2d = J · Σ · J^T
```

这个性质叫"闭合性"（closure under linear transformation）。其他分布没有这个优势。

---

### 4.5 从渲染角度理解高斯

#### Alpha Blending 的数学形式

假设沿一条射线有 N 个高素，第 i 个的不透明度是 `α_i`，颜色是 `c_i`：

```math
C = Σᵢ (α_i · c_i · Πⱼ<ᵢ (1-α_j))
```

**解读**：
- `α_i`：当前高斯的贡献权重
- `Π(1-α_j)`：前面所有高斯都没挡住的概率（透射率）
- 从近到远累加，直到透射率接近 0

#### 为什么用高斯定义 α？

3D 高素在屏幕上的投影是 2D 高斯：
```math
α(x) = exp(-½ d²(x, μ_2d))
```

其中 `d²` 是马氏距离（考虑协方差的加权距离）。

**优势**：
- ✅ 中心 α≈1（不透明），边缘平滑衰减到 0
- ✅ 可微分（exp 函数处处光滑）
- ✅ 解析积分（不需要数值采样）

看到公式别慌，先跑代码看效果：

```python
# 示例：直观感受协方差矩阵的作用
import torch
import matplotlib.pyplot as plt

# 同一个中心，不同 Σ
mu = torch.tensor([0.0, 0.0])

Sigma1 = torch.eye(2) * 0.5       # 各向同性（圆）
Sigma2 = torch.diag(torch.tensor([2.0, 0.2]))  # 拉伸（椭圆）

# 采样可视化
samples1 = torch.distributions.MultivariateNormal(mu, Sigma1).sample((1000,))
samples2 = torch.distributions.MultivariateNormal(mu, Sigma2).sample((1000,))

plt.scatter(samples1[:,0], samples1[:,1], alpha=0.3)  # 圆形分布
plt.scatter(samples2[:,0], samples2[:,1], alpha=0.3)  # 椭圆分布
plt.show()
```

**直观理解 > 公式推导**。先建立直觉，再补充严谨性。

## 七、本章核心记忆点

✅ **数学不是前置知识**——它是解决问题的自然延伸，每个概念都被具体问题"逼"出来的

✅ **线性代数的核心直觉**：
- 矩阵 = 线性变换（旋转、缩放）
- `Σ = R @ Λ @ R.T` = "形状 + 方向"的编码方式
- 维度匹配是调试关键（列数=行数）

✅ **微积分的核心直觉**：
- 梯度 = "loss 上升最快的方向"
- 链式法则 = 复合函数的求导规则（反向传播的基础）
- 大规模优化只用一阶方法（二阶太贵）

✅ **概率论的核心直觉**：
- 高斯是"最少假设"的分布（最大熵原理）
- 闭合性质 = "投影后还是高斯"（3DGS 选择它的真正原因）
- α blending = 透视累积公式（从近到远累加透明度）

✅ **学习策略**：Just-in-Time + 代码优先 + 跳过证明细节
