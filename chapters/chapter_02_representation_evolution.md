## 二、体素时代：朴素的离散化

### 2.1 体素工作原理

**3D网格离散化流程**:

```
3D空间  →  划分为 V×V×V 体素网格
              ↓
每个体素存储 occupancy/color
              ↓
Ray marching: 沿ray采样t
              ↓
Trilinear插值读取体素值
              ↓
累加颜色得到像素
```

**数学公式**:
```
设体素网格函数: V(x,y,z) ∈ [0,1] (occupancy) 或 RGB
ray参数化: p(t) = o + t·d, t∈[t_near, t_far]
渲染:
  C = ∫ T(t) · α(p(t)) · c(p(t)) dt
  其中 T(t) = exp(-∫_{t_near}^t α(s) ds)
  离散化: C ≈ Σ T_i · α_i · c_i · Δt_i
```

**需要细化吗？** volume rendering 的积分公式是基础。想展开推导为什么是这个形式吗？

---

#### 🔬 **第一性原理分析：为什么体素是"第一反应"？**

**Step 1: Axioms（公理）**

1. **3D空间可离散化**：连续空间可以被网格划分
2. **渲染需要查询**：给定3D位置，需要知道颜色和密度
3. **查询必须O(1)**：实时渲染要求常数时间

**Step 2: Forced Problems（矛盾）**

- 离散化必然 → 网格分辨率V的选取
- 内存计算 → V³ 立方增长
- 质量要求 → V不能太小

**Step 3: Solution Path（体素的必然与局限）**

```
方案: 等距网格 + 三线性插值
优点: 
  - 查询O(1) ✅ (直接索引)
  - 结构规整 ✅
  - 实现简单 ✅
缺点:
  - 内存O(V³) ❌ (立方增长)
  - 边界锯齿 ❌ (离散化损失)
```

**Step 4: Compression Mechanisms（压缩尝试）**

体素尝试的优化：
- **稀疏体素**：只存非零 → 哈希表/八叉树
- 代价：查询O(log N) vs O(1)
- 无法根本解决立方增长

**Step 5: Verification（重推检验）**

- 掩住答案，能否从"快速查询+内存有限"推导出体素的瓶颈？
  - 快速查询 → 需要数组索引 → 规则网格
  - 内存有限 → V不能太大 → 质量差
  - 能否其他离散方式？自适应网格 → 查询变慢
  - **结论**：体素在"快速查询"约束下是必然选择，但内存墙无法突破

**需要独立重推吗？** 试着从"实时渲染需要快速查询"这个公理出发，推导为什么体素是必然但不可持续的。

### 2.2 体素的"内存墙"（量化分析）

**内存需求计算表**:

```
+--------+-----------+----------------+-----------+
| 分辨率 | 体素数    | 内存 (float32) | 可行性    |
+--------+-----------+----------------+-----------+
| 256³   | 16.7M     | 64 MB          | ✅ 可行    |
| 512³   | 134M      | 512 MB         | ✅ 可接受  |
| 1024³  | 1.07B     | 4 GB           | ⚠️ 边缘   |
| 2048³  | 8.59B     | 32 GB          | ❌ 不可行 |
+--------+-----------+----------------+-----------+
```

**内存增长规律可视化**:

```
内存 (GB)
   ^
 32 |                        *
    |                   *
  4 |              *
    |         *
0.5 |    *
    | *
  0 +----------------------------> V
     256  512  1024 2048

数学: 内存 ∝ V³
V提高2倍  →  内存增加 2³ = 8倍
V提高4倍  →  内存增加 4³ = 64倍
```

**为什么是立方？** 
- 1D: 长度L → 内存O(L)
- 2D: 面积L² → 内存O(L²)
- 3D: 体积L³ → 内存O(L³)
这是维度增加的数学必然。

**需要细化吗？** 这个立方增长是数学必然，无法绕过。想讨论稀疏体素如何缓解吗？

---

#### 🔬 **第一性原理分析：体素质量天花板**

**Axioms**:
1. 表面是连续的（真实物体）
2. 体素是离散的（网格单元）
3. 渲染需要插值（最近邻/三线性）

**Contradictions**:
- 连续表面 vs 离散网格 → 必然有量化误差
- 插值 → 边界平滑但仍有锯齿

**验证 - 边界锯齿可视化**:

```
理想光滑表面:  ~~~~~~~~~~~~~~~~~~~~~~
                (无限分辨率，连续)

体素表示表面:  `````````````````````
                (离散台阶，V=64)

高分辨率体素:  `````````````````
                (V=512，仍可见锯齿)
```

**结论**：体素质量有**硬上限**，需要高V才能平滑，但V受内存限制。

---

### 2.3 体素的遗产与局限

**✅ 遗产（为什么体素仍是重要一步）**:
1. 证明了"预计算存储 + 快速查询"范式可行
2. 确立了"渲染 = 投影 + 累加"基本框架
3. 为后续离散表示打下基础

**❌ 局限（为什么必须淘汰）**:
1. **内存立方增长** → 不可扩展
2. **表面不连续** → 质量天花板低
3. **ray marching仍慢** → 采样点多（虽然查询快，但采样数N_s大）

**历史定位**: 体素是"第一反应"，但很快被证明是**死胡同**。它的遗产是范式，它的局限是教训。

## 三、点云革命：稀疏性的觉醒

### 3.1 点云的核心洞察

**关键洞察**: **大部分3D空间是空的！**

这是3D表示方法的**范式转变**。

**3D空间占用率估算**（真实场景）:

```
+----------------------+--------+--------+
| 区域类型             | 占比   | 说明   |
+----------------------+--------+--------+
| 空气/空区域          | 99.9%  | 无物体 |
| 物体表面             | 0.1%   | 有物体 |
+----------------------+--------+--------+

推论: 为什么要存储空体素? 只存表面点!
```

**内存对比**（关键数字）:

```
体素: V³ (如 512³ = 134M)
点云: N (表面点数, 通常 10⁵-10⁶)
内存减少: 100-1000 倍 ✅
```

**需要细化吗？** 这个"99.9%空间是空的"洞察是点云的基石。想讨论SfM如何直接输出点云吗？

---

#### 🔬 **第一性原理分析：稀疏性的必然性**

**Axioms**:
1. 真实场景物体只占小部分空间
2. 内存是稀缺资源
3. 我们需要高质量渲染

**Contradictions**:
- 体素存储所有空间（包括空的）→ 内存浪费
- 但如何知道哪里是空的？→ 需要先有表示才能判断

**Solution Path**:
```
观察: SfM (Structure from Motion) 直接输出点云
      → 点云已经是稀疏表面采样

决策: 放弃"完整空间表示"，只存表面点
优点: 内存 O(N) << O(V³)
风险: 点零维度，无法直接渲染
```

**Verification**:
- 掩住答案，能否从"内存有限+场景稀疏"推导出只存表面？
  - 内存有限 → 不能存所有体素
  - 场景稀疏 → 大部分体素是空的
  - 只存非空 → 需要先知道哪里非空 → SfM给了点云
  - **结论**：点云是体素内存墙的**必然逃生方案**

---

### 3.2 点云渲染的"三次尝试"（失败史）

点云知道"哪里有表面"，但不知道"表面有多大"。三次尝试都在解决"如何给点加体积"。

---

#### 尝试1: 点精灵 (Point Sprites)

```
方法: 每个点画一个面向相片的方块 (Billboarding)

  3D点:     *               2D投影:  □
            |                          (面向相机)
            |
            V

问题:
  • 点之间有空隙 → 不连续
  • 方块角度固定 → 不是真3D
  • 放大后明显看到方块边界

结果: ❌ 质量差
```

**诊断**: 方块有面积，但**面积固定且不随视角变化** → 不是真3D体积。

---

#### 尝试2: Splatting (2D高斯)

```
方法: 每个点画一个高斯模糊圆
      G(x) = exp(-|x-μ|²/2σ²)

  3D点:     *               2D投影:  ●●●

问题:
  • 这个"高斯"是2D屏幕空间的 → 不是真3D
  • 只是"贴纸"，没有3D几何约束
  • 不同视角下，同一个3D点应该投影到
    不同大小的2D高斯 → 点云方法做不到

关键问题: 2D高斯的尺度σ如何确定?
  • 如果固定σ: 近大远小 → 透视错误 ✗
  • 如果根据深度调整: 需要知道3D"体积"
    → 点云没有3D体积信息! ✗
```

**诊断**: 2D高斯缺少**3D几何约束**，无法正确表达透视变化。

---

#### 尝试3: 表面重建 (Poisson → Mesh)

```
方法: 点云 → Poisson重建 → Mesh → 三角形渲染

  点云  →  三角网格  →  光栅化

结果:
  • 质量提升 ✅
  • 但过程复杂 ❌ (Poisson重建慢)
  • 丢失细节 (Mesh简化) ❌
  • 无法端到端优化 ❌
```

**诊断**: Mesh 质量好，但**失去了稀疏性和可微性**，是"退步"。

---

### 3.3 点云的"最后一公里"问题

**问题诊断**:

```
点云: 位置 + 颜色
    → 渲染需要: 每个像素有3D元素"负责" → 需要面积
    → 质量需要: 连续投影 → 需要连续概率分布
    → 速度需要: 直接投影 → 需要解析公式

缺口: 点云有位置，但没有"影响范围"（体积）
```

**突破口的思考**（关键洞察）:

```
如果在3D空间给每个点定义一个"影响范围"（椭球），
让它自然投影到2D，会怎样?

3D点 + 3D椭球体积  →  投影  →  2D高斯
        ↑
    这就是3DGS的核心思想
```

**需要细化吗？** 这个"3D体积自然投影到2D"是3DGS的eureka moment。想深入讨论为什么是椭球吗？

---

#### 🔬 **第一性原理分析：为什么需要"体积"？**

**Axioms**:
1. 渲染需要面积覆盖（每个像素必须有3D元素负责）
2. 质量需要连续（不能有锯齿或点状伪影）
3. 点云是稀疏的（内存O(N)）

**Contradictions**:
- 点云是零维的（没有面积）❌
- 直接投影到2D导致点间空隙（不连续）❌
- 但点云稀疏性好（内存O(N)）✅

**Solution Path**:
```
给每个点定义一个"影响范围"（3D体积）
这个体积投影到2D应该是连续的（如高斯）
体积形状应能表示各向异性（拉长的椭球）
```

**候选形状对比**:

```
形状      | 投影复杂度 | 边界连续? | 各向异性?
----------|------------|-----------|----------
立方体    | 复杂       | ❌ (六边形) | ✅
球体      | 简单       | ✅         | ❌ (只能各向同性)
椭球      | 中等       | ✅         | ✅
高斯      | 简单       | ✅ (解析)  | ✅ (协方差)
```

**为什么高斯是唯一解？** 下一章详细推导。

---

### 3.4 点云的遗产

**✅ 遗产**:
1. 证明了**稀疏性**是必要且可行的（内存减少100倍）
2. 证明了**直接投影**可以快速（无需ray marching）
3. SfM可直接输出点云 → 数据获取方便

**❌ 遗留问题**（被3DGS解决）:
1. **零维度**：点没有面积 → 无法直接渲染
2. **需要后处理**：Splatting/Mesh都是补救措施
3. **不是真3D**：2D贴纸缺乏几何约束

## 四、NeRF的"结构性瓶颈"

### 4.1 NeRF的核心思想

**公式化表示**:
```
场景表示为神经网络:
  F_θ(x, y, z, direction) → (color, density)

渲染 (ray marching):
  C(r) = ∫_{t_near}^{t_far} T(t) · α(t) · c(t) dt
  其中 T(t) = exp(-∫_{t_near}^t α(s) ds)
```

**训练**:
- 每张图像: 采样ray，计算渲染颜色
- 损失: L2(渲染, GT)
- 反向传播: 更新θ

**需要细化吗？** volume rendering 的积分是NeRF的理论基础。想展开推导吗？

---

### 4.2 NeRF的"三宗罪"（结构性问题）

---

#### 罪1: MLP查询慢（无法避免）

**复杂度分析**（关键计算）:

```
每ray采样数: N_s ≈ 100-200
每样本MLP FLOPs: ≈ 10⁴
每ray FLOPs: N_s × 10⁴ ≈ 10⁶

图像ray数: N_p ≈ 2M (1920×1080)
总FLOPs/帧: N_p × N_s × 10⁴ ≈ 2×10¹² = 2 TeraFLOPs!

理论下限 (RTX 4090, 80 TFLOPS):
  2 / 80 = 0.025 秒 = 25ms

实际 NeRF: 30 秒/帧
差距: 1000 倍（内存访问、非峰值算力、非并行）
```

**为什么MLP不能缓存?**
- 每个ray采样点不同 → 无法重用
- 视角变化 → 采样点变化 → 必须重新计算
- **结构性缺陷**：3D信息编码在权重中，无法直接查询

---

#### 罪2: ray marching本质离散

**采样-质量权衡**（无解）:

```
采样点N_s ↑ → 质量↑ 但 速度↓
N_s=64:  快但噪点多
N_s=128: 平衡
N_s=256: 质量好但慢 2 倍
```

**为什么必须采样？**
- 连续函数f(x)的积分无解析解（f是MLP）
- 必须数值积分 → 离散采样
- **这是数学必然，无法避免**

---

#### 罪3: 训练时间长（数据量爆炸）

**原因**:
- 每样本都要MLP查询（慢）
- 每epoch所有图像都要跑完整管线
- 数据量: M=100-200张图, 每图2M ray → 每epoch 200M ray查询

**时间估算**:
```
每ray 1ms (含MLP) → 每epoch 200s
30k步 ≈ 1500 epochs → 300,000s ≈ 83小时
(实际有优化, 但仍需数小时-数天)
```

---

#### 🔬 **第一性原理分析：NeRF的"三宗罪"根源**

**Axioms**:
1. 场景是连续函数（MLP输入x,d输出color,density）
2. 渲染需要沿射线积分（体积渲染方程）
3. MLP是通用函数逼近器

**Contradictions**:
- MLP每帧必须重新计算（无法缓存3D结构）
- 采样数N_s与质量正相关，但速度负相关
- 训练需要大量射线（每帧2M射线 × N_s样本）

**Solution Path**:
```
问题：MLP查询慢 + ray marching慢
思路：能否避免ray marching？能否避免每样本MLP？

方向1：预计算3D结构 → 缓存（Instant-NGP: hash grid）
方向2：改变表示 → 可直接投影的primitive（3DGS: 高斯）
```

**Verification**:
- 掩住答案，能否从"连续表示+实时渲染"推导出NeRF的瓶颈？
  - 连续表示 → 需要积分 → 需要采样
  - MLP → 每样本计算 → 无法缓存
  - 能否预计算？ → Instant-NGP尝试，但仍有查询开销
  - **结论**：NeRF的瓶颈是"连续函数+实时查询"的**结构性矛盾**

---

### 4.3 NeRF的遗产与局限

**✅ 遗产**:
1. 证明了**神经连续表示**可行
2. 视角依赖效果（镜面反射）建模成功
3. 端到端可微训练范式

**❌ 局限**（必须突破）:
1. **MLP瓶颈**: 无法通过硬件线性加速
2. **离散采样**: ray marching固有缺陷
3. **无缓存**: 每帧必须重新计算

**历史定位**: NeRF是"质量巅峰"，但速度无法接受。它的遗产是证明了连续表示的价值，它的局限是需要被取代的原因。

### 2. 投影公式的推导

**问题**：假设3D高斯Σ是各向同性的（σ²I），投影到2D后Σ₂D是什么形式？画图说明。

**推导**:
```
3D高斯: N(x|μ, Σ)  Σ = σ²I (球体)
投影: 去掉z维度 → 2D高斯 Σ_2D = σ²I_2

结论: 各向同性3D高斯 → 各向同性2D高斯
      σ_2D = σ (相同尺度)

如果Σ是椭球 (对角线σ_x², σ_y², σ_z²):
  Σ_2D = diag(σ_x², σ_y²)  (丢弃σ_z²)

如果Σ有旋转R:
  Σ_2D = R_2D · diag(σ_x², σ_y²) · R_2Dᵀ
  其中R_2D是R的前两行两列
```

**可视化**:

```
3D椭球:       •••••        投影到2D:
         •••     •••      →  ●●●
        •         •       →  ●  ●
        •         •       →  ●  ●
         •••     •••      →  ●●●
          •••••           (2D高斯)
```

### 3. 内存-质量权衡

**问题**：
- 体素内存V³，点云内存N，如果场景复杂度固定（表面点数固定），哪种表示的内存随分辨率增长更快？

**分析**:
```
体素: 内存 ∝ V³ (分辨率提高 → V↑ → 内存立方增长)
点云: 内存 ∝ N (固定，与分辨率无关)

结论: 点云内存恒定，与渲染分辨率无关！
      体素内存随分辨率立方增长！
```

**关键洞察**：点云的内存优势是**绝对的**，不依赖分辨率。

### 4. 为什么其他分布不行？

**问题**：尝试用均匀分布代替高斯，投影公式会怎样？

```
3D均匀分布: 在椭球内均匀，外部0
投影: 需要计算椭球在2D的截面面积
      → 截面形状复杂（椭圆，但边界不规则）
      → 无解析公式 ❌

高斯: 投影仍是高斯 ✅ (这是唯一性)

结论: 高斯是唯一满足"3D→2D投影封闭"的常见分布
```

**数学验证**：
- 正态分布的线性变换仍是正态分布
- 投影是线性变换（去掉维度）→ 仍是正态分布
- 其他分布（均匀、指数）不满足
