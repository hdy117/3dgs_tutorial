## 二、体素：第一反应，但很快撞墙

### 2.1 体素在干嘛？

**思路很简单**：把3D空间切成小立方体，每个存颜色和密度。

```
3D空间  →  V×V×V 体素网格
           ↓
每个体素存值
           ↓
Ray marching: 沿射线采样
           ↓
三线性插值读取
           ↓
累加颜色
```

**数学**：
```
体素函数 V(x,y,z)
射线 r(t) = O + t·d
渲染: C = ∫ T(t)·α(r(t))·c(r(t)) dt
离散化: C ≈ Σ T_i·α_i·c_i·Δt
```

**为什么当时选它？**
- 查询O(1)（直接索引）
- 结构规整，实现简单
- 预计算存储，实时查询 - 听起来很美

---

### 2.2 内存墙：立方增长的数学必然

**数字说话**：

```
V=256  →  16.7M体素  →  64 MB
V=512  →  134M体素   →  512 MB
V=1024 →  1.07B体素  →  4 GB
V=2048 →  8.59B体素  →  32 GB  ❌
```

**为什么是V³？**
- 3D空间，每个维度V个单元
- 总单元数 = V × V × V = V³
- 这是数学，不是实现问题

**增长有多恐怖**：
```
V提高2倍  →  内存增加 8倍
V提高4倍  →  内存增加 64倍
```

消费级GPU最大24GB，V=2048直接超标。**体素分辨率有硬上限**。

---

#### 🔬 **第一性原理：为什么体素是"第一反应"？**

**公理**：
1. 3D空间可离散化
2. 渲染需要快速查询
3. 实时系统需要O(1)

**矛盾**：
- 快速查询 → 需要数组索引 → 规则网格
- 规则网格 → 内存V³
- 内存有限 → V不能大 → 质量差

**结论**：体素在"快速查询"约束下是必然选择，但内存墙无法突破。

---

### 2.3 质量天花板：锯齿无法避免

**问题**：离散网格 + 插值 → 边界必然锯齿。

```
理想表面:  ~~~~~~~~~~~~~~~~~~~~~~
            (连续曲线)

体素表面:  `````````````````````
            (离散台阶)
```

**本质**：连续函数用离散网格近似，必然有量化误差。V越大越平滑，但V受内存限制。

**体素的遗产与局限**：
- ✅ 证明了"预计算 + 快速查询"范式可行
- ✅ 确立了"渲染 = 投影 + 累加"框架
- ❌ 内存立方增长 → 不可扩展
- ❌ 边界锯齿 → 质量天花板低

**历史定位**：第一反应，死胡同。它的遗产是范式，教训是内存墙。

---

## 三、点云：发现99.9%空间是空的

### 3.1 核心洞察：稀疏性

**关键数据**：
```
3D空间占用率:
  空气/空区域: 99.9%
  物体表面:   0.1%
```

**推论**：为什么要存空体素？只存表面点！

**内存对比**：
- 体素: V³ (512³ = 134M)
- 点云: N (10⁵-10⁶)
- **减少100-1000倍** ✅

**这是范式转变**：从"完整空间表示"到"只存可见表面"。

---

#### 🔬 **第一性原理：稀疏性的必然性**

**公理**：
1. 真实场景物体只占小部分空间
2. 内存稀缺
3. 需要高质量渲染

**矛盾**：
- 体素存所有空间（包括空的）→ 浪费
- 但怎么知道哪里是空的？→ 需要先有表示

**解决路径**：
- SfM直接输出点云 → 已经是稀疏表面采样
- 决策：只存表面点
- 优点：内存O(N) << O(V³)
- 风险：点零维度，无法直接渲染

**验证**：从"内存有限+场景稀疏"能推导出只存表面吗？
- 内存有限 → 不能存所有体素
- 场景稀疏 → 大部分体素空
- 只存非空 → 需要先知道哪里非空 → SfM给了点云
- **结论**：点云是体素内存墙的必然逃生方案

---

### 3.2 点云的"三次尝试"：怎么给点加体积？

点云知道"哪里有表面"，但不知道"表面有多大"。三次尝试都在解决这个问题。

---

#### 尝试1: 点精灵 (Point Sprites)

```
方法: 每个点画一个面向相机的方块

问题:
  • 点间有空隙 → 不连续
  • 方块角度固定 → 不是真3D
  • 放大见方块边界

结果: ❌ 质量差
```

**诊断**：方块有面积，但**面积固定不随视角变** → 不是真3D体积。

---

#### 尝试2: Splatting (2D高斯)

```
方法: 画高斯圆 G(x) = exp(-|x-μ|²/2σ²)

问题:
  • 2D屏幕空间 → 不是真3D
  • 只是贴纸，无3D约束
  • 同一点不同视角应投影到不同大小 → 做不到

关键: σ怎么定？
  固定σ → 近大远小 → 透视错误 ✗
  根据深度调 → 需要3D体积 → 点云没有! ✗
```

**诊断**：2D高斯缺少**3D几何约束**，无法正确表达透视。

---

#### 尝试3: Poisson重建 → Mesh

```
方法: 点云 → Poisson → Mesh → 三角形渲染

结果:
  ✅ 质量提升
  ❌ 过程复杂 (Poisson慢)
  ❌ 丢失细节 (简化)
  ❌ 无法端到端优化
```

**诊断**：Mesh质量好，但**失去稀疏性和可微性**，是退步。

---

### 3.3 点云的"最后一公里"问题

**缺口**：

```
点云: 位置 + 颜色
   需要: 每个像素有3D元素负责 → 需要面积
   需要: 连续投影 → 需要连续分布
   需要: 直接投影 → 需要解析公式
   没有: 影响范围（体积）
```

**突破点思考**（eureka moment）：

```
如果给每个点定义一个"影响范围"（椭球），
让它自然投影到2D，会怎样?

3D点 + 3D椭球  →  投影  →  2D高斯
        ↑
    这就是3DGS的核心
```

**为什么是椭球？候选形状对比**：

```
形状    | 投影复杂度 | 连续? | 各向异性?
--------|------------|-------|----------
立方体  | 复杂       | ❌    | ✅
球体    | 简单       | ✅    | ❌ (只能各向同性)
椭球    | 中等       | ✅    | ✅
高斯    | 简单       | ✅    | ✅ (协方差)
```

**高斯是唯一解？** 下一章详细推导。

---

**点云的遗产**：
- ✅ 证明稀疏性可行（内存减100倍）
- ✅ 证明直接投影可以快（无需ray marching）
- ❌ 遗留：零维度 → 需要给点加体积

---

## 四、NeRF：连续表示的代价

### 4.1 NeRF在做什么？

**核心想法**：用神经网络表示连续场景。

```
F_θ(x, y, z, direction) → (color, density)
渲染: C(r) = ∫ T(t)·σ(t)·c(t) dt
训练: L2(渲染, GT) → 反向传播更新θ
```

**优点**：
- 连续表示，质量好
- 视角依赖效果（镜面反射）建模成功
- 端到端可微

**但速度...**

---

### 4.2 NeRF的"三宗罪"

#### 罪1: MLP查询慢（无法避免）

**复杂度分析**：

```
每ray采样数: N_s ≈ 100-200
每样本MLP: ≈ 10,000 FLOPs
每ray: N_s × 10⁴ ≈ 10⁶ FLOPs

图像ray数: 1920×1080 ≈ 2M
总FLOPs/帧: 2M × 100 × 10⁴ = 2×10¹² = 2 TeraFLOPs!

理论下限 (RTX 4090, 80 TFLOPS): 25ms
实际NeRF: 30秒/帧
差距: 约1200倍
```

**为什么MLP不能缓存？**
- 每个ray采样点不同 → 无法重用
- 视角变化 → 采样点变 → 必须重算
- **结构性缺陷**：3D信息编码在权重中，无法直接查询

---

#### 罪2: ray marching本质离散

**采样-质量权衡**（无解）：

```
N_s ↑ → 质量↑ 速度↓
N_s=64:  快但噪点多
N_s=128: 平衡
N_s=256: 质量好但慢2倍
```

**为什么必须采样？**
- 连续函数f(x)的积分无解析解（f是MLP）
- 必须数值积分 → 离散采样
- **数学必然，无法避免**

---

#### 罪3: 训练时间长

```
每ray 1ms → 每epoch 200秒
30k步 ≈ 1500 epochs → 300,000秒 ≈ 83小时
(实际有优化, 但仍需数小时-天)
```

**原因**：每帧都要跑完整管线，每样本都要MLP查询。

---

#### 🔬 **第一性原理：NeRF瓶颈的根源**

**公理**：
1. 场景是连续函数（MLP输入x,d输出color,density）
2. 渲染需要沿射线积分
3. MLP是通用函数逼近器

**矛盾**：
- MLP每帧必须重算 → 无法缓存3D结构
- 采样数N_s与质量正相关，速度负相关
- 训练需要大量射线

**解决路径**：
```
问题: MLP慢 + ray marching慢
思路: 能否避免ray marching？能否避免每样本MLP？

方向1: 预计算3D结构 → 缓存 (Instant-NGP: hash grid)
方向2: 改变表示 → 可直接投影的primitive (3DGS: 高斯)
```

**验证**：从"连续表示+实时渲染"推导：
- 连续表示 → 需要积分 → 需要采样
- MLP → 每样本计算 → 无法缓存
- 能否预计算？ → Instant-NGP尝试，仍有查询开销
- **结论**：NeRF的瓶颈是"连续函数+实时查询"的**结构性矛盾**

---

**NeRF的遗产**：
- ✅ 证明神经连续表示可行
- ✅ 视角依赖效果建模成功
- ✅ 端到端可微范式
- ❌ 局限：MLP瓶颈、离散采样、无缓存 → 必须突破

---

## 五、演进逻辑：约束如何传递？

### 5.1 设计约束的传递链

```
需求: 实时高质量渲染
  ↓
约束1: 快速投影 → 需要O(1)查询
  ↓
排除: NeRF(MLP慢)
  ↓
约束2: 连续质量 → 需要连续PDF
  ↓
排除: 体素(离散)、点云(δ函数)
  ↓
约束3: 稀疏内存 → N<<空间体积
  ↓
保留: 点云思想(稀疏)+高斯(连续+投影解析)
  ↓
方案: 3D高斯椭球 + 直接投影 + Alpha Blending
```

**每一代都在传递约束**：体素的内存墙 → 点云的稀疏性；点云的无体积 → NeRF的连续性；NeRF的速度问题 → 3DGS的直接投影。

---

### 5.2 为什么是"高斯"？形状选择决策树

```
需要3D primitive 有体积+可投影
        |
        +---> 立方体?  → 投影复杂，边界不连续 → ✗
        |
        +---> 球体?    → 只能各向同性，无法拉长 → ✗
        |
        +---> 椭球?    → 任意形状，但投影公式? → 需推导
        |
        +---> 高斯?    → ✅ 任意形状，投影解析，数学优良 → ✓
```

**高斯的三大优势**：
1. ✅ **投影封闭性**：3D高斯 → 2D高斯有闭式解（下一章推导）
2. ✅ **形状完整**：协方差矩阵描述任意椭球（尺度+旋转）
3. ✅ **数学优良**：可微、旋转不变、中心极限定理

**需要独立重推吗？** 试着从"体积+投影解析+可微"三个需求，推导为什么只有高斯满足。

---

### 5.3 关键概念总结

| 概念 | 为什么重要？ |
|------|--------------|
| **稀疏性** | 内存效率100-1000倍的关键 |
| **直接投影** | 速度O(1) per primitive，避免ray marching |
| **连续表示** | 渲染质量无锯齿 |
| **可微性** | 端到端优化 |
| **协方差** | 描述椭球形状（尺度+旋转） |

### 2. 投影公式推导

**问题**：假设3D高斯Σ是各向同性的（σ²I），投影到2D后Σ₂D是什么形式？

**推导**：
```
3D高斯 Σ = σ²I (球体)
投影: 去掉z维度 → 2D高斯 Σ_2D = σ²I_2

结论: 各向同性3D高斯 → 各向同性2D高斯
      σ_2D = σ
```

**如果Σ是椭球 (对角线σ_x², σ_y², σ_z²)**：
```
Σ_2D = diag(σ_x², σ_y²)  (丢弃σ_z²)
```

**关键**：投影是线性变换（去掉维度），正态分布的线性变换仍是正态分布。

---

### 3. 内存-质量权衡

**问题**：体素内存V³，点云内存N，如果场景复杂度固定（表面点数固定），哪种表示的内存随分辨率增长更快？

**分析**：
```
体素: 内存 ∝ V³ (分辨率↑ → V↑ → 立方增长)
点云: 内存 ∝ N (固定，与分辨率无关)

结论: 点云内存恒定！体素内存立方增长！
```

**关键洞察**：点云的内存优势是**绝对的**，不依赖分辨率。

---

### 4. 为什么其他分布不行？

**问题**：尝试用均匀分布代替高斯，投影公式会怎样？

```
3D均匀分布: 椭球内均匀，外部0
投影: 需要计算椭球在2D的截面面积
      → 截面形状复杂（椭圆，边界不规则）
      → 无解析公式 ❌

高斯: 投影仍是高斯 ✅ (唯一性)

结论: 高斯是唯一满足"3D→2D投影封闭"的常见分布
```

**数学验证**：
- 正态分布的线性变换仍是正态分布
- 投影是线性变换（去掉维度）→ 仍是正态分布
- 其他分布（均匀、指数）不满足
