# 第4章：可微分渲染管线

**学习路径**：`invention → verification`

---

## 引言：从数学公式到实际渲染

第3章我们推导了3D高斯投影到2D的公式。本章将其转化为实际可计算的渲染管线：

```
输入：{ (μ_i, Σ_i, α_i, c_i) }，相机参数 K, R, t
输出：2D图像 C（H×W×3）
过程：
  1. 投影：μ_i → 2D坐标，Σ_i → 2D协方差
  2. 计算2D高斯值（每个像素）
  3. 按深度排序
  4. Alpha blending 累加颜色
```

---

## Step 1：3D → 2D投影（透视相机）

### 1.1 3D中心投影

**标准小孔模型**：
```
x_2d = K * (R * μ + t)
其中：
  μ ∈ ℝ³ 世界坐标
  R, t 相机外参
  K ∈ ℝ³ˣ³ 内参矩阵（fx, fy, cx, cy）
```

**深度z**：
- 在相机坐标系：z = (Rμ + t)_z
- 用于排序

---

### 1.2 3D协方差投影到2D

**挑战**：透视投影是非线性的（x = X/Z）

**推导**（见3DGS论文附录）：

令相机坐标系下的3D点：X = R * μ + t
透视投影：x = K * [X_x/X_z, X_y/X_z]ᵀ

**雅可比矩阵 J**（投影变换的线性近似）：
```
J = ∂x/∂X = [1/Z  0  -X_x/Z²]
            [0  1/Z  -X_y/Z²]  * K
```

**投影协方差**：
```
Σ_2D = J * W * Σ * Wᵀ * Jᵀ
其中 W = R（将世界坐标转到相机坐标系）
```

**简化理解**：
- 透视投影相当于在3D空间先乘以R，再用非线性x=X/Z投影
- 雅可比J近似了非线性在局部的影响
- 对于靠近中心线（小视差）的高斯，近似是准确的

**实际实现**：
- 计算 Σ_cam = R * Σ * Rᵀ（转到相机坐标系）
- 计算 Σ_2D = J * Σ_cam * Jᵀ

---

## Step 2：2D高斯在像素网格上的评估

### 2.1 2D高斯函数

给定：
- 中心 μ_2D = (u, v) 像素坐标
- 协方差 Σ_2D = [[σ_uu, σ_uv], [σ_uv, σ_vv]]

对于任意像素 x = (i, j)，高斯值为：
```
G(x) = exp(-0.5 * (x - μ)^T Σ⁻¹ (x - μ))
```

**计算优化**：
- 直接计算逆矩阵 Σ⁻¹ 是O(9)操作，可接受但略慢
- 实际实现：先计算协方差的行列式 detΣ，然后计算二次型
- 或者：对 Σ 做 Cholesky 分解 L Lᵀ = Σ，然后解 L d = x-μ

**数值稳定性**：
- 添加小量 ε 到对角线防止奇异
- 限制协方差的最小/最大特征值（避免过扁或过圆）

---

### 2.2 渲染窗口（Tile-based）

**问题**：每个高斯理论上影响整个图像，但实际上只影响μ附近3σ范围

**优化**：
- 计算2D高斯的"包围盒"：μ ± 3 * sqrt(最大特征值)
- 只在该包围盒内评估高斯值
- 进一步：按图像分块（tile），每个tile预计算需要的高斯列表

**复杂度**：
- 无优化：O(H*W*N) → 不可接受
- 有窗口：每个高斯只影响 ~100像素 → 总复杂度 O(N * 100)
- 对于 N=1M，H*W=2M → 加速100倍

---

## Step 3：深度排序

### 3.1 为什么需要排序？

Alpha blending 公式：
```
C = Σ (g_i.α * g_i.c * Π_{j<i} (1 - g_j.α))
```

只有从远到近累加，才能正确计算遮挡。

### 3.2 排序依据

- 使用投影后的深度 z = (Rμ + t)_z
- 所有高斯按 z 降序排列（远的先画）

**近似性**：
- 高斯有体积，不是点 → 严格说应该按"质心深度"排序
- 但对于细长高斯，中心深度是合理近似
- 实际效果：轻微渲染瑕疵，但可接受

### 3.3 实现

- 每帧：计算所有高斯的 z，然后排序 O(N log N)
- N=1M 时，排序 ~ O(20M) 操作 → 在GPU上很快
- 优化：如果高斯集合不变（推理时），排序可缓存

## Step 4：Alpha Blending（逐tile累加）

**伪代码**：
```python
# image: H×W×3，初始化为0
# accum_alpha: H×W，初始化为0

for g in sorted_gaussians:
    # 计算g影响的范围 [x_min, x_max] × [y_min, y_max]
    for i in range(x_min, x_max):
        for j in range(y_min, y_max):
            # 计算高斯值
            value = g.α * exp(-0.5 * quad_form)
            # Alpha blending
            image[j,i] += value * g.c * (1 - accum_alpha[j,i])
            accum_alpha[j,i] += value
```

**并行化策略**：
- 每个tile独立 → 多线程并行处理不同tile
- GPU实现：每个block处理一个tile，线程处理像素
- 需要原子操作更新 accum_alpha（或使用Warp-level reduction）

## 可微分性验证

**整个管道的可微节点**：

```
μ, Σ, α, c  ← 梯度 ← 最终图像 C
  ↑              ↑
投影公式        Alpha blending
  ↑              ↑
矩阵乘法        乘加运算
```

**关键点**：
- 投影公式：矩阵乘法 → 可微
- 高斯评估：指数+二次型 → 可微
- Alpha blending：乘加 → 可微

**唯一"问题"**：
- 排序是离散操作 → 梯度不连续
- 实际：在训练中，排序顺序相对稳定（高斯不会突然跳变）
- 或使用软化排序（如使用α值加权中心深度的连续近似）

## 实现要点（CUDA视角）

### 内存布局
- 所有高斯参数存储在GPU全局内存：N × 13 float
- 按SoA（Structure of Arrays）或AoS（Array of Structures）均可，SoA更利于向量化

### Kernel设计
1. **Projection kernel**：计算每个高斯的 μ_2D, Σ_2D, depth → 存储到全局数组
2. **Sort kernel**：按depth排序（Thrust库）
3. **Render kernel**：
   - 每个block处理一个tile（如16×16像素）
   - 加载该tile涉及的所有高斯到共享内存
   - 线程遍历像素，累加可见高斯

### 内存访问模式
- 高斯数据：连续读取（coalesced）
- 图像写入：同一block的线程写入相邻像素 → coalesced
