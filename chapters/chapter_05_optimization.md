# 第 5 章：让高斯学会“正确”的事——优化目标为什么要这样设计

**本章核心问题**：第 4 章已经解释了为什么渲染链可以反向传播。但“梯度能回去”不等于“模型会学对”。现在真正的问题是：

> 什么叫学对？我们到底要优化什么？哪些东西应该写进 loss，哪些东西反而应该变成训练过程本身的规则？

如果前两章解决的是表示和渲染，这一章解决的就是：

> 怎样把“高斯能渲染”这件事，变成“高斯能被稳定地训练成正确场景”这件事。

---

## 一、先把“优化”看成两类动作

很多人一提优化，就下意识只想到：

```text
render
-> compare with ground truth
-> compute loss
-> backward
-> step
```

这当然是主干，但对 3DGS 来说还不够。

因为 3DGS 训练时，真正发生的事情其实有两类：

### 1.1 连续参数更新

也就是你熟悉的那一类：

- 调高斯中心 `mu`
- 调协方差 `Sigma`
- 调不透明度 `alpha`
- 调颜色参数

这些都是连续变量，所以适合交给梯度下降。

### 1.2 结构性编辑

但还有另一类动作，不是“把某个参数改小一点”，而是：

- 这个区域高斯不够，要不要多加几个？
- 这个高斯太大了，要不要拆开？
- 这个高斯几乎没贡献了，要不要删掉？

这类动作本质上不是微调，而是：

> 调整表示容量本身。

所以你如果想真正理解 3DGS 的优化，脑子里一定要分清两层：

```text
第一层：连续优化参数值
第二层：动态优化表示结构
```

这两层一起，才构成完整训练过程。

---

## 二、第一层：先定义“图像更对”到底是什么意思

训练时，给定一个采样到的相机视角 `cam_k`，渲染器会产生一张预测图像 `C_pred`。

而数据集里有对应的真实图像 `C_gt`。

所以最基本的目标当然是：

```text
让 C_pred 尽量接近 C_gt
```

把这件事写成更紧凑的形式，就是：

```text
min over Theta  E_k [ L_img(render(Theta, cam_k), C_gt^k) ]
```

这里 `Theta` 表示所有高斯参数的集合。

问题马上来了：

> 这个 `L_img` 到底该怎么定义？

### 2.1 为什么不能直接无脑用 L2

最先想到的通常是均方误差：

```text
L2 = mean((C_pred - C_gt)^2)
```

它的问题不是“错”，而是它太容易被大误差点主导。

如果某个像素因为遮挡边界、反光、高频纹理或者少量噪声而误差很大，L2 会把那个位置平方放大。

这会带来一个很典型的后果：

- 少数大误差像素拿走了过多梯度预算
- 优化开始过度讨好离群点
- 大部分区域反而学得不够稳定

所以 L2 的问题不在于不可微，而在于：

> 它对“少数很错”的地方反应过强。

### 2.2 为什么 L1 更适合作为基底

L1 写起来更简单：

```text
L1 = mean(|C_pred - C_gt|)
```

它的直觉是：

- 每个像素的误差都按线性方式计入
- 大误差不会像平方那样突然变得压倒性重要
- 梯度更稳，更鲁棒

所以在 3DGS 里，L1 很自然地成了主基底。

你可以把它理解成：

> 先保证颜色整体别跑偏，别让少数异常像素把训练方向拉歪。

### 2.3 但只用 L1 还不够

如果你只盯住逐像素绝对误差，会出现另一个问题：

- 整体亮度可能对了
- 局部均值可能也对了
- 但边缘、纹理、结构可能还是糊的

原因很简单。

人眼判断“像不像”，并不只是看每个像素差多少，更看：

- 边界还在不在
- 局部对比度还在不在
- 结构有没有被抹平

这就是为什么要引入 SSIM。

### 2.4 SSIM 在补什么

SSIM 不是在逐像素问“你差了多少”，而是在局部窗口里问：

- 亮度关系像不像
- 对比度关系像不像
- 结构关系像不像

所以它提供的是一种更接近视觉直觉的信号：

> 不是只看数值误差，还看局部结构有没有守住。

在实现里，常见写法是把它转成损失项：

```text
L_ssim = 1 - SSIM(C_pred, C_gt)
```

### 2.5 所以经典组合是：L1 + 一点 SSIM

3DGS 最经典的选择不是“全靠 SSIM”，而是：

```text
L_img = (1 - lambda_dssim) * L1 + lambda_dssim * (1 - SSIM)
```

一个常见设置是：

```text
lambda_dssim = 0.2
```

也就是大致：

```text
L_img ≈ 0.8 * L1 + 0.2 * (1 - SSIM)
```

这里非常值得停一下。

很多人会下意识以为“既然结构重要，那 SSIM 权重应该更大”。

但 3DGS 这里的判断其实很工程化：

- `L1` 负责提供稳定、直接、鲁棒的像素监督
- `SSIM` 负责补充结构偏好，让边缘和局部对比别被抹掉

也就是说：

> L1 是主粮，SSIM 是结构校正，而不是反过来。

这和前面“工程偏爱可控”那条主线其实完全一致。

## 三、但只定义图像 loss，还远远不够

到这里你可能会觉得：

```text
好，图像项已经有了
那训练不就是 minimize L_img 吗？
```

还不够。

因为 3DGS 的难点不只是“参数值怎么调”，还有：

> 当前这套高斯表示，本身到底够不够表达这个场景？

### 3.1 一个区域高斯太少，loss 自己不会凭空造出新高斯

假设某片树叶区域本来就只有很少几个高斯覆盖。

那即使梯度不断告诉你“这里拟合不好”，连续优化也只能做这几件事：

- 挪一挪已有高斯
- 改一改已有高斯的形状
- 调一调已有高斯的颜色和透明度

它做不到的是：

```text
“这里本来应该有 20 个高斯，但你现在只有 3 个，所以我给你自动长出 17 个”
```

单纯的 loss 不会自己扩容表示。

### 3.2 一个高斯太大，也可能把图像先糊过去

另一个问题是，如果某个高斯特别大，它可能暂时把一大片区域平均抹过去，让某些低频误差下降。

但这不代表表示健康。

因为一个过大的高斯往往意味着：

- 它覆盖了太多不该一起解释的区域
- 它会压掉高频细节
- 它会让后续 densify 和细化变得更难

### 3.3 完全没贡献的高斯也不会自动优雅消失

还有一些高斯会慢慢变成：

- 不透明度极低
- 几乎没有可见贡献
- 占参数、占显存、占排序和混合成本

如果你不主动处理，它们不会因为“没用”就自动从参数集合里蒸发。

所以从这里你应该看到一件很关键的事：

> 图像 loss 只负责回答“当前这套高斯渲染得像不像”，但它不负责回答“这套高斯的结构是不是合理”。

---

## 四、3DGS 的关键决定：不要把所有问题都硬塞进一个大 loss

这正是 3DGS 很漂亮、也很工程的一点。

它没有走下面这条路：

```text
搞一个巨大无比的总损失
= 图像项 + 尺度项 + 稀疏项 + 透明度项 + 数量项 + 形状项 + ...
然后指望一次梯度下降把所有问题都解决
```

它更像是这样分工：

### 4.1 loss 负责连续监督

也就是：

- 渲染结果和真实图像像不像
- 当前参数朝哪个连续方向改更好

### 4.2 训练策略负责结构管理

也就是：

- 什么时候加高斯
- 什么时候拆高斯
- 什么时候删高斯
- 什么时候放慢学习率，进入收敛阶段

这背后的思想其实特别值得记住：

> 不是所有约束都应该被写成一个可微标量项。有些问题，直接做成训练流程里的离散规则，反而更自然、更稳。

这和上一章里“排序虽然不严格可微，但工程上仍然可用”是同一种精神。

### 4.3 一些实现会再加轻量稳定项

在实际实现里，你也会看到一些额外处理，例如：

- 对过大的尺度加轻量惩罚或裁剪
- 对透明度做阈值控制
- 对数值不稳定的协方差做正则

但这些通常都不是主角。

主角仍然是：

```text
图像重建项 + 动态密度控制
```

这条主线一定不能丢。

## 五、那“哪里该加高斯”到底由什么决定？

这里是 3DGS 优化最聪明的一部分之一。

它依赖的不是玄学，而是两个非常朴素的信号。

### 5.1 信号一：梯度持续很大

如果某个高斯相关的梯度长期偏大，通常说明一件事：

```text
当前这附近的表示还没解释好图像误差
```

你可以把它理解成：

- 如果位置梯度大，说明“放这儿不太对”
- 如果形状相关梯度大，说明“这个 footprint 还不对”
- 如果这种情况持续出现，说明“不是随便抖一下就能解决，表示本身可能不够细”

所以大梯度本质上在说：

> 这里还缺表达能力。

### 5.2 信号二：屏幕 footprint 太大

另一个信号来自高斯在屏幕上的投影半径或覆盖范围。

如果某个高斯在图像里占了一大片区域，往往说明：

- 它太粗
- 它想同时解释太多局部结构
- 它更适合被拆成更小的几个

所以你可以把它理解成：

```text
大梯度告诉你：这里没学好
大 footprint 告诉你：这里太粗了
```

两者合起来，就给出了 densification 的核心依据。

## 六、于是就有了三种结构动作：clone、split、prune

### 6.1 Clone：不够用，但已经够小

如果某个高斯：

- 梯度大
- 但自身尺度并不算大

那更可能意味着：

> 它不是太粗，而是人数不够。

这时候一个自然动作就是复制一个相近的高斯，让这片区域有更多局部自由度。

你可以把 clone 理解成：

```text
这个工位的人干不过来
那就再派一个人来一起干
```

### 6.2 Split：又大又学不好

如果某个高斯：

- footprint 很大
- 同时相关误差又持续下不去

那更合理的判断通常是：

> 它不是“再努力一点就能学好”，而是“一个高斯本来就不该解释这么大一块区域”。

这时候就该 split。

也就是把一个大高斯拆成两个更小的高斯，通常沿它最主要的伸展方向分开。

直觉非常简单：

```text
一块太大的模糊刷子
不容易画细节
不如换成两把更小的刷子
```

### 6.3 Prune：几乎没贡献，就别占位了

如果某个高斯长期表现成：

- 透明度极低
- 或者尺度退化到几乎没意义
- 或者整体贡献已经可以忽略

那删掉它通常比继续留着更好。

因为它会白白增加：

- 显存压力
- 排序和 tile 映射负担
- 每像素混合负担

所以 prune 的本质不是“惩罚失败者”，而是：

> 把训练预算和渲染预算留给真正有用的表示单元。

---

## 七、为什么 densify / prune 不能每一步都做

到这里你可能会想：

```text
既然这些规则这么有用
那为什么不每一步都执行？
```

因为太频繁地改结构会带来三个问题。

### 7.1 梯度信号会太噪

单步梯度很容易受：

- 当前采样到的视角
- 当前局部遮挡关系
- 当前 batch 的偶然性

影响。

如果你每一步都根据瞬时梯度做增删，结构会非常抖。

所以实践里通常会：

- 缓存一段时间的梯度统计
- 周期性地做 densify / prune

这样信号更稳。

### 7.2 优化器状态会被频繁打断

一旦高斯数量变了，参数张量就变了。

这会直接影响优化器内部状态，比如 Adam 里的动量缓存。

所以结构编辑不是“免费操作”，而是会打断一部分连续优化节奏。

### 7.3 前期和后期需要的策略根本不同

训练前期往往是：

- 表示不足
- 很多地方还没被覆盖
- 更需要增殖和细化

训练后期更像是：

- 主体结构已经成形
- 更需要收敛和清理冗余
- 过度 densify 反而会让系统发散或臃肿

所以合理调度通常是：

```text
前期：更积极 densify
中期：降低增殖频率，专心细调
后期：停止增长，开始 prune 和收敛
```

这就是为什么 3DGS 的优化，不是一个静态公式，而是一套动态策略。

## 九、本章你真正应该能自己重建的几个问题

读完这一章，你最值得能自己讲清楚的是：

1. 为什么“可微”不等于“会学对”？
2. 为什么图像项不能无脑只用 L2？
3. 为什么 L1 和 SSIM 要组合，而不是只选一个？
4. 为什么经典 3DGS 里，L1 通常是主基底，SSIM 更像结构校正？
5. 为什么只最小化图像 loss，仍然不能解决“高斯数量够不够”这个问题？
6. 为什么 densify / prune 更适合作为训练规则，而不是硬塞进一个统一可微损失里？
7. clone 和 split 的区别到底是什么？
8. 为什么结构编辑不能每一步都做，而要周期性地做？

如果这些问题你能自己推回来，这一章就真的进入你的脑子了。
