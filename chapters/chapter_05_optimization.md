# 第 5 章：让高斯学会"正确"的事——优化目标的完整设计

你已经有了一套可微分的渲染管线 —— 给定一组高斯，能算出图像；给定图像误差，能反向传播梯度。但这里有个根本问题：

**"正确"到底是什么？**

是像素颜色接近？边缘锐利？还是几何准确？或者三者都要？本章我们从头推导：**损失函数该怎么设计，才能让高斯学到真正有用的表示。**

---

## 从直觉出发：我们希望优化什么？

想象你在训练一个 3DGS 模型。训练一开始，渲染出来的图像可能完全是噪点。怎么告诉它"往哪个方向改"？

最直接的思路很简单：**让预测接近真实**。用数学写就是：

```
最小化：距离 (C_pred, C_gt)
```

但问题立刻来了：**"距离"选什么？**

### 第一个选择：像素级误差

你可能想到 L2（均方误差）—— 毕竟这是机器学习里最常见的损失：

```python
L2 = ((C_pred - C_gt) ** 2).mean()
```

**直觉上很合理对吧？** 但让我们用第一性原理拆开看：

- L2 的梯度是 `2·(C_pred - C_gt)`
- 如果某个像素误差特别大（比如传感器噪点），它的梯度会主导整个更新
- **结果**：整张图为了"讨好"几个离群点，妥协了大部分区域的质量

试试这个思想实验：一张 100×100 的图，99% 像素误差是 0.1，但有一个噪点误差是 10。L2 损失里，那个噪点的贡献是 `10² = 100`，而所有正常像素总共才 `9900 × 0.1² = 99`。**一个噪点的权重超过了整张图其他部分的总和。**

这不合理。我们需要更鲁棒的方案。

### 转向 L1：饱和的梯度

L1 损失（平均绝对误差）给出恒定梯度的 ±1：

```python
L1 = torch.abs(C_pred - C_gt).mean()
```

**关键差异**：无论误差是 0.1 还是 10，梯度大小都是 1。离群点无法主导优化。

但 L1 有自己的问题：**它不区分"小误差"和"大误差"的严重性。** 更重要的是，纯 L1 训练的图像往往偏模糊 —— 因为梯度没有结构感知能力，只是盲目地往"减小绝对误差"的方向走。

**到这里你发现没？** 单一损失函数似乎都有缺陷：
- L2 → 对离群点敏感
- L1 → 可能模糊，缺乏结构感知

### SSIM：让梯度"看到"结构

这时候引入一个新视角：**人眼怎么判断图像质量？**

不是看每个像素的误差，而是看**结构是否保持**。边缘锐利吗？纹理清晰吗？物体轮廓完整吗？

SSIM（Structural Similarity Index）正是基于这个直觉设计的。它在局部窗口内比较：
- 亮度相似度（均值）
- 对比度相似度（方差）
- 结构相似度（协方差）

**多尺度 SSIM (MSSIM)** 进一步在多个窗口大小下计算，捕捉不同频率的结构信息。

```python
SSIM = ms_ssim(C_pred, C_gt)  # 范围 [0,1]，越大越好
L_ssim = 1 - SSIM             # 转为损失（越小越好）
```

**SSIM 的梯度有什么特殊？** 它不是单个像素的误差，而是窗口统计量的导数。这意味着：
- 如果某个区域整体偏暗，梯度会让整个区域均匀变亮
- 如果边缘模糊了，梯度会试图恢复对比度
- **结构信息被保留下来**

但 SSIM 也有代价：计算量比 L1/L2 大（虽然 MSSIM 实现优化后其实还好），而且梯度在某些情况下可能不稳定。

### 组合策略：L1 + SSIM

3DGS 的最终选择是聪明的：**两者都用，权重偏向 SSIM。**

```python
λ_ssim = 0.8
L_img = (1 - λ_ssim) * L1 + λ_ssim * (1 - SSIM)
#     = 0.2 × L1   + 0.8 × L_ssim
```

**为什么是这个比例？**
- L1（权重 0.2）提供稳定的"基底梯度"，防止离群点干扰
- SSIM（权重 0.8）主导优化方向，确保结构清晰
- **实验表明这个组合在 PSNR 和视觉质量上取得平衡**

## 但等等：只优化图像质量够吗？

假设你现在只用 L1+SSIM。训练跑起来，PSNR 确实上升了。**但这能保证高斯参数是健康的吗？**

让我们用第一性原理想想：**什么情况下损失会下降，但表示却变差了？**

### 问题 1：尺度退化（scale degeneration）

协方差矩阵 Σ 决定了每个高斯的形状和大小。如果没有任何约束，优化可能走向极端：

- **σ → ∞**：一个巨大高斯覆盖整个图像
  - 损失会下降吗？可能会 —— 它"抹平"了所有细节，平均误差降低
  - 但这显然不是我们想要的表示
  
- **σ → 0**：高斯退化为点
  - 单独一个点没什么危害（影响范围小）
  - 但如果大量高斯都趋向于点，渲染效率会崩溃

### 问题 2：不透明度退化（opacity degeneration）

α 控制每个高斯的可见度。极端情况包括：

- **α → 1** 且 σ 很大：一个完全不透明的巨大斑块
  - 它会遮挡后面所有高斯
  - 破坏 alpha blending 的层次感
  
- **α → 0**：完全透明的高斯
  - 它对渲染没贡献，纯属浪费参数
  - 但这种情况可以直接删除（后面会讲）

### 问题 3：密度不均

初始点云（比如从 COLMAP 来）可能在某些区域过密、某些区域稀疏。**损失函数本身不会自动解决这个问题** —— L1+SSIM 只关心最终图像，不在乎用了多少高斯、分布是否合理。

---

## 正则化设计：只惩罚"真正有害"的极端值

现在的问题是：**怎么加约束，既防止退化，又不束缚正常优化？**

### 尺度正则：只罚大，不罚小

3DGS 的选择很有意思：

```python
σ_max = 1.0  # 最大允许尺度（世界坐标单位）
L_scale = torch.clamp(scales - σ_max, min=0).mean()
```

**关键设计决策**：只惩罚超过 σ_max 的部分，对"小尺度"完全不约束。

**为什么不对称？**

- **大尺度的危害是全局的**：一个巨大高斯会污染整张图，必须阻止
- **小尺度的危害是局部的**：点状高斯最多影响几个像素，而且可以通过 densify 自动增加数量
- **更重要的是**：场景中有天然的大尺度结构（比如远处的墙），惩罚太重会把它们强行压缩

**用 clamp 而不是平方？** 
```python
# clamp: L = max(0, σ - σ_max)      → 线性惩罚
# square: L = (σ - σ_max)²          → 二次惩罚
```
线性更温和，允许"适度大尺度"（比如表面点沿法线方向拉长）。

---

### 不透明度正则：只罚接近 1

另一个不对称设计：

```python
L_opacity = torch.clamp(alpha - 0.99, min=0).mean()
```

**阈值为什么是 0.99？**

想想 alpha blending：透射率 `T = Π(1-α_i)`。如果某个高斯 α≥0.99，它后面的所有高斯都会被几乎完全遮挡。**0.99 这个阈值是为了防止"不透明斑块"破坏混合层次。**

**为什么不惩罚小 α？**
因为 α<0.001 的高斯可以直接删除（prune），不需要损失函数慢慢推。

## 总损失：所有部分组装

现在把所有组件拼起来：

```python
L_total = L_img + λ_scale × L_scale + λ_opacity × L_opacity

其中:
  L_img       = 0.2 × L1        + 0.8 × (1 - SSIM)
  L_scale     = mean(clamp(scales - 1.0, min=0))
  L_opacity   = mean(clamp(alpha - 0.99, min=0))
  
λ_ssim  = 0.8    # SSIM 权重（在 L_img 内部）
λ_scale = 0.01   # 尺度正则权重
λ_opac  = 0.01   # 不透明度正则权重
```

**权重的直觉层次：**
- 重建损失：权重 1.0（主要目标）
- 正则化：权重 0.01（辅助约束，防止退化但不过度束缚）

## 但还有一个问题：高斯数量怎么定？

训练开始前，你从 COLMAP 点云得到初始高斯 —— 可能几千、几万。但这个数量合理吗？

- **简单区域**（天空、墙壁）也许几百个就够了
- **复杂区域**（树叶、毛发）可能需要成千上万
- **COLMAP 的分布本身就不均匀**，有些区域根本没特征点

**损失函数只能优化"现有高斯的参数"。如果某个区域高斯数量不够，L1+SSIM 再低也救不了。**

我们需要一个机制：**在训练过程中动态调整高斯的数量和分布。**

## 密度控制：从梯度信号中推断"哪里需要更多表示"

这是 3DGS 最聪明的设计之一。让我们用第一性原理推导**为什么梯度能告诉我们"这里 densify"**。

### Axiom 1：梯度大 = "不理解"

反向传播后，每个高斯的参数都有梯度 `∂L/∂θ`。**梯度的物理意义是什么？**

- 如果某高斯的 μ（位置）梯度很大 → 说明它的位置对误差贡献很大
- 如果它的 Σ（协方差）梯度很大 → 形状需要大幅调整才能降低损失
- **综合**：梯度大意味着"这个高斯没学好，或者不够用"

### Axiom 2：投影尺度大 = "3D 尺度过大"

渲染管线会输出每个高斯的 `radii`（在屏幕上的影响半径，单位像素）。如果 `radii > 阈值`：
- 说明这个高斯在 3D 空间里太大
- **一个巨大高斯很难同时拟合精细结构** → 应该分裂成两个小的

### Densify 的触发条件

结合这两个信号：

```python
if radii[i] > scale_threshold AND (grad_mu[i] > grad_threshold OR grad_Sigma[i] > grad_threshold):
    densify_gaussian(i)
```

**注意逻辑是"且"**：
- 尺度大 **但** 梯度小 → 这个高斯已经稳定，只是太大 → 应该分裂（不依赖梯度条件）
- 梯度大 **但** 尺度小 → 已经很精细了但误差还大 → 应该克隆一份增强

实际实现中会根据具体触发情况选择不同操作。

## Densify 操作：克隆 vs 分裂

### 克隆（Clone）

场景：梯度大但尺度小（已经很精细的高斯，但误差还很大）

```python
def clone(i):
    return Gaussian(
        mu = mu[i].clone(),
        Sigma = Sigma[i].clone(),
        alpha = alpha[i].clone() * 0.5,  # 透明度减半，避免叠加后过曝
        color = SH2RGB(sh[i])           # 同样颜色
    )
```

**为什么 α 要减半？** 两个完全重合的高斯如果 α 都是原值，叠加后会过饱和。减半让它们"分享"原来的不透明预算。

### 分裂（Split）

场景：尺度大（在屏幕上覆盖很大区域）

```python
def split(i):
    # 1. 特征值分解：Σ = VΛVᵀ
    eigvals, eigvecs = torch.linalg.eigh(Sigma[i])
    
    # 2. 主方向 = 最大特征值的特征向量（高斯"最长"的方向）
    main_dir = eigvecs[:, -1]
    
    # 3. 新尺度：沿主方向分裂，总"体积"不变
    new_scales = sqrt(eigvals) / sqrt(2)
    new_Sigma = eigvecs @ diag(new_scales²) @ eigvecsᵀ
    
    # 4. 新位置：沿主方向稍微分开
    offset = 0.01 × main_dir
    mu1 = mu[i] + offset
    mu2 = mu[i] - offset
    
    return Gaussian(mu1, new_Sigma, ...), Gaussian(mu2, new_Sigma, ...)
```

**为什么是 `sqrt(2)`？** 
- 高斯"体积" ≈ det(Σ)^(1/2) = σ_x × σ_y × σ_z
- 分裂成两个后，每个的尺度除以 √2 → 体积变为原来的 (1/√2)³ ≈ 0.35
- **两个加起来** ≈ 0.7 原体积（稍微收缩一点是合理的）

**为什么沿主方向分裂？**
这是最优选择：高斯椭球在最长方向上"最不确定"，沿这个方向分裂能最大程度减小每个子高斯的尺度。

## Prune：删除冗余的高斯

Densify 负责增加，Prune 负责清理。**什么情况下该删？**

### 条件 1：透明度太低

```python
if alpha[i] < 0.001:
    delete(i)  # 几乎完全透明，无渲染贡献
```

### 条件 2：尺度太小

```python
scales = sqrt(eigvalsh(Sigma))
if scales.max() < 1e-6:
    delete(i)  # 数值不稳定，退化为点
```

**为什么 prune 只在后期（15k 步后）开始？**

前期高斯数量普遍不足，需要 densify 快速扩张。这时候删掉"低 α"的高斯可能误伤——它们也许还没学好，但有潜力。**后期数量趋于饱和了，再开始清理冗余。**

---

## 调度策略：三阶段训练动态

3DGS 的训练不是匀速的，而是分阶段的：

### 阶段 1：快速生长（0 - 7.5k 步）
- **densify**：每 500 步一次
- **prune**：关闭
- **目标**：高斯数量从几千膨胀到几十万，覆盖场景所有细节区域

### 阶段 2：精细调整（7.5k - 15k 步）
- **densify**：每 1000 步一次（频率减半）
- **prune**：仍关闭
- **学习率**：在 7.5k 和 15k 各衰减 10 倍
- **目标**：稳定优化，避免过度 densify

### 阶段 3：收敛期（15k - 30k 步）
- **densify**：关闭
- **prune**：每 1000 步一次
- **学习率**：继续衰减
- **目标**：清理冗余高斯，最终确定表示

---

## 🤔 这个调度背后的直觉

为什么是"先扩张后收缩"？类比神经网络训练：
- **前期**：欠拟合 → 增加容量（densify）
- **中期**：接近最优 → 精细调整（降低学习率）
- **后期**：过拟合风险 → 正则化清理（prune）

**你能想到其他调度策略吗？** 比如根据验证集 PSNR 动态决定是否 densify？或者用早期停止（early stopping）避免过度 prune？

## 学习率分组：为什么不同参数需要不同 LR？

Adam optimizer 的配置很关键：

```python
optimizer = Adam([
    {'params': mu,     'lr': 1.6e-4},   # 位置
    {'params': Sigma,  'lr': 1e-3},     # 协方差
    {'params': alpha,  'lr': 5e-2},     # 不透明度（最大！）
    {'params': sh,     'lr': 5e-3}      # 球谐系数（颜色）
])
```

**为什么 α的学习率比 μ 大100倍？**

- **α的变化范围**：[0, 1] —— 很小的区间
- **μ的变化范围**：可能是 [-100, 100] 甚至更大（世界坐标）
- **梯度量级**：即使损失对 α和 μ的敏感度相同，由于参数尺度差异，实际更新步长也会不同

如果所有参数用同一个 LR，要么 α更新太慢（学不透明度的过程拖沓），要么 μ更新太快（位置震荡）。

## 完整训练循环的关键细节

```python
for step in range(30000):
    # 1. 前向渲染
    rendered, radii = render(gaussians, camera)
    
    # 2. 计算损失
    L_total = compute_loss(rendered, gt_image, gaussians)
    
    # 3. 反向传播
    optimizer.zero_grad()
    L_total.backward()
    optimizer.step()
    
    # 4. 缓存梯度（用于 densify 决策）
    if step % 100 == 0:
        grad_cache = gaussians.mu.grad.detach().norm(dim=1)
    
    # 5. 密度控制（每 1000 步）
    if step % 1000 == 0 and step > 500:
        densify_and_prune(gaussians, radii, grad_cache)
        
        # ⚠️ 关键：高斯数量变了，要重建 optimizer！
        optimizer = rebuild_optimizer(gaussians)
    
    # 6. 学习率调度（7.5k, 15k 步衰减）
    if step in [7500, 15000]:
        for param_group in optimizer.param_groups:
            param_group['lr'] *= 0.1
```

**重建 optimizer 是很多人踩过的坑**。densify/prune 会改变张量长度，旧的 optimizer 状态（momentum buffer）就失效了。

## 本章核心记忆点

✅ **重建损失**：`L_img = 0.2×L1 + 0.8×(1-SSIM)` — L1 鲁棒 + SSIM 结构感知

✅ **正则化设计哲学**：只惩罚"真正有害"的极端值
- 尺度：只罚大（>1.0），不罚小
- 不透明度：只罚高（>0.99），低 α直接 prune

✅ **Densify 触发**：投影尺度大 + 梯度大 → 分裂或克隆

✅ **Prune 触发**：α<0.001 或 尺度<1e-6 → 删除

✅ **三阶段调度**：快速生长（0-7.5k）→ 精细调整（7.5k-15k）→ 收敛清理（15k-30k）

✅ **学习率分组**：α需要最大 LR（5e-2），μ需要最小（1.6e-4）——参数尺度差异决定

## 思考题（第一性原理式）

**1. 如果 SSIM 权重从 0.8 降到 0.5，图像质量会怎么变？**

**2. 尺度正则阈值 σ_max=1.0 是经验值。如果场景尺度很大（比如城市级重建），这个阈值会误伤吗？**
提示：3DGS 的实际实现中，σ_max 通常是动态的 —— 根据初始点云的尺度分布自适应调整。你能推导出合理的自适应公式吗？

**3. Densify 基于梯度信号，但梯度本身是"噪声"（每步都在变）。为什么要缓存 N 步的平均梯度？**
提示：单步梯度可能受 batch/图像影响很大。平均能稳定决策。

**4. 分裂时为什么沿主方向？如果高斯接近球形（各向同性），特征值都差不多，这时候怎么 split？**
提示：实际实现中会检查特征值的条件数（最大/最小比值）。如果太接近，可能选择克隆而不是分裂。
