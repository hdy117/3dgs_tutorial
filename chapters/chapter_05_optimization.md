# 第5章：优化目标与损失函数

**学习路径**：`verification`

---

## 引言：如何训练3DGS？

**问题**：我们有可微分渲染管线，但优化什么？损失函数如何设计？

**核心思路**：
- **重建损失**：让渲染图像 ≈ 真实图像
- **正则化**：防止高斯退化（无限扁、无限不透明）
- **密度控制**：动态调整高斯数量（densify/prune）

---

## 1. 重建损失（Image Reconstruction Loss）

### 1.1 L1 Loss

**定义**：
```
L_L1 = |C_pred - C_gt|_1 = Σ_{pixels} |C_i - C_i^gt|
```

**性质**：
- ✅ 对离群点鲁棒
- ✅ 保证颜色不漂移
- ❌ 可能模糊（L1倾向于中等解）

**梯度**：
```
∂L_L1/∂C_pred = sign(C_pred - C_gt)
```

---

### 1.2 SSIM Loss（结构相似性）

**动机**：L1是像素级损失，不考虑结构。人类视觉对结构/纹理更敏感。

**SSIM公式**（简化，单通道）：
```
SSIM(x, y) = (2μ_x μ_y + C1)(2σ_xy + C2) / (μ_x² + μ_y² + C1)(σ_x² + σ_y² + C2)
```
其中 μ, σ 是局部窗口的均值和方差，σ_xy 是协方差。

**3DGS用的变体**：论文用 **MSSIM**（多尺度SSIM），在多个分辨率下计算。

**损失**：
```
L_SSIM = 1 - MSSIM(C_pred, C_gt)
```

**为什么加SSIM？**
- 提升视觉质量（纹理更清晰）
- 减少模糊伪影

---

### 1.3 总重建损失

3DGS论文：
```
L_img = (1 - λ) * L_L1 + λ * L_SSIM
通常 λ = 0.8
```

**解读**：
- SSIM权重更高（0.8），因为对质量影响大
- L1保证基础颜色准确

---

## 2. 正则化项

### 2.1 尺度正则（Scale Regularization）

**问题**：高斯尺度 Σ 可以任意大或任意小

- 尺度无限大 → 高斯覆盖整个图像（模糊）
- 尺度无限小 → 退化为点（噪声）

**正则化**：
```
L_scale = Σ_i max(0, σ_i - 1.0)  # 惩罚过大尺度
```
或对过小尺度也可加惩罚（但论文主要惩罚过大）

**为什么只惩罚大尺度？**
- 小尺度 → 点状，但不会模糊
- 大尺度 → 会"污染"大范围像素，必须限制

---

### 2.2 不透明度正则（Opacity Regularization）

**问题**：α 可以接近 1（完全不透明）或接近 0（完全透明）

- α=1 且尺度大 → 不透明大斑块（遮挡问题）
- α=0 → 无贡献，浪费参数

**正则化**（论文未明确，但实践中常用）：
- 鼓励 α 接近 0.5-1 之间的合理值
- 或使用稀疏性诱导（如L1 on α）

---

## 3. 密度控制（Densification & Pruning）

**这是3DGS的核心创新之一**：动态调整高斯数量

### 3.1 为什么需要？

- 初始高斯来自SfM点云 → 通常稀疏且分布不均匀
- 训练中：某些区域高斯过少（重建空洞），某些区域过多（冗余）
- 需要自动调整：不够的地方加，太多的地方删

### 3.2 Densify（分裂）策略

**触发条件**：
- 一个高斯在某个视图下被**投影到较大屏幕空间区域**（> 阈值，如3像素）
  - 说明该高斯在3D空间尺度太大，需要分裂成多个小高斯
- 且该高斯**梯度较大**（|∂L/∂μ| 或 |∂L/Σ| 大）
  - 说明该位置重建误差大，需要更多表示能力

**操作**：
- 克隆该高斯（复制一份）
- 或者：在3D空间沿主方向分裂成两个（尺度减半，位置微调）

---

### 3.3 Prune（删除）策略

**触发条件**：
- 高斯不透明度 α 太小（如 < 0.001）
- 且尺度太小（几乎不可见）

**操作**：
- 直接删除该高斯

**为什么能删？**
- α=0 对渲染无贡献
- 尺度小且α小 → 该高斯"死了"，留着浪费内存

---

### 3.4 密度控制频率

- 每N次迭代（如每1000次）执行一次densify/prune
- 不能每帧都做（太慢）

---

## 4. 总体损失函数

**完整训练损失**：
```
L_total = L_img + λ_scale * L_scale + λ_opacity * L_opacity
```

其中 λ_scale, λ_opacity 是超参数（通常 ~0.01）

**训练循环**：
```python
for iteration in range(total_steps):
    # 1. 渲染
    image = render(gaussians, camera)  # 可微分

    # 2. 计算损失
    loss = L1(image, gt_image) + λ * (1 - SSIM(image, gt_image))
    loss += λ_scale * scale_reg(gaussians)
    loss += λ_opacity * opacity_reg(gaussians)

    # 3. 反向传播
    loss.backward()

    # 4. 梯度更新（Adam）
    optimizer.step()

    # 5. 密度控制（每N步）
    if iteration % control_interval == 0:
        densify_and_prune(gaussians, optimizer)
```

---

## 5. 关键超参数总结

| 参数 | 典型值 | 作用 |
|------|--------|------|
| λ (SSIM权重) | 0.8 | 平衡L1和SSIM |
| λ_scale | 0.01 | 尺度正则强度 |
| λ_opacity | 0.01 | 不透明度正则 |
| densify_interval | 100-1000步 | 密度控制频率 |
| scale_threshold | 3像素 | 多大尺度触发分裂 |
| grad_threshold | 高梯度阈值 | 梯度多大触发分裂 |
| prune_alpha_threshold | 0.001 | 低于此α删除 |
| learning_rate | 1e-3 ~ 1e-4 | 参数更新步长 |
