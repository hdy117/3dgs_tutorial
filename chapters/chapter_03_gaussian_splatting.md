# 第3章：核心发明 - 高斯椭球体作为3D表示

**学习路径**：`invention`（完整的第一性推导）

**核心目标**：从第一性原理推导3DGS为什么必须用高斯椭球，理解其数学本质

---

## 一、引言：我们要解决什么？

### 1.1 设计目标

我们要设计一个3D primitive（基元），满足：
1. **快速查询**：给定相机位姿，能快速投影到2D
2. **连续覆盖**：投影后能连续覆盖像素，无空隙
3. **可微分**：参数可优化，梯度可反向传播
4. **内存高效**：稀疏表示，N ~ 10⁵-10⁶

### 1.2 上一章的线索

- 点云快但零维 → 需要加"体积"
- 如果在3D中给每个点一个"影响范围"（有体积），投影后自然就是有面积的2D高斯

**核心问题**：
> 3D空间中的"面积"是什么？如何定义它的投影？

---

## 二、Axioms：不可约的基础事实

### 2.1 Axiom 1：3D空间中的位置

每个 primitive 有一个中心位置 **μ** ∈ ℝ³

**为什么需要位置？**
- 这是3D表示的最基本信息：点在哪里
- 没有位置，无法知道"哪里"有东西

---

### 2.2 Axiom 2：3D空间中的"范围"需要二阶统计量

**一阶 vs 二阶**：
- 一阶（位置）：只 tells us 中心在哪
- 二阶（协方差）： tells us 数据在哪些方向扩散

**为什么协方差是必要的？**
- 一个点没有范围 → 零维
- 要定义"范围"，需要知道数据在x、y、z方向的扩散程度
- 协方差矩阵 Σ ∈ ℝ³ˣ³ 正好捕捉：
  - 各轴尺度（variance）
  - 轴之间的相关性（rotation）

**协方差矩阵的性质**：
- 对称正定：Σ = Σᵀ, Σ ≻ 0
- 特征分解：Σ = R Λ Rᵀ
  - R：旋转矩阵（特征向量）
  - Λ：对角矩阵，对角元素 = 各轴方差 σ₁², σ₂², σ₃²
- 等密度面是椭球，半轴 = √σᵢ

---

### 2.3 Axiom 3：高斯分布是连续概率密度函数的自然选择

**为什么选高斯？**
1. **中心极限定理**：大量独立误差叠加 → 高斯
2. **数学性质优良**：
   - 可解析积分
   - 可微分
   - 旋转不变性
3. **其他选择都不如高斯**：
   - 均匀球：边界不连续，导数不连续
   - 拉普拉斯：尾巴太重，数学复杂
   - 指数分布：不对称

**结论**：高斯是"最平滑"的连续分布

---

## 三、Contradictions：从Axioms引出的矛盾

### 3.1 矛盾1：离散 vs 连续

- 体素是离散的 → 边界锯齿
- 我们需要连续表示 → 必须用概率密度函数（PDF）

**如何解决？**
- 放弃离散，用连续PDF → 高斯分布

---

### 3.2 矛盾2：零维 vs 有体积

- 点云是δ函数（零维）→ 投影后面积为零
- 我们需要投影后有面积 → 3D primitive 本身必须有"厚度"

**如何解决？**
- 给点加"体积" → 用协方差矩阵定义3D椭球

---

### 3.3 矛盾3：刚体变换下的投影行为

- 如果我们定义一个3D球体，旋转后投影到2D会**变形**
- 如何保证投影后的2D形状是"正确的"？需要精确的投影公式

**如何解决？**
- 用数学推导投影公式 → 3D高斯 → 2D高斯的变换是解析的

---

### 3.4 矛盾总结

> 我们需要一个3D连续分布，它：
> 1. 有有限体积（不是δ函数）
> 2. 旋转时，其2D投影能自动计算正确的形状和大小
> 3. 投影是快速可计算的（避免ray marching）

---

## 四、Solution Path：唯一合理的解决路径

### 4.1 Step 1：选择3D高斯分布作为primitive

**定义**：
对于每个 primitive i，参数为：

| 参数 | 符号 | 维度 | 含义 |
|------|------|------|------|
| 位置 | **μ** | (3,) | 3D中心 |
| 协方差 | Σ | (3,3) | 3D形状（尺度+旋转） |
| 颜色 | c | (3,) | RGB |
| 不透明度 | α | (1,) | 0~1 |

**3D高斯PDF**：
```
G(x) = exp(-0.5 * (x - μ)^T Σ⁻¹ (x - μ))
```

**为什么协方差矩阵能表示任意椭球？**
- Σ 的特征分解：Σ = R Λ Rᵀ
  - R：旋转矩阵（特征向量）
  - Λ：对角矩阵，对角元素 = 各轴方差 σ₁², σ₂², σ₃²
- 等密度面是椭球，半轴 = √σᵢ
- 任意椭球 = 单位球 + 旋转 + 各轴缩放

---

### 4.2 Step 2：投影公式推导（核心发明）

**问题**：给定3D高斯 G(x)，在相机成像平面上的2D投影是什么？

**数学工具**：
- 相机模型：世界点 X ∈ ℝ³ → 像素坐标 x ∈ ℝ²
  - 投影函数：x = π(X) = K * (R * X + t)
  - K：内参矩阵，R/t：外参
- 我们需要：3D高斯在2D上的**边际分布**

#### 推导1：正交投影（简化）

令投影矩阵为 W ∈ ℝ²ˣ³（从3D到2D线性变换）

3D高斯：N(μ, Σ)
2D投影：x = W * X

则投影分布为：
```
x ~ N(μ₂D, Σ₂D)
其中：
  μ₂D = W * μ
  Σ₂D = W * Σ * Wᵀ
```

**关键洞察**：
- 投影是**解析的**！不需要采样，直接计算均值和协方差
- 速度O(1) per primitive（矩阵乘法）

---

#### 推导2：透视投影（实际相机）

**挑战**：透视投影是非线性的（x = X/Z）

**小孔模型**：
```
x = K * [X_x/X_z, X_y/X_z]ᵀ
```

**推导**（见3DGS论文附录）：

令相机坐标系下的3D点：X = R * μ + t
透视投影：x = K * [X_x/X_z, X_y/X_z]ᵀ

**雅可比矩阵 J**（投影变换的线性近似）：
```
J = ∂x/∂X = [1/Z  0  -X_x/Z²]
            [0  1/Z  -X_y/Z²]  * K
```

**投影协方差**：
```
Σ_2D = J * W * Σ * Wᵀ * Jᵀ
其中 W = R（将世界坐标转到相机坐标系）
```

**简化理解**：
- 透视投影相当于在3D空间先乘以R，再用非线性x=X/Z投影
- 雅可比J近似了非线性在局部的影响
- 对于靠近中心线（小视差）的高斯，近似是准确的

**实际实现**：
- 计算 Σ_cam = R * Σ * Rᵀ（转到相机坐标系）
- 计算 Σ_2D = J * Σ_cam * Jᵀ

---

### 4.3 Step 3：2D高斯的Alpha Blending

**2D高斯在像素上的贡献**：
对于屏幕上的2D高斯 N(μ₂D, Σ₂D)，在像素 x 处的值为：
```
value = α * exp(-0.5 * (x - μ₂D)^T Σ₂D⁻¹ (x - μ₂D))
```

**渲染方程（Alpha Blending）**：
按深度（μ₂D的z值）从前往后排序所有高斯：
```
C = 0
for g in sorted_gaussians:
    C = C + g.α * g.color * (1 - accumulated_α)
    accumulated_α += g.α
```

**为什么需要排序？**
- Alpha blending 不满足交换律
- 必须保证远的先画，近的覆盖
- 实际实现：按投影后的深度z排序（近似）

---

### 4.4 Step 4：参数优化 - 梯度反向传播

**可微性**：
- 投影公式可微（矩阵乘法）
- Alpha blending 可微（每项都是连续函数）
- 因此整个渲染管线可微

**梯度来源**：
1. 图像重建损失（L1 + SSIM）对最终颜色C求导
2. 链式法则：dL/d(μ, Σ, α, c) = dL/dC * dC/d(参数)

**关键优化**：
- 梯度通过渲染管线反向传播到每个高斯的参数
- 使用梯度下降更新参数（Adam等）

## 五、Compression Mechanisms：如何可扩展

### 5.1 稀疏性

**动态密度控制**：
- 只保留"有意义"的高斯（densification/pruning）
- 初始：~100k高斯（从SfM点云初始化）
- 训练中：动态densify（分裂）和prune（删除）
- 最终：~1-5M高斯（取决于场景复杂度）

### 5.2 内存效率

每个高斯：μ(3) + Σ(6对称) + α(1) + c(3) = 13 float
- 1M高斯 = 52 MB（float32）→ 可接受

### 5.3 渲染速度

- 每帧投影所有高斯：O(N)矩阵乘法
- 排序：O(N log N)（但N=1M时仍可接受）
- CUDA并行：每个像素同时处理多个高斯

## 七、为什么高斯是必然的？（总结）

### 7.1 需求对比表

| 要求 | 体素 | 点云 | NeRF | **3DGS** |
|------|------|------|------|----------|
| 快速查询 | ✅ | ✅ | ❌ | ✅ |
| 连续渲染 | ❌ | ❌ | ✅ | ✅ |
| 内存效率 | ❌ | ✅ | ✅ | ✅ |
| 可微 | ✅ | ✅ | ✅ | ✅ |
| 质量 | ❌ | ❌ | ✅ | ✅ |

### 7.2 必然性链条

1. 需要连续 → 必须用PDF（不是离散体素或δ点）
2. 需要投影快速 → PDF必须是高斯（其他分布投影复杂）
3. 需要任意形状 → 必须用协方差矩阵（不是球）
4. 需要稀疏表示 → 多个高斯叠加（不是单一高斯）

### 7.3 3DGS的简洁与强大

- 只增加13个参数/primitive
- 但获得完整的3D形状 + 2D投影 + 可微渲染
- 没有额外网络，纯数学公式
