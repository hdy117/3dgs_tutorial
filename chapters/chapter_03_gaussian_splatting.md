# 第 3 章：核心发明——为什么是高斯？

**本章核心问题**：从“点云不够用”出发，如果你必须发明一种既稀疏、又连续、还能快速投影到图像上的 3D 表示，为什么最后会走到高斯？

如果你刚读完 [../linear_algebra.md](../linear_algebra.md)，这一章你应该带着一个很明确的视角进来：

- 向量描述位置和方向
- 矩阵描述空间动作
- 协方差矩阵可以把一个椭球的方向和尺度压进 `3 x 3` 数表
- 局部非线性问题常常会被 Jacobian 拉回线性代数能处理的范围

3D Gaussian Splatting 的核心发明，本质上就是把这些东西接成一条完整的工程链。

---

## 一、从点云的困境开始

### 1.1 点为什么不够

点云最开始看起来很诱人。

原因很简单：

- 稀疏
- 内存小
- 每个点只要存位置和颜色

但它有一个致命问题：**点是零维的。**

你把点投到屏幕上，很容易出现这样：

```text
屏幕上你真正想看到的是连续表面

xxxxxxxxxxxx
xxxxxxxxxxxx
xxxxxxxxxxxx

但点云投影后往往像这样

x .  x   . x
.   . x .  .
x .    .  x
```

也就是说：

- 点和点之间有洞
- 换个视角，洞的位置会变
- 稍微放大就容易看到离散感

所以如果你想做的是“看起来像真实表面”的新视角合成，只有点是不够的。

### 1.2 那我们到底缺什么

我们缺的不是更多点，而是：

> 一个围绕点展开的、连续的、局部有体积的影响区域。

换句话说，我们不再想让一个点只贡献“一个像素点”，而是想让它贡献“一小团连续影响”。

你可以把这个需求想成：

```text
点          -> 太尖，太脆，太离散
小体积元    -> 能在屏幕上形成连续 footprint
```

### 1.3 这个“小体积元”必须同时满足哪些约束

如果你真要发明它，至少会被下面五个约束同时逼住：

| 约束 | 为什么必须满足 |
|------|----------------|
| 稀疏 | 否则会退回体素那种爆内存方案 |
| 连续 | 否则屏幕上还是会有硬边和空洞 |
| 可各向异性 | 否则无法表达“沿某个方向很长、沿另一个方向很薄”的局部表面 |
| 投影要快 | 否则会重新掉进 NeRF 式的慢渲染 |
| 可微 | 否则训练时梯度没法稳定流回参数 |

所以真正的问题不是“随便找个 3D 形状”，而是：

> 找一种在这五个约束下最自然的局部 primitive。

## 二、候选形状逐个看：为什么最后是高斯

### 2.1 先把常见候选摆上桌

你可能会想到这些东西：

| 候选 | 看起来好在哪 | 真正的问题 |
|------|--------------|------------|
| 固定方块 / billboard | 简单，容易画 | 边界硬，透视下容易假 |
| 球 | 连续，参数少 | 只能各向同性，太笨 |
| 一般椭球 | 能各向异性 | 作为“均匀实体”时，投影和积分处理不够优雅 |
| 高斯椭球 | 连续、各向异性、局部衰减自然 | 需要接受“它不是硬表面，而是一团密度云” |

### 2.2 为什么固定方块不够好

最简单的想法是：

```text
每个点别再当点了
给它一个固定大小的小方块
```

这确实能让点不再是零维，但问题也很快出现：

- 方块边界是硬的
- 不同视角下容易出现不自然的遮挡和边缘
- 它本质上更像“贴片”而不是“局部几何密度”

所以它虽然能补洞，但补得很粗糙。

### 2.3 为什么球也不够

球比方块好，因为：

- 它连续
- 它旋转不变
- 屏幕 footprint 也比较自然

但球有一个根本短板：**它太各向同性了。**

真实场景中的局部表面往往更像这样：

```text
薄薄一层表面

================
```

你希望局部 primitive：

- 沿切平面方向展开一些
- 沿法线方向薄一些

也就是它应该更像一个扁椭球，而不是球。

球做不到这点。

### 2.4 一般椭球已经很接近了

如果你继续往前想，很自然会走到椭球：

```text
球        -> 所有方向一样
椭球      -> 不同方向可以不同
```

这已经非常接近正确答案了。

因为局部表面就是需要：

- 某些方向展开更宽
- 某些方向收得更窄
- 还能整体旋转到任意朝向

所以从几何表达力上看，椭球几乎已经到位。

### 2.5 真正决定胜负的，不是“像不像椭球”，而是“变换后还好不好算”

到这一步，关键问题已经不再是“谁更像一个形状”，而是：

> 你把它放进完整渲染链之后，谁更适合被计算？

尤其是下面这件事：

```text
3D 里的 primitive
    -> 世界到相机
    -> 透视投影
    -> 屏幕上的 2D footprint
    -> 像素贡献
```

谁在这条链上：

- 形式保持得更好
- 参数传播更自然
- 更容易做局部线性化
- 更容易可微分

谁就更有资格成为工程答案。

而高斯在这里几乎是压倒性胜出。

## 三、高斯真正赢在哪：它不是“某种形状”，而是一种特别好传播的局部密度

### 3.1 先别把它当概率论公式，先把它看成一团局部影响云

在 3DGS 里，一个高斯最应该先被想成：

> 围绕某个中心展开的一团连续影响云。

它不是硬表面，不是实心球，也不是有锋利边界的体块。

它更像：

```text
离中心近 -> 影响大
离中心远 -> 影响快速衰减
```

这件事非常重要，因为它天然带来：

- 连续边界
- 平滑混合
- 局部支持域
- 可微的衰减曲线

### 3.2 一个 3D 高斯最核心的几何参数是什么

一个 3D 高斯最核心的几何参数只有两个：

- `mu`：中心
- `Sigma`：形状

如果你只看几何，不看颜色和透明度，那么它就是：

```text
高斯 = 中心 mu + 协方差 Sigma
```

其中：

- `mu` 决定它在空间哪里
- `Sigma` 决定它像什么椭球

### 3.3 为什么 `Sigma` 可以直接看成椭球

这是整章最重要的桥梁之一。

如果 `Sigma` 是对称正定矩阵，那么它可以拆成：

```text
Sigma = R @ diag(sx^2, sy^2, sz^2) @ R.T
```

这行式子的几何意义非常直接：

- `R`：把局部主轴转到正确朝向
- `diag(sx^2, sy^2, sz^2)`：沿三个主轴分别决定展开强度

也就是说：

```text
Sigma 不是抽象矩阵
Sigma 就是“旋转过的椭球尺度信息”
```

### 3.4 一个小画面：为什么这就是椭球

你可以把它想成这样。

先在局部坐标里放一个轴对齐椭球：

```text
        z
        ^
        |
      __|__
   .-'  |  '-.
  /     |     \
 |------o------| -> x
  \     |     /
   '-.__|__.-'
        |
        v y
```

然后再整体转到场景中的正确朝向。

这一步就是 `R` 在做的事。

所以：

- 椭球的主轴方向来自 `R`
- 椭球的三条轴长来自 `sx, sy, sz`
- 这两者一起被压进 `Sigma`

### 3.5 为什么 3DGS 实现里不直接优化 `Sigma` 的 6 个自由度

因为直接优化矩阵虽然看起来直观，但训练时容易出数值问题。

更常见的实现是：

```text
先存 scale + rotation
再重建 Sigma
```

也就是：

```text
Sigma = R @ diag(scale^2) @ R.T
```

这样做的好处是：

- `scale > 0` 更容易保证正定性
- `R` 用四元数或正交化方式更容易保持“真旋转”
- 优化更稳定

所以从工程角度看，`Sigma` 是几何概念上的核心，`scale + rotation` 是数值实现上的核心。

## 四、为什么高斯在变换下特别“听话”

这一步是高斯真正胜出的关键。

### 4.1 先看一句最重要的话

如果有一个高斯：

```text
X ~ N(mu, Sigma)
```

然后你对它做一个线性或仿射变换：

```text
Y = A X + b
```

那么结果还是高斯，而且参数会这样变：

```text
Y ~ N(A mu + b, A Sigma A.T)
```

这句话特别重要，因为它意味着：

> 高斯在空间变换之后，不会突然变成一种完全不好处理的新对象。

它只是：

- 中心被移走了
- 椭球被转了/拉了

但“高斯”这个身份没丢。

### 4.2 这件事为什么在 3DGS 里如此关键

因为渲染链里最基本的动作恰好就是各种变换：

```text
局部空间 -> 世界空间 -> 相机空间
```

而这些步骤里，大量都是线性或仿射动作。

一旦你用高斯，你就可以非常自然地写出：

```text
mu_cam    = R @ mu_world + t
Sigma_cam = R @ Sigma_world @ R.T
```

这不是硬背出来的公式，而是“高斯在线性变换下封闭”这件事的直接结果。

### 4.3 换句话说，高斯赢在它和线性代数特别兼容

这也是它和前两篇文档完全接上的地方。

高斯之所以强，不只是因为它“长得像个椭球”，而是因为：

- 它能用 `mu` 和 `Sigma` 紧凑表达
- `Sigma` 本身就是线性代数对象
- 空间变换时参数传播非常自然
- 局部投影时又能继续借助 Jacobian 传播协方差

所以高斯并不是一个孤立技巧，它其实是线性代数语言在 3D 表示上的一次自然落地。

## 五、从 3D 高斯到 2D 屏幕 footprint

到这里，我们已经有了空间里的高斯椭球。接下来要做的，是把它变成屏幕上的 2D 影响区域。

### 5.1 第一步：先把高斯送到相机坐标系

这一步最干净，因为它基本是刚体/仿射变换。

```text
mu_cam    = R @ mu_world + t
Sigma_cam = R @ Sigma_world @ R.T
```

这一步之后，你可以把每个高斯理解成：

> 已经从“世界里的椭球”变成了“相机眼里看到的椭球”。

### 5.2 第二步：中心的透视投影是精确的

高斯中心 `mu_cam = [X, Y, Z]` 的屏幕投影中心是：

```text
u = fx * X / Z + cx
v = fy * Y / Z + cy
```

也就是常见的小孔成像模型。

所以 2D footprint 的中心位置很好算。

### 5.3 真正麻烦的是：椭球的形状怎么投到屏幕上

问题在这里：

```text
(X, Y, Z) -> (X/Z, Y/Z)
```

透视投影里有除法，所以它不是纯线性的。

这意味着：

- 你不能直接拿一个固定 `2 x 3` 矩阵，把 3D 协方差一次性“精确投”到 2D
- 全局上它是非线性的

但这还不是坏消息。

因为每个高斯本来就很局部。

### 5.4 关键洞察：高斯很小，所以在中心附近可以线性化

这正是 3DGS 最漂亮的一步。

如果一个高斯足够小，那么在它的中心附近，透视投影虽然全局非线性，但局部可以近似成线性。

也就是说，在 `mu_cam` 附近，小扰动 `dx` 满足：

```text
dp ≈ J dx
```

其中 `J` 是投影函数在 `mu_cam` 处的 Jacobian。

这句话翻译成人话就是：

> 你不用精确理解整个透视投影怎样扭曲空间，只需要知道在这个小高斯周围，它局部像一个什么线性拉伸/压缩。

### 5.5 于是协方差就能按线性规则传播

一旦局部线性关系成立，协方差就自然满足：

```text
Sigma_2d ≈ J @ Sigma_cam @ J.T
```

这就是 3DGS 里那个最著名的式子之一。

它的真正含义不是“又一个推导公式”，而是：

> 3D 椭球在当前视点附近，被局部线性地压成了屏幕上的 2D 椭圆。

### 5.6 一个画面：空间椭球变成屏幕椭圆

```text
3D 里：

        一个有朝向的椭球
              ______
          _.-'      '-._
       .-'              '-.
      /                    \
      \                    /
       '-._            _.-'
            '--.___.--'

投到屏幕上后：

         一个 2D 椭圆 footprint
            .-''''-.
         .-'        '-.
        /              \
        \              /
         '-.        .-'
            '-.__.-'
```

所以 3DGS 的“splat”并不是随便画一个圆，而是：

- 先从 3D 高斯得到 2D 协方差
- 再用这个 2D 协方差决定屏幕 footprint 的椭圆形状

## 六、从 2D 椭圆 footprint 到像素贡献

现在每个高斯在屏幕上都有：

- 一个 2D 中心 `mu_2d`
- 一个 2D 协方差 `Sigma_2d`

它于是定义了一个 2D 高斯。

### 6.1 为什么像素贡献要写成二次型

对某个像素 `p`，相对中心的偏移记作：

```text
d = p - mu_2d
```

高斯值会写成类似：

```text
exp(-0.5 * d.T @ inv(Sigma_2d) @ d)
```

这不是装饰，它在测量：

> 在这个椭圆自己的主轴尺度下，这个像素离中心有多远。

如果某个方向很宽，那这个方向偏一点并不算远。
如果某个方向很窄，那同样的像素偏移就可能已经很远了。

所以 `Sigma_2d` 决定的不是“一个普通距离”，而是“椭圆几何下的距离”。

### 6.2 加上透明度和颜色，一个高斯就能给像素贡献颜色

于是一个高斯在某个像素处的有效权重大致是：

```text
weight = alpha * gaussian_2d_value
```

它再乘上自己的颜色，就得到对这个像素的贡献。

### 6.3 但一个像素通常会被多个高斯同时影响

这时就需要把多个高斯按深度顺序混合起来。

也就是经典的 alpha blending。

从物理直觉上看：

- 远处的先贡献
- 近处的再遮住一部分
- 累积起来形成最终像素颜色

所以高斯不是“谁最大就用谁”，而是“多个局部密度层一起混合”。

## 七、为什么这套表示又快又能训

到这里你可以回头看，3DGS 的答案之所以漂亮，是因为它同时解决了表示、投影、渲染、优化四件事。

### 7.1 表示层面：它足够稀疏

每个高斯只存：

- 中心
- 形状
- 不透明度
- 颜色

你不需要像体素那样把整个空间填满。

### 7.2 几何层面：它足够连续

高斯没有硬边，它天然给出平滑的局部过渡。

这让屏幕 footprint 不再像硬贴片那样容易穿帮。

### 7.3 投影层面：它和线性代数极度兼容

你几乎可以把整条核心链压缩成三句话：

```text
世界到相机：      Sigma_cam = R @ Sigma @ R.T
局部投影线性化：  dp ≈ J dx
屏幕协方差：      Sigma_2d ≈ J @ Sigma_cam @ J.T
```

这就是为什么前面的线性代数不是“前置知识负担”，而是这里的母语。

### 7.4 渲染层面：它只影响局部像素

高斯值会快速衰减，所以你根本不需要让一个高斯去影响整张图。

这带来两件工程优势：

- 局部包围盒很好算
- tile-based culling 很有效

也就是说，它既连续，又局部。

这正是高效渲染最喜欢的组合。

### 7.5 优化层面：它整体可微

- 位置可微
- 协方差传播可微
- 2D 高斯评估可微
- alpha blending 大部分链路可微

这就让“从图像误差反向调整 3D 高斯参数”变得可行。

这件事非常关键，因为没有它，3DGS 就只能是一个表示，不会是一个能训练起来的系统。

## 八、把整章压成一条最短主线

如果你只想记住一条故事线，就记这个：

```text
点云太稀，投到屏幕上有洞

    ↓

我们需要“围绕点展开的一小团连续体积影响”

    ↓

这个局部 primitive 必须：稀疏、连续、可各向异性、易投影、可微

    ↓

球太笨，方块太硬，一般实体椭球不够好传播

    ↓

高斯最自然，因为它既是椭球云，又在线性/局部线性变换下特别好传

    ↓

于是得到：
mu + Sigma 表示 3D 椭球云
R @ Sigma @ R.T 做空间传播
J @ Sigma_cam @ J.T 做局部投影传播

    ↓

最后在屏幕上形成 2D 椭圆 footprint，再做 alpha blending
```

这就是 3D Gaussian Splatting 的核心发明。

## 九、本章你真正应该会的，不是定义，而是重建能力

遮住答案后，你至少应该能自己回答这几个问题：

1. 为什么点云天然会有洞？
2. 为什么我们想要的是“局部连续影响区域”，而不是单个点？
3. 为什么球不足以表达真实局部表面？
4. 为什么 `Sigma` 可以被看成椭球？
5. 为什么 `Sigma = R @ diag(scale^2) @ R.T` 这件事几何上完全合理？

7. 为什么 `Sigma_2d ≈ J @ Sigma_cam @ J.T` 不是魔法，而是局部线性化 + 协方差传播？
8. 为什么 3DGS 里的 screen-space splat 最终是一个椭圆，而不是圆或方块？

如果这些问题你能自己重建出来，这一章就不是“会背”，而是真的进入你的脑子里了。
