## 二、形状的候选与淘汰——数学筛选过程

### 2.1 大比拼：常见 3D 形状的投影特性

假设我们要给每个点定义一个"影响范围"（3D 体积），有哪些候选？

| 形状 | 参数 | 投影到 2D 是什么？ | 边界连续？ | 各向异性？ | 投影复杂度 |
|------|------|-------------------|------------|------------|------------|
| **立方体** | 中心 +3 尺度 | 六边形（多边形） | ❌ 不连续 | ✅ 可拉伸 | 复杂（需光栅化） |
| **球体** | 中心 + 半径 | 圆 | ✅ 连续 | ❌ 只能各向同性 | 简单 |
| **椭球** | 中心 +3 尺度 + 方向 | 椭圆 | ✅ 连续 | ✅ 可拉伸 | 中等（需推导公式） |
| **高斯分布** | μ+Σ(9 参) | **仍是高斯** | ✅ 解析连续 | ✅ 协方差完整描述 | **闭式解** |

**淘汰逻辑**：

**立方体为什么不行？**
- 投影是六边形 → 边界不连续（放大看有棱角）
- 光栅化复杂 → 需要逐像素判断是否在多边形内
- ❌ 违反"连续性 + 快速投影"

**球体为什么不够？**
- 只能各向同性（半径 r，所有方向一样大）
- **但真实表面需要各向异性**！比如：
  - 桌子表面的点应该沿法线方向拉长
  - 边缘的点应该在切平面内扩散
- ❌ 表达能力不足

#### 🔍 关键概念细化：**什么是"各向异性"**？
<details>
<summary>点击展开</summary>
各向同性（球）：所有方向尺度一样 → radius = 0.1（一个数）
各向异性（椭球）：不同方向可以不同 → scale_x, scale_y, scale_z = (0.5, 0.5, 0.05)
真实表面需求：平坦处沿切平面扩散，细长物沿轴向拉长。

**椭球看起来不错？——但有陷阱**
- 可以描述任意方向的拉伸 ✅
- **但投影公式呢？** ——均匀分布的椭球，投影截面面积计算复杂
- 需要数值积分或近似 → ❌ 违反"解析解"要求

**高斯为什么胜出？**
- 3D 高斯投影到 2D **仍是高斯**（有闭式解）✅
- 协方差矩阵 Σ完整描述椭球形状（各向异性）✅
- 数学性质好：可微、旋转不变、数值稳定 ✅

### 2.2 🔬 第一性原理推导：为什么只有高斯？

让我们从公理出发，看看这是不是**唯一解**。

**公理列表**：
```
A1: 需要连续函数（质量要求）→ 概率密度函数 (PDF)
A2: 投影要有解析公式（速度要求）→ 线性变换封闭性
A3: 形状要各向异性（表达能力）→ 协方差矩阵描述
A4: 可微分（训练要求）→ 所有操作需支持反向传播
```

**问题**：哪些概率分布满足"线性变换后仍是同类分布"？

**候选排查**：

| 分布 | PDF 形式 | 线性变换封闭性？ | 各向异性？ |
|------|----------|------------------|------------|
| 均匀分布 (Uniform) | 1/V 在椭球内，0 在外 | ❌ 投影截面复杂 | ✅ |
| 指数分布 | exp(-\|\|x\|\|/λ) | ❌ 无解析形式 | ⚠️ 各向同性变体 |
| **高斯 (正态)** | exp(-½(x-μ)ᵀΣ⁻¹(x-μ)) | ✅ **仍是高斯** | ✅ Σ完整描述 |

**关键数学事实**：

#### 🔍 关键概念细化：**为什么高斯有线性变换封闭性？（简短证明）**

<details>
<summary>点击展开</summary>

**问题**: X ~ N(μ, Σ)，Y = AX + b，证明 Y ~ N(Aμ+b, AΣAᵀ)

**证明思路**：用概率密度函数的变量代换公式。

```math
# X 的 PDF:
p_X(x) ∝ exp(-½(x-μ)ᵀ Σ⁻¹ (x-μ))

# Y = AX + b → x = A⁻¹(y-b) （假设 A 可逆）
# Jacobian: |∂x/∂y| = |A⁻¹|

# 变量代换：
p_Y(y) = p_X(A⁻¹(y-b)) · |A⁻¹|
       ∝ exp(-½(A⁻¹(y-b) - μ)ᵀ Σ⁻¹ (A⁻¹(y-b) - μ)) · |A⁻¹|

# 化简指数项：
A⁻¹(y-b) - μ = A⁻¹(y - b - Aμ) = A⁻¹(y - (Aμ+b))

代入：
(y - (Aμ+b))ᵀ (A⁻¹)ᵀ Σ⁻¹ A⁻¹ (y - (Aμ+b))
= (y - (Aμ+b))ᵀ (AΣAᵀ)⁻¹ (y - (Aμ+b))

# 结论：p_Y(y) = N(y | Aμ+b, AΣAᵀ) ✅
```

**关键步骤**: `(A⁻¹)ᵀ Σ⁻¹ A⁻¹ = (AΣAᵀ)⁻¹`（矩阵逆的性质）

**物理意义**: 高斯分布在线性变换下"形状不变"——只是均值和协方差跟着变换。这就是为什么：
- **投影是线性的（局部近似）** → 3D 高斯投影后仍是 2D 高斯
- **旋转/缩放也是线性变换** → 高斯在这些操作下保持形式

其他分布（均匀、指数等）没有这个性质——它们在线性变换后会变成完全不同的形式！
</details>

> 如果 X ~ N(μ, Σ)，对任意线性变换 Y = AX，那么 Y ~ N(Aμ, AΣAᵀ)

这就是**投影封闭性**——高斯的唯一性来源。

### 2.3 验证：掩住答案，能独立重推吗？

试着从三个需求出发推导：
1. **稀疏** → 点云思想（只存表面）✅
2. **连续体积** → 不能是离散形状（立方体/点），需要 PDF ✅
3. **解析投影** → 线性变换封闭性 → **只有高斯满足** ✅

**结论**：从约束出发，高斯不是"偶然发现"——它是**唯一满足所有条件的解**。

## 三、3D 高斯的数学表示——完整推导

### 3.1 定义与参数化

**3D 高斯概率密度函数**：

```math
G(x) = \frac{1}{\sqrt{(2\pi)^3 |\Sigma|}} \exp\left(-\frac{1}{2} (x - \mu)^\top \Sigma^{-1} (x - \mu)\right)
```

**拆解每个符号**：
- **μ ∈ ℝ³**：中心位置（世界坐标），3 个参数
- **Σ ∈ ℝ³ˣ³**：协方差矩阵，对称正定，6 个独立参数
- **|Σ|**：行列式，表示"体积平方"
- **Σ⁻¹**：精度矩阵（inverse covariance）


#### 🔍 关键概念细化：**为什么是 6 个独立参数，不是 9 个？**

<details>
<summary>点击展开</summary>

**协方差矩阵是对称的！**
```math
Σ = [σ_xx σ_xy σ_xz; σ_yx σ_yy σ_yz; σ_zx σ_zy σ_zz]
其中 σ_ij = σ_ji（对称性）→ 独立参数只有 6 个：3 个对角 +3 个上三角

等价表示方式对比：
- 直接存 Σ: 6 个数，但梯度下降可能失去正定性 ❌  
- 尺度 (3) + 四元数 (4, |q|=1): scale>0 + 归一化 → **自动保证正定** ✅

这就是 3DGS 为什么用"scale + quat"参数化的原因！
</details>

**总自由度**：3 + 6 = 9 个参数描述一个 3D 高斯

### 3.2 协方差矩阵的几何意义

Σ是核心——它决定了椭球的形状和朝向。怎么理解？

**特征值分解**：
```math
\Sigma = V \cdot \Lambda \cdot V^\top
```
- **V**：特征向量矩阵（正交），每一列是一个主轴方向 → 旋转信息
- **Λ**：对角线矩阵，元素是特征值 λ₁,λ₂,λ₃ → 各轴方差

**等价表示**（更直观）：
```math
\Sigma = R \cdot \text{diag}(\sigma_x^2, \sigma_y^2, \sigma_z^2) \cdot R^\top
```
- **R**：旋转矩阵（3 个自由度，欧拉角或四元数）
- **σₓ, σᵧ, σ₂**：各轴标准差（尺度），半长轴长度

**为什么是"协方差"？** —— 统计学的直觉：
```
G(x) ∝ exp(-马氏距离² / 2)
马氏距离 = \sqrt{(x-μ)ᵀ Σ⁻¹ (x-μ)}
```

这个"马氏距离"定义了椭球——距离中心越远，密度越低。Σ决定了"多远算远"（各轴尺度不同）。

### 3.3 🔬 深入：协方差分解的数值实现

**问题**：训练时怎么优化 Σ？直接存 6 个元素会有问题...

**陷阱**：对称矩阵在梯度下降中可能失去正定性 → exp(负数) 爆炸！

**实际做法（3DGS 论文）**：
```python
# 不直接存储 Σ，而是分解成"尺度 + 旋转"
scale = (s_x, s_y, s_z) ∈ ℝ³₊  # 各轴半长 > 0
rot   = (r_w, r_x, r_y, r_z) ∈ 𝕊³  # 四元数，单位长度

# 重建 Σ:
R = quaternion_to_matrix(rot)      # 4→3×3 旋转矩阵
S = diag(scale²)                   # 方差（尺度平方）
Σ = R @ S @ Rᵀ                     # 保证对称正定 ✅
```

**为什么这么做？**
- scale > 0 → 特征值 > 0 → 正定性保证
- 四元数归一化 → 旋转矩阵保持正交性
- **数值稳定，不会优化崩溃**

## 四、关键突破：3D→2D 投影的闭式解

### 4.1 问题定义

现在你有一个 3D 高斯 G(x) = N(μ, Σ)，想把它投影到 2D 成像平面。

**物理场景**：
```
世界坐标系 → [相机外参 R,T] → 相机坐标系 → [透视投影 K] → 图像像素坐标
```

**简化问题**：先在相机坐标系里推导（R=I, T=0），最后加一般情况。

### 4.2 透视投影的非线性陷阱

**朴素想法**：直接投影均值 μ，协方差 Σ 怎么变？

**3D 点在相机系**：x_c = [X, Y, Z]ᵀ  
**投影到归一化平面**：x_n = [X/Z, Y/Z]ᵀ ← **这不是线性变换！**

```
问题：Z 在分母上，协方差没法直接乘矩阵...
```

### 4.3 🔬 3DGS 的巧妙近似：局部线性化

**核心洞察**：高斯在空间中是"局域"的（集中在 μ附近）。如果高斯足够小，可以假设它在投影时 Z 变化不大 → **用一阶泰勒展开近似**。

**步骤 1: 均值投影（精确）**
```math
\mu_c = [\mu_X, \mu_Y, \mu_Z]^\top
\mu_n = [\mu_X/\mu_Z, \mu_Y/\mu_Z]^\top      # 归一化坐标
\mu_{2d} = K \cdot \begin{bmatrix}\mu_n \\ 1\end{bmatrix} = [f_x \mu_X/\mu_Z + c_x,\; f_y \mu_Y/\mu_Z + c_y]^\top
```

**步骤 2: 协方差投影（近似）**

透视投影的映射：
```math
\Pi(X,Y,Z) = (X/Z, Y/Z)
```

在 μ_c 附近做一阶泰勒展开，得到**雅可比矩阵 J**：
```math
J = \frac{\partial \Pi}{\partial [X,Y,Z]}\bigg|_{x=\mu_c} 
  = \begin{bmatrix}
      \frac{\partial (X/Z)}{\partial X} & \frac{\partial (X/Z)}{\partial Y} & \frac{\partial (X/Z)}{\partial Z} \\
      \frac{\partial (Y/Z)}{\partial X} & \frac{\partial (Y/Z)}{\partial Y} & \frac{\partial (Y/Z)}{\partial Z}
    \end{bmatrix}_{x=\mu_c}

  = \begin{bmatrix}
      1/\mu_Z & 0 & -\mu_X/\mu_Z^2 \\
      0 & 1/\mu_Z & -\mu_Y/\mu_Z^2
    \end{bmatrix}
```

**J 的物理含义**：描述了"3D 空间在 μ_c 附近如何被拉伸/压缩到 2D"。

协方差变换公式（误差传播定律）：
```math
\Sigma_n = J \cdot \Sigma_c \cdot J^\top      # 归一化平面的 2×2 协方差
\Sigma_{2d} = K \cdot \Sigma_n \cdot K^\top   # 像素坐标的 2×2 协方差

其中 K = \begin{bmatrix}f_x & 0 & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1\end{bmatrix}
```

**完整公式（世界→相机→像素）**：
```math
# 世界坐标 → 相机坐标
\mu_c = R_{wc} \cdot (\mu_w - C)    # C 是相机中心
\Sigma_c = R_{wc} \cdot \Sigma_w \cdot R_{wc}^\top

# 相机坐标 → 像素（如上）
J = \begin{bmatrix}1/\mu_Z & 0 & -\mu_X/\mu_Z^2 \\ 0 & 1/\mu_Z & -\mu_Y/\mu_Z^2\end{bmatrix}
\Sigma_{2d} = K \cdot (J \Sigma_c J^\top) \cdot K^\top
```

**关键结论**：3D 高斯投影到 2D **仍是高斯**，参数有闭式解！

### 4.4 🔬 第一性验证：为什么近似有效？

**问题**：透视投影是非线性的（Z 在分母），为什么线性化后还能用？

**答案**：3DGS 的每个高斯都很小（尺度 σ~0.01-0.1 米），而场景深度 Z~几米。所以：
```
相对变化 = σ / Z ~ 0.1 / 5 = 2% → 非线性项可忽略 ✅
```

**验证方法**：如果高斯太大（σ > Z/3），近似会失效 → 3DGS 的 densify 策略会自动拆分大高斯，保证局部线性化有效。

## 五、渲染管线：从投影到像素混合


#### 🔍 关键概念细化：**Alpha Blending 公式的物理推导**

<details>
<summary>点击展开</summary>

**从光学基本定律出发**：

考虑光线穿过半透明介质，在位置 t 处有微元 dt：
- **被吸收的光**: dI = I(t) · σ(t) · c(t) · dt  
- **透过的光**: I(t+dt) = I(t) - dI = I(t)·(1 - σ·c·dt)

**累积效果（从 t_n 到 t_f）**：
```
I_out = ∫ T(t)·σ(t)·c(t) dt
其中 T(t) = exp(-∫_{t_n}^{t} σ(s)ds) = "从起点到 t 的透射率"
```

**离散化（N 层半透明物体）**：
```python
C = 0, T_acc = 1.0   # 初始：黑色 + 全透射
for layer in layers_sorted_by_depth:  # 从远到近！
    alpha = layer.transparency  # ∈ [0,1]，该层的不透明度
    
    C += T_acc * (1-alpha) * layer.color  # 新增贡献 = 剩余光线 × 透过率 × 颜色  
    T_acc *= alpha                        # 累积透射率衰减
```

**为什么必须从后到前？**
- **物理正确性**: 先计算远处的贡献，再被近处遮挡
- **早停优化**: T_acc < ε时可以直接 break（前面已经挡住了）
- **不可交换性**: (A 混合 B) ≠ (B 混合 A)，顺序很重要！

**对比 NeRF vs 3DGS**：
- NeRF: Ray Marching = 沿射线积分体积密度
- 3DGS: Alpha Blending = 离散高斯层的叠加（数学等价，但计算方式不同）

这就是为什么 2D 投影后要做"排序 + 混合"——这是物理光传输的必然要求！
</details>

### 5.1 Alpha Blending——为什么是"从后到前"？

现在有 N 个 2D 高斯投影在图像上，每个都有颜色 cᵢ和透明度 αᵢ。怎么合成最终像素颜色？

**物理直觉**：光线从眼睛射出 → 先遇到远处的物体 → 再被近处的遮挡。

```python
C = [0, 0, 0]      # 初始颜色（黑色背景）
T = 1.0            # 累积透射率（100% 光线通过）

# 按深度排序：从远到近
for i in sorted_gaussians(by=depth, ascending=False):
    # 该高斯在像素 p 处的 2D 密度值
    g_2d = exp(-0.5 * (p - μ_2d)ᵀ Σ_2d⁻¹ (p - μ_2d))
    
    # 有效贡献权重（透明度 × 局部密度）
    w_i = α_i * g_2d
    
    # Alpha blending 公式
    C += T * w_i * c_i          # 当前高斯贡献 = 剩余光线 × 权重 × 颜色
    T *= (1 - w_i)              # 更新透射率（被吸收一部分）
    
    # 早停优化：T < 0.001 时后续贡献可忽略
    if T < 0.001: break

最终像素颜色 = C / max(T, ε)   # 归一化（避免全黑）
```

**变量含义拆解**：
- **C**：累积 RGB 颜色，三维向量
- **T**：标量，表示"还有多少光线能穿过当前高斯层"
  - T=1 → 完全透明，所有光通过
  - T=0.5 → 50% 被吸收
  - T≈0 → 不透明，后面都看不到
- **wᵢ**：单个高斯在像素 p 处的有效贡献（0≤w≤1）

```python
# Step 1: 分块
tile_width, tile_height = 16, 16
num_tiles_w, num_tiles_h = ceil(1920/16), ceil(1080/16)  # 120×68 = 8160 tiles

# Step 2: 对每个高斯，计算它影响的 tile 列表
for gaussian in gaussians:
    bbox_2d = compute_bounding_box(gaussian.mean_2d, gaussian.Sigma_2d)  
    # Σ_2d 的 3σ范围覆盖~99.7%的概率质量
    
    affected_tiles = []
    for tile_x in [bbox_min_w//16, ..., bbox_max_w//16]:
        for tile_y in [bbox_min_h//16, ..., bbox_max_h//16]:
            affected_tiles.append((tile_x, tile_y))

# Step 3: 构建"tile → 高斯列表"的索引
tile_to_gaussians = defaultdict(list)
for gaussian, tiles in zip(gaussians, all_affected_tiles):
    for tile in tiles:
        tile_to_gaussians[tile].append(gaussian)

# Step 4: 渲染时，每个 tile 只混合相关高斯
for tile_id in all_tiles:
    relevant_gaussians = tile_to_gaussians[tile_id]  # ~30-100 个
    render_tile(relevant_gaussians)  # 快速！

# 复杂度：O(N + M×k)，k=每像素平均高斯数 (~30) << N (100K)
```

**GPU 优化细节**:
- **并行化**: 每个 tile 独立渲染 → 完美映射到 GPU thread block
- **内存访问**: 连续读取一个 tile 的所有高斯参数 → cache-friendly
- **早停**: T_acc < ε时提前结束该 pixel 的混合

这就是为什么<10ms/帧能做到——不是靠更快的硬件，而是**算法复杂度从 O(N×M) 降到 O(M×k)**！
</details>

# Tile-based culling（分块剔除）
1. 把图像分成 16×16 的小块（tiles）
2. 对每个高斯，计算它的 2D 边界框（bounding box）
3. 只遍历"被该高斯覆盖的 tiles"
4. 在每个 tile 内，只混合影响它的高斯

效果：平均每个像素只混合 ~10-50 个高斯，而不是全部 N 个
```

**实际性能**（典型场景）：
- N = 100K 高斯
- 图像 = 1920×1080 ≈ 2M 像素
- 每像素混合 ~30 个高斯
- **总计算量**：2M × 30 ≈ 60M 次操作 → <10ms/帧

对比 NeRF：2M × 100 = 2亿次 MLP 查询 → **快约 3000 倍**。

```

- **NeRF**：满足 A2/A3 ✅，但违反 A1/A4 ❌（ray marching=结构性慢）

```
从点云出发（稀疏性 ✅）

```

## 七、核心记忆点与思考题

### 7.1 📝 本章必须记住的

✅ **3D 高斯参数化**：μ(位置) + Σ(协方差=旋转×尺度²) + α(透明度) + c(颜色)  
✅ **投影闭式解**：μ_2d = K·R·μ, Σ_2d = J·Σ·Jᵀ（雅可比近似）  
✅ **Alpha Blending**：从后到前混合，T=累积透射率  
✅ **高斯唯一性**：线性变换封闭性是投影解析的关键  
✅ **Tile Culling**：每像素只混合~30 个高斯 → <10ms/帧  

### 7.2 🤔 思考题

**Q1: "协方差矩阵必须正定，为什么？"**
<details>
<summary>提示</summary>
Σ⁻¹出现在 exp(-½(x-μ)ᵀΣ⁻¹(x-μ)) 中。如果 Σ不正定：
- 可能有负特征值 → 某些方向上 x→∞时 exp(正数) → ∞（概率密度爆炸）
- 物理意义破坏：密度应该从中心向外衰减，不是增长！
</details>

**Q2: "为什么 densify 基于位置梯度？"**
<details>
<summary>提示</summary>
位置梯度大 = 移动这个高斯能显著降低损失 = 该区域细节不足 → 拆分成两个高斯增加表达能力。类似"误差大的地方增加采样点"的第一性直觉。
</details>

**Q3: "透视投影的线性化近似何时失效？"**
<details>
<summary>提示</summary>
当 σ/Z > ~1/3 时（高斯尺寸相对于深度太大），Z 的变化范围大，非线性项不可忽略 → 投影不再是精确的高斯。这就是为什么 densify 要拆分大高斯。
</details>
