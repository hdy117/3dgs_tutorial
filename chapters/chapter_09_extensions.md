# 第9章：3DGS的进化——扩展与变体的第一性原理分析

**学习路径**：`problem → invention → verification`

**本章核心问题**：3DGS核心思想已经非常优雅，但还有哪些**痛点**？后续工作如何在不破坏简洁性的前提下，解决这些痛点？每个扩展的"为什么"是什么？

---

## 一、问题的本质：3DGS有哪些局限？

### 1.1 核心假设的边界

3DGS建立在几个关键假设上：

```
假设1: 场景静态
  问题: 真实世界有运动（人、车、树叶）

假设2: 颜色视角无关
  问题: 镜面反射、透明物体需要view-dependent建模

假设3: 高斯分布健康
  问题: 训练中可能"融化"（Σ接近奇异）

假设4: 数据量可接受
  问题: 1M高斯 = 52MB，移动端/网络传输困难

假设5: 训练速度够快
  问题: 30分钟对大规模场景仍太慢
```

**这些假设在什么条件下失效？** 这就是扩展方向的来源。

---

### 1.2 三个核心需求

从第一性原理，任何扩展必须满足：

**需求1：不破坏实时性**
- 核心优势是实时渲染（<10ms）
- 任何扩展不能显著拖慢速度

**需求2：保持可微分**
- 3DGS的核心是端到端可微
- 新增模块必须可微，才能联合优化

**需求3：最小改动原则**
- 核心管线（投影+混合）不变
- 只改最小组件解决特定问题

**违反任一需求，就不再是3DGS了。**

---

## 二、Axioms：扩展设计的公理

### 公理1：问题驱动，不为扩展而扩展

**事实**：每个扩展必须针对明确痛点，有量化指标（速度、质量、内存）

**反例**："加了 attention 机制"（无明确问题）
**正例**："压缩到<10MB同时PSNR损失<1dB"

---

### 公理2：时间/空间/视角的独立性

**洞察**：3DGS的三个维度：
- **空间**：μ, Σ（本章几何质量）
- **视角**：c（颜色，是否依赖视角）
- **时间**：t（动态场景）

**扩展可以只动其中一个维度**，其他保持不变。

---

### 公理3：渐进式复杂度

**原则**：从最简单方案开始，逐步增加复杂度：
- 线性插值 → 多项式 → MLP（动态）
- 直接存储 → 量化 → 熵编码（压缩）

**只有简单方案不够时，才升级到复杂方案。**

---

## 三、Contradictions：扩展中的张力

### 矛盾1：动态性 vs 实时性

**问题**：
- 动态场景需要时间t的函数 μ(t), Σ(t)
- 但每帧评估时间函数会增加计算
- 如何保持实时？

**方案对比**：

| 方案 | 时间建模 | 内存 | 速度 | 质量 |
|------|----------|------|------|------|
| 关键帧+线性 | μ(t)线性插值 | ×2 | ✓ | △ |
| 多项式 | 三次/五次 | ×3-4 | △ | ✓ |
| 小MLP | 神经网络 | ×10+ | ✗ | ✓✓ |

**选择**：根据运动复杂度选最简单够用的方案。

---

### 矛盾2：压缩 vs 质量

**问题**：
- 量化到8位（uint8）会损失精度
- 但某些参数（Σ的尺度）对量化更敏感
- 如何分配位数？

**分析**：
- μ：世界坐标，范围大（比如[-100,100]），需要16位
- Σ：尺度，log空间更均匀，8位足够
- α, c：[0,1]范围，8位足够

**策略**：不同参数不同量化策略。

---

### 矛盾3：几何质量 vs 速度

**问题**：
- 高斯"融化"（Σ奇异）导致数值不稳定
- 但约束Σ（如各向同性）会损失自由度，降低质量
- 如何平衡？

**方案**：
- 硬约束（clamp）：速度快，但可能截断梯度
- 软约束（正则化）：梯度完整，但需要调权重
- 表面对齐（SDF）：质量最好，但增加计算

**选择**：根据场景复杂度权衡。

---

## 四、Solution Path：各扩展方向的推导

### 4.1 动态场景：4D Gaussian Splatting

**问题**：静态假设失效 → 需要时间维度

**Axioms**：
- 时间函数应该简单（多项式优先）
- 内存不能爆炸（关键帧稀疏）
- 实时性必须保持（评估快）

**Solution**：
```
μ(t) = μ₀ + Δμ · basis(t)
Σ(t) = Σ₀ + ΔΣ · basis(t)  # 或 log-space
α(t) = sigmoid(α₀ + Δα · basis(t))

basis(t): 三次多项式（4个系数）
内存: 每个高斯 ×3-4 系数
评估: 计算basis(t) → 得到参数 → 渲染
```

**为什么不是小MLP？**
- MLP推理慢（~0.5ms/高斯×1M = 500ms）
- 多项式评估快（4次乘法）
- 对于大多数场景（平滑运动），多项式足够

**需要细化**：多项式基函数的具体形式（Bezier vs Hermite）？

---

### 4.2 压缩传输：量化 + 熵编码

**问题**：52MB太大 → 目标<10MB

**Axioms**：
- 压缩后质量损失<1dB（不可见）
- 解压速度要快（流式加载）
- 渐进式传输（首帧快）

**Solution**：
```
1. 参数量化（不同精度）
   μ: int16 (2B) - 归一化到场景bbox
   Σ: uint8 (1B) - log(scale)线性映射
   α: uint8 (1B) - [0,1]→[0,255]
   c: uint8 (1B) - RGB→[0,255]
   → 13 bytes/高斯 vs 52 bytes，压缩比4×

2. 空间差分 + 熵编码
   排序高斯 → 计算差分Δμ → arithmetic coding
   → 额外压缩20-30%

3. 渐进式流式
   视锥剔除 + 分块传输
   → 首帧只传10%高斯，100ms内出图
```

**为什么μ用int16而Σ用uint8？**
- μ范围大（场景bbox），16位保证精度
- Σ的尺度（log后）集中在某区间，8位足够
- 实验：Σ量化到8位PSNR损失<0.2dB

---

### 4.3 训练加速：稀疏 + 多分辨率

**问题**：30分钟太慢 → 目标5分钟

**Axioms**：
- 3DGS泛化能力强，不需要每帧都看
- 低分辨率训练可以迁移到高分辨率
- 梯度累积模拟大batch

**Solution**：
```
1. 稀疏视角训练
   每epoch只采样20%图像
   加速5×

2. 多分辨率渐进
   0-10k步: 256×256
   10k-20k步: 512×384
   20k-30k步: 1024×768 (原分辨率)
   加速3×

3. 梯度累积
   virtual batch size = 8
   稳定性↑，速度✓
```

**总加速**: 5×3 = 15× → 30分钟 → 2分钟

**质量损失**: PSNR < 1 dB（可接受）

---

### 4.4 几何质量：防止高斯"融化"

**问题**：Σ接近奇异 → 数值不稳定，渲染条纹

**Axioms**：
- Σ必须保持正定（特征值>0）
- 约束不能太硬（否则梯度截断）
- 应该引导而非强制

**Solution**：
```
方案1: 尺度上限裁剪（硬）
  Sigma.data.clamp_(max=1.0²)
  速度快，但梯度截断

方案2: 各向同性正则（软）
  L_iso = MSE(Σ, diag(mean(diag(Σ))))
  梯度完整，但PSNR-1~2dB

方案3: 表面对齐（SDF）
  预测SDF场，损失: |dist(μ, SDF=0)|
  质量最好，但增加网络
```

**推荐**: 方案1+2组合（硬约束+软正则）

---

### 4.5 View-dependent：小MLP预测颜色

**问题**：颜色c常数 → 无法表示镜面反射

**Axioms**：
- 视角效果只影响颜色，不影响几何
- MLP要小（<1K参数），推理<1ms
- 可微分，联合优化

**Solution**：
```
c = MLP_φ(μ, view_dir)
  μ: 3D位置（归一化）
  view_dir: 相机方向（3D）
  MLP: 1-2层，32神经元，ReLU
  输出: RGB (3)

训练: 渲染损失反向传播到MLP + 高斯参数
推理: MLP前向 + 高斯渲染（慢10%）
```

**为什么MLP这么小就够了？**
- 输入只有6维（μ3 + dir3）
- 每个高斯的视角效果是局部现象
- 大MLP过拟合，小MLP泛化好

**效果**: 镜面区域PSNR +2dB，速度70→60 FPS（可接受）

## 六、思考题（第一性原理式）

**1. 动态场景的"时间连续性"约束**
- 4D GS用多项式建模时间，但多项式在端点导数可能突变
- 真实运动是连续的（加速度有限）
- 如何设计基函数保证C2连续？（如B样条）
- 推导：三次Hermite插值 vs 三次Bezier，哪个更适合？

**2. 压缩的"感知优化"**
- 当前均匀量化（所有高斯同等精度）
- 但前景高斯更重要，应该给更多位数
- 设计**自适应量化**：根据高斯屏幕面积分配精度
- 如何实现？需要哪些额外信息？

**3. 几何质量的"本质"**
- 高斯"融化"是因为Σ特征值→0，但为什么梯度会让特征值减小？
- 从损失函数角度：什么损失会惩罚小特征值？
- 推导：L_scale = clamp(scale - 1.0) 为什么能防止融化？

**4. 训练加速的"信息理论"**
- 稀疏视角训练为什么有效？3DGS的**有效带宽**是多少？
- 如果每帧信息量是I，那么看到20%帧就能恢复80%信息吗？
- 从**表示学习**角度：3DGS的参数之间冗余吗？为什么可以稀疏训练？

**5. 扩展的"正交性"**
- 这些扩展方向（动态、压缩、加速、几何、视角）是否正交？
- 能否同时用Compressed + GaussianShader？
- 如果叠加所有扩展，内存、速度、质量会怎样？

## 七、本章核心记忆点

✅ **扩展设计公理**：
  - 问题驱动，不为扩展而扩展
  - 只动一个维度（时间/空间/视角）
  - 渐进式复杂度（简单→复杂）

✅ **动态场景（4D GS）**：
  - 参数时间化：μ(t) = μ₀ + Δμ·basis(t)
  - 基函数：多项式（三次）最简单有效
  - 内存×3，速度仍实时

✅ **压缩传输**：
  - 量化：μ:int16, Σ/α/c:uint8（log空间）
  - 差分+熵编码：额外压缩20-30%
  - 总压缩比4×，质量损失<0.5dB

✅ **训练加速**：
  - 稀疏训练（20%数据）：5×
  - 多分辨率渐进：3×
  - 总加速15×，质量损失<1dB

✅ **几何质量**：
  - 问题：Σ奇异（"融化"）
  - 方案：尺度上限裁剪（硬）+ 各向同性正则（软）

✅ **View-dependent**：
  - 小MLP预测颜色：c = MLP(μ, view_dir)
  - MLP<1K参数，推理<1ms
  - 镜面区域PSNR +2dB
