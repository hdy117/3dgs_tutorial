# 第9章：扩展与变体

**学习路径**：`problem → invention`（进阶）

**核心目标**：了解3DGS的后续发展，理解每个扩展的"问题→发明"逻辑

---

## 一、引言：3DGS只是开始

2023年3DGS论文发表后，学术界快速迭代出多个变体，解决原始方法的局限性：

| 扩展方向 | 核心问题 | 代表工作 | 解决思路 |
|----------|----------|----------|----------|
| 动态场景 | 3DGS假设静态场景 | 4D Gaussian, Dynamic 3DGS | 时间参数化 |
| 压缩传输 | 高斯数据量大，难实时流式 | Compressed 3DGS, Streaming 3DGS | 量化 + 熵编码 + 渐进式 |
| 训练加速 | 训练仍较慢（30分钟-数小时） | Fast 3DGS, Mini-Splatting | 稀疏训练 + 多分辨率 |
| 几何质量 | 高斯可能"融化"（几何不稳定） | Surf-GS, Mip-Splatting | 约束 + 正则化 |
| 与NeRF结合 | 继承NeRF的view-dependent效果 | GaussianShader, NeRF-Gaussian | 小MLP + 高斯 |

本章概览这些扩展的 **problem → invention** 逻辑。

## 二、动态场景：4D Gaussian Splatting

### 2.1 Problem：静态假设失效

- 3DGS假设场景静态，但真实世界有运动（行人、车辆、树叶）
- 如何表示时间维度？

---

### 2.2 Invention：时变高斯参数

**思路**：
- 每个高斯参数（μ, Σ, α, c）是**时间t的函数**
- 给定任意时刻 t，计算该时刻的高斯集合，然后渲染

**实现方案**：

#### 方案A：参数化为基础值 + 时变增量

```
μ(t) = μ₀ + Δμ(t)
Σ(t) = Σ₀ + ΔΣ(t)
α(t) = sigmoid(α₀ + Δα(t))
```

其中 Δμ(t), ΔΣ(t), Δα(t) 用小MLP或多项式建模

**优点**：
- 连续时间插值
- 可学习复杂运动

**缺点**：
- 每个高斯额外参数 → 数据量翻倍
- MLP推理增加开销

#### 方案B：关键帧 + 插值

- 只在关键帧存储完整高斯
- 中间帧：线性插值 μ, Σ
- 速度：足够快即可

**优点**：
- 内存效率高
- 实现简单

**缺点**：
- 只能表示线性运动
- 复杂运动需要密集关键帧

---

### 2.3 代表工作

- **4D Gaussian Splatting**（2023）：用多项式建模运动轨迹
- **Dynamic 3DGS**：将高斯基元绑定到动态物体的rigid transforms

---

## 三、压缩与流式传输

### 3.1 Problem：数据量太大

- 1M高斯 × 13 float = 52 MB（float32）
- 移动端/网络传输压力大
- 需要压缩到 < 10MB 才能实时流式

---

### 3.2 Invention：量化 + 熵编码 + 渐进式

#### 方案1：参数量化

| 参数 | float32 | uint8 | 压缩比 |
|------|---------|-------|--------|
| μ（位置） | 4字节 | 2字节（归一化到场景边界） | 2× |
| Σ（尺度） | 4字节 | 1字节（log空间） | 4× |
| α | 4字节 | 1字节（0-255） | 4× |
| c | 4字节 | 1字节（0-255） | 4× |
| **总计** | **52 MB** | **~13 MB** | **4×** |

**量化策略**：
- μ：归一化到场景边界 [-10, 10] → int16（范围-32768~32767）
- Σ：取对数后归一化 → uint8
- α, c：直接线性映射到 [0, 255]

**精度损失**：< 0.5 dB PSNR（可接受）

---

#### 方案2：空间结构编码

- 高斯天然可排序（按位置）
- 使用差分编码（相邻高斯Δμ）→ 熵编码（如arithmetic coding）
- 进一步压缩 20-30%

---

#### 方案3：渐进式流式

- 只传输屏幕空间可见的高斯（视锥剔除后）
- 分块传输：先传中心区域，再传边缘
- LOD：远距离用更少高斯（合并或删除）

**效果**：
- 首帧快速（只传视锥内）
- 后续增量更新

---

### 3.3 代表工作

- **Compressed 3DGS**：混合量化 + 字典编码，压缩至5MB
- **Streaming 3DGS**：网络传输时只加载视锥内高斯

---

## 四、训练加速

### 4.1 Problem：训练仍太慢

- 原始3DGS：30k步 ~ 30分钟（RTX 4090）
- 大规模场景：数小时
- 能否更快？

---

### 4.2 Invention：稀疏训练 + 多分辨率

#### 方案1：稀疏视角训练

- 只用10-20%的输入图像训练
- 其余图像作为验证
- 效果：损失很小（因为高斯泛化能力强）

**实验**：用1/10视角训练，PSNR下降 < 1 dB

---

#### 方案2：多分辨率渐进

- 前期：低分辨率（如 400×300）快速收敛
- 后期：高分辨率微调
- 加速：低分辨率渲染快，梯度稳定

**调度**：
```
0-10k步：256×256
10k-20k步：512×384
20k-30k步：1024×768（或原分辨率）
```

---

#### 方案3：梯度累积

- 小batch导致梯度噪声大
- 累积多帧梯度再更新 → 减少步数
- 但总时间可能不变（只是步数减少，每步更贵）

---

### 4.3 代表工作

- **Fast 3DGS**：结合稀疏训练 + 多分辨率，训练加速5×
- **Mini-Splatting**：更小的初始高斯数量，加速收敛

---

## 五、几何质量提升

### 5.1 Problem：高斯可能"融化"

- 训练中，高斯可能变得极扁（Σ接近奇异矩阵）
- 渲染出现条纹或噪声
- 几何不"干净"

---

### 5.2 Invention：约束 + 正则化

#### 方案1：各向同性约束

- 强制 Σ 为 σ² * I（球对称）
- 优点：避免退化，简化优化
- 缺点：失去旋转自由度，质量略降（PSNR -1~2 dB）

**实现**：
```python
def isotropic_regularization(Sigma):
    # 强制 Sigma = diag(mean_diag)
    diag = Sigma.diagonal(dim1=1, dim2=2)  # (N, 3)
    mean_diag = diag.mean(dim=1, keepdim=True)
    return F.mse_loss(Sigma, torch.diag_embed(mean_diag))
```

---

#### 方案2：表面对齐

- 额外网络预测 SDF 或深度，约束高斯贴近表面
- 损失：L_surface = Σ distance(gaussian_center, surface)

**问题**：需要额外网络，增加复杂度

---

#### 方案3：尺度上限（已在第5章）

- 在损失函数中惩罚过大的尺度
- 更激进：硬性裁剪 Σ 到最大阈值

```python
max_scale = 1.0
Sigma.data.clamp_(max=max_scale**2)
```

---

### 5.3 代表工作

- **Surf-GS**：结合表面约束，高斯贴合几何表面
- **Mip-Splatting**：类似mipmap，防止远处高斯过扁

---

## 六、与神经方法结合：3DGS + NeRF

### 6.1 Problem：3DGS的view-dependent效果弱

- 3DGS颜色是常数（或仅学习率调整）
- 真实场景有view-dependent效果（镜面反射、透明物体）
- NeRF用视角相关MLP建模，但慢

---

### 6.2 Invention：小网络 + 高斯

**思路**：
- 保持高斯快速渲染
- 但颜色 c 由一个小MLP（输入：位置 + 视角方向）预测
- MLP训练快（因为MLP只负责颜色，几何由高斯负责）

**公式**：
```
C(p, d) = MLP(p, d)  # p=高斯中心，d=视角方向
```

**训练**：
- 可微分渲染仍通过高斯投影
- 但最终颜色来自MLP，梯度更新MLP + 高斯参数

**优势**：
- 保留3DGS几何 + 渲染速度
- 增加NeRF的视角依赖效果

**代价**：
- MLP推理增加少量开销（可接受）
- 训练稍复杂（两个模块）

---

### 6.3 代表工作

- **GaussianShader**：用小MLP预测颜色
- **NeRF-Gaussian**：混合表示，高斯占主导

---

## 七、场景编辑与生成

### 7.1 Problem：如何编辑3DGS场景？

- 原始3DGS是重建工具，编辑困难
- 需要：添加/删除物体、改变材质、动画

---

### 7.2 Invention：结构化 + 生成先验

#### 方案1：实例分离

- 训练时，每个物体独立高斯集合
- 编辑：替换/修改单个实例
- 需要实例mask（来自SfM或手动）

#### 方案2：文本编辑

- 用Diffusion模型生成新视角
- 或：用文本到3D生成新物体，融合到原场景

#### 方案3：参数解耦

- 将高斯参数分解为：base_color + perturbation
- 编辑base_color → 改变材质

---

### 7.3 代表工作

- **GaussianEditor**：文本驱动编辑
- **DreamGaussian**：从文本生成3DGS场景

## 八、总结：3DGS的设计哲学

### 8.1 核心原则

1. **简洁性**：
   - 原始3DGS只用13参数/primitive，无额外网络
   - 扩展时，尽量保持核心简单

2. **可微分**：
   - 所有操作解析可微 → 梯度优化稳定

3. **稀疏性**：
   - 只存可见表面 → 内存和计算都高效

4. **投影优先**：
   - 设计时就考虑"如何快速投影"，而非"如何重建完美3D"

---

### 8.2 扩展的共性

- **问题驱动**：每个扩展都针对一个具体痛点
- **最小改动**：在核心基础上加最小必要组件
- **保持可微**：新增模块仍可微
- **性能意识**：不牺牲实时性

---

### 8.3 未来方向

- **更大场景**：城市级别（>100M高斯）
- **动态+编辑**：实时可编辑的动态场景
- **端侧部署**：手机/AR眼镜实时运行
- **多模态融合**：结合文本、音频控制
