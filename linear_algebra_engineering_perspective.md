# 工程视角：为什么工程师总想把世界压回线性

**这份文档不重复 `linear_algebra.md` 的基础定义。**

那一份回答的是：

- 向量、矩阵、特征值、SVD 分别是什么
- 它们在 3DGS、图形学、视觉里具体怎么出现

而这一份只回答一个更“工程”的问题：

> 为什么真实世界明明充满非线性，工程师却还是会一遍又一遍地把问题改写成 `Ax`、`Ax+b`、`J dx`、`A^T A`、`U S V^T` 这类线性结构？

如果 `linear_algebra.md` 像“空间直觉手册”，这份更像“建模决策手册”。

先把主线写在前面：

现实问题往往长这样：

```text
y = f(x)
```

而不是：

```text
y = A x
```

但工程师仍然会不断尝试把它压成下面四类结构之一：

1. **全局线性**：`y = A x`
2. **仿射**：`y = A x + b`
3. **局部线性**：`f(x0 + dx) ≈ f(x0) + J(x0) dx`
4. **分段线性**：在不同工作区间里，各自近似成一个 `A_i x + b_i`

原因不是“世界本来就是直线”，而是：

> 线性结构最容易分析、组合、求解、求导、加速和调试。

---

## 一、工程并不偏爱“直线”，工程偏爱“可控”

很多人会误以为：工程里总讲线性，是因为现实世界本来就线性。

恰好相反。

现实世界大多数时候都不是线性的：

- 空气阻力常近似随速度平方变化：`F_d ~ v^2`
- 二极管电流近似指数关系：`I = I_s (e^{V/V_t} - 1)`
- 透视投影里天然带除法：`u = f_x X / Z + c_x`
- 机器人姿态里到处是 `sin(theta)`、`cos(theta)`
- 材料一旦进入大变形或塑性区，刚度就不再是常数
- 神经网络本身就靠非线性激活表达复杂函数

所以工程师真正喜欢的，从来不是“世界是直线”，而是：

> 在当前尺度、当前工作点、当前精度要求下，系统能不能被压成一个足够听话的近似模型。

### 1.1 什么叫“听话”

最经典的“听话”结构，就是线性：

```text
T(a x1 + b x2) = a T(x1) + b T(x2)
```

拆开就是两件事：

```text
T(x1 + x2) = T(x1) + T(x2)    （叠加性）
T(c x)     = c T(x)           （齐次性）
```

这两条一旦成立，工程师会立刻很安心，因为它意味着：

- 复杂输入可以拆开分析
- 缩放关系有稳定直觉
- 多个模块串联后还能继续压成矩阵
- 数值库和 GPU 很容易接手

### 1.2 一个工程比喻

你可以把线性系统想成：

> 一辆还在铁轨上的火车。

它当然可能很长、很重、很复杂，但至少你知道：

- 它会沿轨道走
- 多大的推力大概对应多大的响应
- 前后模块接起来后，整体行为还能继续分析

而强非线性系统更像：

> 一匹脾气很大的野马。

你不是完全不能骑，但你要随时准备面对：

- 工作点一变，性格也变
- 小扰动可能没事，大扰动突然翻脸
- 两个输入叠加后长出你没预料的新行为

## 二、工程师真正害怕的，不是公式复杂，而是系统“不听话”

做工程时，真正可怕的往往不是式子长，而是下面这些现象：

```text
输入多一点，输出突然饱和

两个单独正常的信号，一叠加就长出新频率

同一个控制器，在不同工作点像在控两台不同机器

参数只改一点，系统就从稳定跳到发散

GPU 管线本来全是矩阵链，突然某一步变成大量分支和特判
```

这些问题的共同点是什么？

就是：

> 系统失去了统一结构，结果变得难预测、难调试、难组合。

### 2.1 一个最小诊断量：局部增益是否稳定

在 1D 里，系统的“局部脾气”往往由导数决定：

```text
g(x) = d f(x) / d x
```

如果 `g(x)` 在工作区间里变化很小，你通常就能把系统当成近似线性的：

```text
f(x0 + dx) ≈ f(x0) + g(x0) dx
```

如果 `g(x)` 变化剧烈，那说明：

- 系统在不同位置“性格不同”
- 你用一个固定增益去描述它会出事

在多维情况下，对应的就是 Jacobian：

```text
J(x) = d f / d x
```

你可以把 Jacobian 看成：

> 系统在当前工作点的局部操作说明书。

### 2.2 一个形象例子：饱和函数为什么难伺候

比如：

```text
y = tanh(x)
```

它在 `x = 0` 附近几乎像直线：

```text
tanh(x) ≈ x
```

但当 `|x|` 变大，它会迅速饱和到 `±1`。

所以如果你的信号总在 `0` 附近晃，它就很好建模；
但如果信号老是大幅摆动，一个固定线性模型就会彻底失真。

这就是工程里很常见的一句潜台词：

> 问题不是系统能不能线性化，而是你让它工作在哪。

## 三、线性到底给了工程什么红利

不要把“线性”只理解成一种数学定义。更值得问的是：

> 它到底让什么事情突然变容易了？

### 3.1 红利一：可拆解

如果系统满足叠加性：

```text
T(x1 + x2) = T(x1) + T(x2)
```

你就可以把复杂输入拆成若干简单输入分别分析，再拼回去。

**作用**：把“大系统”拆成“小问题”。

**比喻**：像把一锅混合菜重新分成土豆、青椒、肉丝分别调味。

**工程例子**：

- 电路里的叠加原理
- 信号处理里的频率分量分析
- 结构振动里的模态叠加
- 图像处理中多个滤波器效应的组合

这件事几乎是整个工程分析的地基。

### 3.2 红利二：可缩放

如果系统满足：

```text
T(c x) = c T(x)
```

你至少知道一个极重要的事实：

> 输入放大一倍，输出不会突然变成完全不同量级的怪物。

**作用**：增益设计、量纲直觉、参数调节都会稳定很多。

**比喻**：像一个靠谱的音量旋钮，拧大一点就是更响一点，而不是突然破音。

**工程例子**：

- 放大器小信号模型
- 机械系统的小位移响应
- 线性滤波器的幅值响应

### 3.3 红利三：可压缩成矩阵

一旦系统是线性或局部线性的，你就能把它压成一个统一对象：

```text
y = A x
```

或在局部写成：

```text
dy ≈ J dx
```

这一步非常关键，因为它把：

```text
一个复杂系统的行为
```

变成了：

```text
一个统一可计算的对象
```

于是整套工具链就接上了：

- 特征值
- 秩
- 最小二乘
- SVD
- 稳定性分析
- 自动求导
- BLAS / CUDA / GPU 加速

### 3.4 红利四：可组合

线性模块串起来之后，整体仍然是线性的：

```text
y = A3 A2 A1 x
```

这让工程师特别舒服，因为你可以把系统设计成模块链：

```text
输入
 -> 传感器模型
 -> 坐标变换
 -> 滤波器
 -> 控制器
 -> 执行器
 -> 输出
```

只要每一段都保留良好的线性结构，整体还在一个可分析框架里。

### 3.5 红利五：可预测、可验证、可调试

工程不只关心“能不能跑”，还关心：

- 它会不会稳定
- 参数改了会怎样
- bug 出在哪一段
- 数值条件是否会恶化
- 哪些方向的信息被压没了

线性结构最宝贵的地方之一，就是这些问题都有比较统一的语言：

- 稳定性看特征值
- 信息量看秩和奇异值
- 病态性看条件数
- 最优拟合看最小二乘

这就是为什么工程师看到一个问题能压成矩阵，心里会立刻踏实很多。

## 四、为什么真实世界常常“看起来够线性”

既然世界大多非线性，为什么工程上又常常能成功使用线性工具？

因为下面四件事经常同时成立。

### 4.1 很多基础规律骨子里就带有加法结构

工程世界里最常见的守恒律和组合规律，本身就偏爱“相加”：

- 力会相加
- 电流会相加
- 热流和质量流往往先汇总再传递
- 多个来源的贡献常常先叠加，再经系统传播

所以很多系统从骨架上就已经离线性很近了。

### 4.2 很多器件在正常工作区近似成正比

经典例子：

```text
F = k x
V = I R
sin(theta) ≈ theta      （小角度）
Delta i ≈ g_m Delta v    （小信号）
```

关键词不是“永远成立”，而是：

> 在你关心的工作范围里，它足够像成正比。

### 4.3 平滑的非线性，局部放大后常常像直线

这是最本质的一步。

对一元函数来说，一阶泰勒展开是：

```text
f(x0 + dx) = f(x0) + f'(x0) dx + O(dx^2)
```

对多元函数来说：

```text
f(x0 + dx) ≈ f(x0) + J(x0) dx
```

这里 `J(x0)` 就是 Jacobian。

这句话的几何意义非常直观：

> 一条弯曲曲线，只要你盯住足够小的一段，它看起来就像切线。

这不是在否认非线性，而是在说：

> 对“小扰动传播”这件事，一阶信息往往已经够用了。

### 4.4 工程师会主动把系统限制在线性区工作

这点特别重要，也特别容易被忽略。

很多系统不是“天然线性”，而是**被工程师使用在线性区**：

- 放大器偏置在小信号工作点附近
- 结构件尽量工作在小变形范围
- 控制器围绕平衡点设计
- 传感器量程选在响应最接近线性的区域
- 图形学里对一个足够小的局部 footprint 做 Jacobian 近似

所以更准确的说法是：

> 工程不是发现世界是线性的，而是努力把任务放进线性近似还成立的工作区里。

### 4.5 一个很实用的误差尺子：曲率决定你能走多远

在 1D 里，线性近似误差大致和二阶项有关：

```text
f(x0 + dx) = f(x0) + f'(x0) dx + 1/2 f''(xi) dx^2
```

这里 `xi` 在 `x0` 和 `x0 + dx` 之间。

这告诉你两件事：

1. `dx` 越小，误差越像 `dx^2` 那样迅速缩小
2. 曲率 `f''` 越大，线性近似越容易坏

所以一个非常工程化的判断是：

> 不是问“它是不是非线性”，而是问“在我的 `dx` 范围里，曲率能不能被忽略”。

## 五、别把这些词混成一团：齐次、线性、仿射、局部线性、分段线性

这些词经常一起出现，但它们不是一回事。

| 概念 | 数学形式 | 它真正强调什么 | 典型例子 |
|------|----------|----------------|----------|
| 齐次 | `f(c x) = c^k f(x)` | 缩放如何传播 | `f(x)=x^2` 是 2 次齐次 |
| 线性 | `T(a x + b y) = aT(x) + bT(y)` | 叠加和缩放都成立 | `y = A x` |
| 仿射 | `f(x) = A x + b` | 线性结构外加平移/偏置 | `R x + t`、`W x + b` |
| 局部线性 | `f(x0+dx) ≈ f(x0)+J dx` | 在工作点附近用一阶近似 | Jacobian 传播 |
| 分段线性 | 每一区间一个 `A_i x+b_i` | 全局太弯，就切段处理 | ReLU、查表近似 |

### 5.1 齐次不等于线性

例如：

```text
f(x) = x^2
```

它满足：

```text
f(c x) = c^2 f(x)
```

所以它是 2 次齐次。

但它不满足加法性：

```text
(x + y)^2 != x^2 + y^2
```

所以它不是线性。

这提醒你：

> “缩放有规律”还不够，真正让工程师高兴的是“还能拆开叠加”。

### 5.2 仿射为什么工程上特别常见

很多真实系统都不是严格线性的，因为它们有偏置项：

```text
y = A x + b
```

**作用**：保留了大部分线性结构，同时允许原点平移。

**比喻**：像你先做一个规整的变换，再把整张图一起搬走。

**工程例子**：

- 相机外参：`x_cam = R x_world + t`
- 神经网络线性层：`y = W x + b`
- 机器人位姿更新

### 5.3 仿射怎么重新塞回矩阵框架：齐次坐标

为了把：

```text
x' = R x + t
```

重新写成一次矩阵乘法，工程里常常升一维：

```text
[x']   [R  t] [x]
[1 ] = [0  1] [1]
```

这就是齐次坐标最本质的意义：

> 把“线性 + 平移”重新打包进更高一维的线性框架里。

### 5.4 分段线性：现代工程里的务实折中

很多系统全局不线性，但局部结构又很规整。

工程上常见做法不是硬求一个全局线性模型，而是：

```text
这段用模型 A1
那段用模型 A2
切换区间用逻辑判断
```

典型例子：

- ReLU：`max(0, x)`
- 饱和器近似
- 分段查表控制
- 非线性系统在多个工作点做 gain scheduling

这体现的是一种很工程的思路：

> 只要全局太难，就把它切成几块，每块驯化成线性。

## 六、一个很实用的建模梯子：工程师通常不是直接硬打非线性

面对真实问题时，工程师脑中的顺序通常更像这样：

### 6.1 第一步：能不能直接用全局线性

如果系统在工作区间里非常稳定、参数近似常数、叠加关系成立，那最舒服：

```text
y = A x
```

**常见场景**：线性滤波、稳态小信号电路、刚体旋转矩阵作用于向量。

### 6.2 第二步：不行的话，能不能用仿射

很多时候主要问题不是非线性，而只是有偏置：

```text
y = A x + b
```

**常见场景**：带偏置的传感器、相机外参、神经网络的仿射层。

### 6.3 第三步：再不行，就围绕工作点做局部线性化

如果全局太弯，但你只关心某个工作点附近的小扰动，那就写：

```text
dy ≈ J(x0) dx
```

**常见场景**：

- 机器人姿态误差传播
- SLAM / Bundle Adjustment 的线性化
- 3DGS 里的投影 Jacobian
- 控制器的平衡点线性化

### 6.4 第四步：如果工作区很宽，就分段线性化

不同工作点行为差别太大时，一个 Jacobian 不够，就做多模型：

```text
region 1 -> A1 x + b1
region 2 -> A2 x + b2
region 3 -> A3 x + b3
```

### 6.5 第五步：真的绕不过去，再让非线性当主角

当饱和、接触、切换、强耦合、大姿态变化本身就是问题核心时，才真正进入：

- 非线性优化
- 牛顿法 / Gauss-Newton / LM
- 非线性控制
- 数值仿真
- PDE 求解

这不是说非线性不能做，而是说：

> 工程师通常会先把问题往线性这边推到推不动为止，再接受完整非线性代价。

## 七、四个经典工程例子：为什么线性模型有时神准，有时会崩

### 7.1 弹簧：小变形线性，大变形就不一定

最熟悉的线性弹簧是：

```text
F = k x
```

这里 `k` 是常数刚度。

**作用**：位移翻倍，力也翻倍，系统特别规整。

**比喻**：像一根老实的皮筋，拉 1 厘米和拉 2 厘米，响应按比例来。

但真实材料常常更像：

```text
F = k x + alpha x^3
```

当 `x` 很小时，三次项可以忽略，于是：

```text
F ≈ k x
```

当 `x` 变大，`alpha x^3` 就开始主导，线性模型会明显偏掉。

**工程结论**：

> 不是弹簧“背叛了线性”，而是你离开了小变形工作区。

### 7.2 单摆：全局非线性，局部线性

完整动力学：

```text
theta_ddot + (g / L) sin(theta) = 0
```

非线性全在 `sin(theta)` 里。

但小角度时：

```text
sin(theta) ≈ theta
```

于是模型瞬间变成：

```text
theta_ddot + (g / L) theta = 0
```

这就是一个干净的线性振动系统。

**作用**：让你能直接分析固有频率、稳定性、相位关系。

**比喻**：远看是一条弯曲轨道，近看像局部直轨。

### 7.3 二极管 / 晶体管：大信号非线性，小信号线性

二极管电流模型：

```text
I = I_s (exp(V / V_t) - 1)
```

这当然不是线性的。

但如果你关心某个偏置点 `V0` 附近的小扰动：

```text
I(V0 + dV) ≈ I(V0) + g_m dV
```

其中小信号增益：

```text
g_m = dI/dV |_{V0}
```

如果用上面的指数模型，近似有：

```text
g_m ≈ I(V0) / V_t
```

这就是模拟电路里极其常见的小信号模型。

**工程结论**：

> 器件本体是非线性的，但围绕工作点的小摆动可以交给线性代数和线性电路理论处理。

### 7.4 图形学 / 3DGS：全局几何带除法，但局部传播还是靠线性代数

相机投影写出来是：

```text
u = f_x X / Z + c_x
v = f_y Y / Z + c_y
```

这显然不是线性的，因为有 `1/Z`。

但如果你只看某个高斯中心附近的微小扰动：

```text
dp ≈ J dx
```

其中 Jacobian 是：

```text
J = [f_x/Z      0      -f_x X/Z^2]
    [0       f_y/Z    -f_y Y/Z^2]
```

于是协方差就能按线性规则传播：

```text
Sigma_2d ≈ J Sigma_cam J^T
```

这几乎可以说是工程思路的教科书级例子：

> 我承认全局是非线性的，但只要局部够小，我就还是尽量把它拉回线性代数框架。

### 7.5 神经网络：整体高度非线性，但局部和分段上又充满线性结构

比如单层 ReLU：

```text
y = max(0, W x + b)
```

全局不是线性的。

但在某个激活模式固定的区域里，它其实是：

- 某些单元通过
- 某些单元截断
- 整体变成一个分段仿射映射

所以现代深度学习虽然是非线性系统，但训练和分析里仍然离不开：

- Jacobian
- Hessian 近似
- SVD
- 低秩结构
- 线性层 / 仿射层

这再次说明：

> 非线性世界里，线性结构依然是最常用的局部工作语言。

## 八、为什么线性代数最后会变成工程的“统一接口”

这部分讲的不是历史八卦，而是为什么它最后坐到了整个工程数学的中心。

### 8.1 一开始只是“联立一次方程”

最早逼出线性代数的问题并不抽象，而是非常工程：

> 有一堆互相耦合的量，怎么系统地一起算？

于是很多问题都长成：

```text
a11 x1 + a12 x2 + ... + a1n xn = b1
...
an1 x1 + an2 x2 + ... + ann xn = bn
```

压缩写法就是：

```text
A x = b
```

### 8.2 一旦能写成 `A x = b`，整个工具箱就来了

比如：

- **直接求解**：`x = A^{-1} b`（如果可逆）
- **最小二乘**：

```text
x* = argmin_x ||A x - b||^2
```

当 `A` 满列秩时，对应正规方程：

```text
A^T A x = A^T b
```

- **稳定性分析**：看特征值
- **信息量分析**：看秩和奇异值
- **压缩与降维**：看 SVD 和低秩近似

也就是说，一旦问题掉进线性框架，原本分散的任务突然共享了一套统一接口。

### 8.3 为什么 GPU、数值库、自动求导都爱它

因为矩阵运算有三个特别重要的工程优势：

1. **规则统一**：接口稳定，代码可组合
2. **数据并行**：特别适合 SIMD、GPU、张量核心
3. **库成熟**：BLAS / LAPACK / cuBLAS / PyTorch 都极度优化了这类算子

这也是为什么你会感觉现代计算系统在疯狂优化下面这些事：

- GEMM（矩阵乘）
- 卷积
- 分解
- 最小二乘
- QR / SVD / eig

因为大量工程任务只要能改写成这些结构，就能吃到现成的巨大工程红利。

### 8.4 一个更深一点的统一视角

从工程眼里看，线性代数的伟大之处不是“概念优雅”，而是：

> 它让看似不同的问题，最后都落到“一个算子如何作用于一个空间对象”这件事上。

比如：

- 图形学里是变换点和椭球
- 机器人里是传播位姿和误差
- 信号处理里是变换信号和频率分量
- 控制里是传播状态和输入
- 机器学习里是变换特征和梯度

外表不同，内核却高度相似。

## 九、什么时候值得先上线性模型，什么时候该立刻警惕

不要把这件事神秘化。很多时候你只需要做一个工程判断。

### 9.1 五个快速问题

看到一个系统时，先问：

| 问题 | 如果答案偏“是” | 倾向 |
|------|------------------|------|
| 两个输入的效果能近似叠加吗？ | `T(x1 + x2) ≈ T(x1) + T(x2)` | 线性 / 近线性 |
| 缩放是否近似同比例传递？ | `T(c x) ≈ c T(x)` | 线性 / 近线性 |
| 参数在工作区内近似常数吗？ | `k`、`R`、`g_m`、`J` 变化不大 | 线性 / 局部线性 |
| 我只关心某个工作点附近的小扰动吗？ | 是 | 局部线性 |
| 我需要的是一阶精度，而不是极高精度吗？ | 是 | 先上近似线性模型 |

如果这五问里前四问大多都是“是”，通常就值得先上线性模型。

### 9.2 什么时候要提高警惕

如果你看到下面这些信号，就要准备面对明显非线性：

- 饱和、削顶、死区、阈值切换
- `x^2`、`x y`、`sin x`、`exp(x)` 这类项成为主角
- 大姿态变化、大位移、大变形
- 材料进入塑性区，或者接触状态频繁变化
- 一叠加就长出新频率 / 新模式
- 不同工作点表现像完全不同的系统
- Jacobian 在工作区间里变化非常剧烈

### 9.3 一个很实用的经验法则

你可以把建模决策压成一句话：

> 小范围、小扰动、局部传播 —— 先用线性；大范围、强切换、主导非线性 —— 准备上完整非线性。

## 十、四个可以直接跑的 Python / Matplotlib 实验

如果你想把“局部线性化”真正变成直觉，最好的办法不是背定义，而是直接把图画出来。

### 10.1 实验一：看一条非线性曲线和它的切线

这个实验回答的问题是：

> 为什么工程师总说“在工作点附近可以线性化”？

```python
import numpy as np
import matplotlib.pyplot as plt


def f(x):
    return np.sin(x) + 0.15 * x**2


def df(x):
    return np.cos(x) + 0.30 * x


x = np.linspace(-3, 3, 400)
x0 = 0.8
y0 = f(x0)
slope = df(x0)

y_tangent = y0 + slope * (x - x0)

plt.figure(figsize=(7, 5))
plt.plot(x, f(x), label='nonlinear f(x)')
plt.plot(x, y_tangent, '--', label='local linearization at x0')
plt.scatter([x0], [y0], color='red')
plt.legend()
plt.title('A nonlinear curve and its tangent line')
plt.xlabel('x')
plt.ylabel('y')
plt.show()
```

你应该观察到：

- 在 `x0` 附近，切线几乎贴着原函数
- 离 `x0` 越远，误差越明显
- 这就是“局部线性，而不是全局线性”的最直观画面

### 10.2 实验二：小角度近似为什么能工作

这个实验回答的是：

> 为什么单摆、旋转误差、小角度控制里总在用 `sin(theta) ≈ theta`？

```python
import numpy as np
import matplotlib.pyplot as plt


theta = np.linspace(-1.5, 1.5, 400)

plt.figure(figsize=(7, 5))
plt.plot(theta, np.sin(theta), label='sin(theta)')
plt.plot(theta, theta, '--', label='theta')
plt.axvline(0, color='gray', linewidth=1)
plt.axhline(0, color='gray', linewidth=1)
plt.legend()
plt.title('Small-angle approximation')
plt.xlabel('theta')
plt.ylabel('value')
plt.show()
```

你应该观察到：

- `theta = 0` 附近，两条曲线几乎重合
- 角度变大以后，误差明显增加
- 这正是“工作区”概念：不是永远对，而是在你关心的范围里够用

### 10.3 实验三：指数器件如何在偏置点附近变成直线

这个实验回答的是：

> 为什么指数型二极管 / 晶体管模型，最后又能长出小信号线性模型？

```python
import numpy as np
import matplotlib.pyplot as plt

Is = 1e-6
Vt = 0.026


def I(V):
    return Is * (np.exp(V / Vt) - 1.0)


def dIdV(V):
    return Is / Vt * np.exp(V / Vt)


V = np.linspace(0.45, 0.75, 400)
V0 = 0.62
I0 = I(V0)
gm = dIdV(V0)
I_tangent = I0 + gm * (V - V0)

plt.figure(figsize=(7, 5))
plt.plot(V, I(V), label='diode I-V curve')
plt.plot(V, I_tangent, '--', label='small-signal tangent at V0')
plt.scatter([V0], [I0], color='red')
plt.legend()
plt.title('Nonlinear device and small-signal linearization')
plt.xlabel('V')
plt.ylabel('I')
plt.show()
```

你应该观察到：

- 整条曲线是明显非线性的
- 但在 `V0` 附近，一条直线已经能描述小扰动传播
- 这就是“小信号模型”的本质，不是把系统说成线性的，而是把扰动说成线性的

### 10.4 实验四：透视投影为什么能在局部用 Jacobian 近似

这个实验直接贴近 3DGS。

我们看一个 3D 点在相机坐标系里的扰动，比较：

- 精确投影
- Jacobian 一阶近似

```python
import numpy as np
import matplotlib.pyplot as plt

fx, fy = 800.0, 800.0
cx, cy = 320.0, 240.0

p0 = np.array([0.3, 0.1, 2.0])


def proj(p):
    X, Y, Z = p
    return np.array([
        fx * X / Z + cx,
        fy * Y / Z + cy,
    ])


X, Y, Z = p0
J = np.array([
    [fx / Z, 0.0, -fx * X / (Z**2)],
    [0.0, fy / Z, -fy * Y / (Z**2)],
])

samples = []
exact = []
linear = []

rng = np.random.default_rng(0)
for _ in range(200):
    dp = rng.normal(scale=[0.01, 0.01, 0.02], size=3)
    p = p0 + dp
    samples.append(dp)
    exact.append(proj(p))
    linear.append(proj(p0) + J @ dp)

exact = np.array(exact)
linear = np.array(linear)

plt.figure(figsize=(7, 6))
plt.scatter(exact[:, 0], exact[:, 1], s=10, label='exact projection')
plt.scatter(linear[:, 0], linear[:, 1], s=10, label='Jacobian approximation')
plt.gca().invert_yaxis()
plt.legend()
plt.title('Perspective projection vs local linearization')
plt.xlabel('u')
plt.ylabel('v')
plt.show()
```

你应该观察到：

- 当扰动足够小时，两堆点会非常接近
- 扰动越大，局部线性化误差越明显
- 这就是 3DGS 里 `Sigma_2d ≈ J Sigma_cam J^T` 的几何前提

## 十一、把整件事压成一个最短心智模型

如果你只想记一条链，就记这个：

```text
真实世界
大多非线性

    ↓

但工程任务通常有工作点、尺度限制、精度要求

    ↓

在这个范围内，很多关系可以用一阶近似描述

    ↓

一阶近似 = 线性 / 仿射 / Jacobian 传播 / 分段线性

    ↓

一旦压成这些结构，就能交给线性代数工具箱

    ↓

得到：可拆解、可组合、可求解、可求导、可加速、可调试
```

这就是为什么你走进工程世界，会反复看到：

- 矩阵
- Jacobian
- 最小二乘
- 特征值
- SVD
- 状态空间
- 协方差传播
- 低秩近似

它们不是数学家的额外装饰，而是工程在追求可控性时，被问题逼出来的最经济语言。

## 十二、一句话总结

> 工程上说“把系统线性化”，通常不是在宣称世界本体就是线性的，而是在说：在当前工作点、当前变量、当前尺度、当前精度要求下，它已经足够像线性，所以值得先交给线性代数处理。

再压缩成四句：

- 世界多数是非线性的
- 工程师尽量让系统工作在线性近似成立的区域
- 一旦线性结构成立，系统就突然变得可控
- 线性代数就是这种可控性的统一语言

## 十三、最后给你一个建模时的快速自问清单

当你下次看到一个新问题，不妨先问自己：

```text
1. 我关心的是全局行为，还是某个工作点附近的小扰动？
2. 在这段范围内，叠加近似成立吗？
3. 缩放近似会同比例传递吗？
4. 能不能把主要关系压成矩阵、Jacobian 或线性算子？
5. 如果不能，问题的非线性到底是“必须保留的主角”，还是“可以局部驯化的细节”？
```

如果前四问大多答案是“能”，那就别犹豫，先上线性代数。

如果第五问的答案是“它就是主角”，那就老老实实准备面对完整非线性。

这就是工程判断，而不是教条。
