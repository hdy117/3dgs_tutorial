## 四、基与坐标：同一根箭头，为什么数字会变？以及向量何时"多余"

这是读图形学和 3DGS 时最容易晕的地方。但我们从**第一性原理**出发——不是背定义，而是理解：**什么问题逼我们发明这些概念？**

---

### 🎯 第一章的终极问题：空间里的箭头怎么被"量化"？

回顾一下：向量是什么？是**有方向的位移**。

但问题来了：

> **如果我只给你一串数字 `[3, 1]`，你能知道这个向量有多长、朝哪指吗？**

答案是：**不能！** 除非你知道这串数字是在什么参考系下量的。

这就是本章要解决的核心问题——**基与坐标的本质关系**。

---

## 从 axioms 出发：我们到底需要什么？

### Step 1: Axioms（不可约的基础事实）

让我们列出一些"不能再简化的事实"：

1. **Axiom 1**: 向量是几何实体，有长度和方向
2. **Aaxiom 2**: 人类计算需要数字表示
3. **Axiom 3**: 不同观察者可能用不同的参考方向

**矛盾出现了（Contradiction）：**
- 如果向量是客观的几何实体
- 但它的数字表示依赖于观察者的参考系
- **那么：同一根箭头的数字会变，但箭头本身不变！**

这就逼我们发明两个概念：
1. **基（Basis）**：观察者用来量空间的"尺子集合"
2. **坐标（Coordinates）**：向量在某个基下的展开系数

---

### Step 2: 什么是"基"？——不是抽象规定，而是"参考箭头集合"

想象你在描述一个点的位置。你最自然的说法是：

> "从原点出发，朝右走 3 米，再朝上走 1 米。"

这里的**"朝右"**和**"朝上"**就是两根参考箭头——它们构成了二维空间的一组基。

更形式化地说：

$$\mathbf{x} = c_1 \mathbf{b}_1 + c_2 \mathbf{b}_2 + \cdots + c_n \mathbf{b}_n$$

其中：
- $\mathbf{b}_1, \mathbf{b}_2, \ldots$ 是**基向量**（参考箭头）
- $c_1, c_2, \ldots$ 是**坐标**（每个基方向上的"走多少米"）

如果把基向量排成矩阵：

$$B = [\mathbf{b}_1 \ \mathbf{b}_2 \ \cdots \ \mathbf{b}_n]$$

那么任何向量 `x` 都可以写成：

$$\mathbf{x} = B\mathbf{c}$$

**关键洞察：**
- $B$ 是"从坐标到几何向量的翻译器"
- $\mathbf{c}$ 是在当前基下的数字表示
- $\mathbf{x}$ 是客观的向量本身（不随基变化）

---

### Step 3: 为什么需要线性无关？——"独立尺子"的定义

现在你有了基，但还有一个关键问题：

> **如果我给你 3 根箭头当"尺子"，它们真的能描述整个二维平面吗？**

答案是：**不一定！** 如果其中两根在一条直线上，第三根就可能是多余的。

这逼我们发明**线性相关/无关**的概念——它是基的"质量检查"：

#### 从直觉出发：什么是"独立"?

```text
情况 A（独立）:           情况 B（冗余）:
   v                       u
   ↑                      ↗
   |                    ↗
   |                  ↗
   → u                → u

两根完全不同方向       两根在同一直线上
(无法互相表示)         (一个可以表示为另一个的倍数)
```

在 **情况 A** 中：
- `u` 朝右，`v` 朝上
- 你不可能用 `u` 的倍数得到 `v`,反之亦然
- 它们是**线性无关**的 → 可以张成整个平面

在 **情况 B** 中：
- `u` 和 `v` 在同一条直线上
- $v = k \cdot u$ 对某个标量 $k$ 成立
- 它们是**线性相关**的 → 只能描述一条直线，不是完整平面

---

#### 数学定义：什么是"线性组合"?

给定一组向量 $\mathbf{v}_1, \mathbf{v}_2, \ldots, \mathbf{v}_n$,任何形如:

$$c_1 \mathbf{v}_1 + c_2 \mathbf{v}_2 + \cdots + c_n \mathbf{v}_n$$

的表达式都叫**线性组合**,其中 $c_i$ 是任意标量。

**关键问题：** 如果我能找到一组"不全为 0"的系数，使得:

$$c_1 \mathbf{v}_1 + c_2 \mathbf{v}_2 + \cdots + c_n \mathbf{v}_n = \mathbf{0}$$

那这组向量就是**线性相关**。

换句话说：**如果我能用其中一些向量的组合"拼出零向量",而且不是所有系数都是 0，那就说明这些向量之间有冗余。**

---

#### 等价表述：什么叫"其中一个可以被其他表示"?

上面的定义有点绕。我们换个更直观的说法:

> **一组向量线性相关 ⇔ 至少有一个向量可以用其他的线性组合来表示**

比如三个向量 $\mathbf{a}, \mathbf{b}, \mathbf{c}$,如果:

$$\mathbf{c} = 2\mathbf{a} - \mathbf{b}$$

那这三个就是线性相关的！因为 $\mathbf{c}$ 完全可以用 $\mathbf{a}$和$\mathbf{b}$"拼出来",没有提供新的信息。

移项后:

$$2\mathbf{a} - \mathbf{b} - \mathbf{c} = \mathbf{0}$$

这就是前面说的"不全为 0 的系数使得线性组合为零"。

---

#### 为什么这很重要？——基必须是线性无关的！

回到基的定义：

> **一组向量要成为空间的"基",必须满足两个条件：**
> 1. **张成性**：任何向量都能用它们的线性组合表示
> 2. **独立性**：没有冗余，每个基向量都提供新信息

而**线性无关**正是条件 2 的数学表达！

**在二维空间里：**
- 你需要**恰好 2 个线性无关的向量**作为基
- 如果只有 1 个独立向量 → 只能描述直线（维度不够）
- 如果有 3 个向量 → 至少有一个是冗余的（维度溢出）

---

#### 矩阵视角：列向量的独立性 = 矩阵的"秩"

现在把基向量排成矩阵 $B$。如果这些列向量**线性无关**,矩阵叫**满秩**(rank = n)。

**例子:**

$$A = \begin{bmatrix} 1 & 2 \\ 3 & 6 \end{bmatrix}$$

第二列是第一列的 2 倍：$\mathbf{a}_2 = 2\mathbf{a}_1$

所以这两列**线性相关**,矩阵的 rank = 1，不是满秩 → **不能作为二维空间的基！**

---

### Step 4: 坐标变换的本质——"换尺子，不换箭头"

现在你已经知道：
- 向量是客观的几何实体（有长度、方向）
- 坐标是在某个基下的数字表示
- 基必须是线性无关的

那么问题来了：

> **如果我换一套不同的基去量同一根箭头，会发生什么？**

答案是：**数字会变，但箭头不变！**

这就是**坐标变换**的本质。

#### 公式推导：从旧基到新基

假设你有两组基：
- 旧基：$B_{old} = [\mathbf{b}_1, \mathbf{b}_2]$
- 新基：$B_{new} = [\mathbf{b}'_1, \mathbf{b}'_2]$

同一个向量 $\mathbf{x}$ 在两组基下的坐标分别是 $\mathbf{c}_{old}$和$\mathbf{c}_{new}$:

$$\mathbf{x} = B_{old}\mathbf{c}_{old} = B_{new}\mathbf{c}_{new}$$

所以：

$$B_{old}\mathbf{c}_{old} = B_{new}\mathbf{c}_{new}$$

解出 $\mathbf{c}_{new}$:

$$\mathbf{c}_{new} = B_{new}^{-1} B_{old} \mathbf{c}_{old}$$

**关键洞察：**
- $P = B_{new}^{-1} B_{old}$ 是**坐标变换矩阵**
- 它告诉你：如何在两套基之间"翻译"数字
- **箭头没变，只是尺子变了！**

---

### Step 5: "转物体"vs"换坐标系"——为什么总让人混？

因为同一个旋转矩阵 `R`,常常有两种解释:

1. **主动变换**:真的把向量转了 → $\mathbf{y} = R\mathbf{x}$
2. **被动变换**:没动向量，只是换了参考坐标系 → $\mathbf{c}_{new} = R^T \mathbf{c}_{old}$

这两种视角不是矛盾，而是在描述同一几何关系的两面。

**一个很好记的比喻是:**
> 主动变换像你真的转了箭头，被动变换像你没转箭头，只是把尺子转了。

数学上，因为旋转矩阵是正交的（$R^T = R^{-1}$），所以：

$$\mathbf{c}_{new} = R^{-1}\mathbf{c}_{old} = R^T \mathbf{c}_{old}$$

于是"世界转相机"和"相机相对世界旋转"经常只差一个转置。

---

## 🌟 为什么这对 3DGS 特别重要？

因为 3DGS 里至少有这几套空间:

- **高斯局部空间**: 每个高斯自己的主轴方向
- **世界空间**: 统一的场景坐标系
- **相机空间**: 从相机视角看世界
- **屏幕空间**: 最终像素位置

一条典型链路是:

```text
高斯局部空间 → 世界空间 → 相机空间 → 屏幕空间
    (旋转)         (平移)      (投影)     (裁剪)
```

你看到的很多"矩阵乘法",本质上都是在做这个多坐标系之间的翻译。

**关键理解：**
- 每次变换 = "换一套尺子去量同一个几何实体"
- 高斯的形状不变，但数字在不同空间里完全不同
- **线性无关的基向量保证了变换可逆、不丢信息！**

## 🖥️ 可视化：同一根向量在两套坐标系里的数字不同

```python
import numpy as np
import matplotlib.pyplot as plt

# 同一个几何向量 (在标准坐标下)
v = np.array([3.0, 1.0])

# 一套旋转了 45 度的新基
th = np.deg2rad(45)
B_new = np.array([
    [np.cos(th), -np.sin(th)],
    [np.sin(th),  np.cos(th)],
])

# v 在新基下的坐标（注意：这是 B_new^{-1} @ v）
c_new = np.linalg.inv(B_new) @ v

print('标准坐标系下:', v)
print('旋转 45°的坐标系下:', c_new)

plt.figure(figsize=(6, 6))
plt.axhline(0, color='gray', linewidth=1)
plt.axvline(0, color='gray', linewidth=1)

# 原坐标基 (标准基 e1, e2)
plt.quiver(0, 0, 1, 0, angles='xy', scale_units='xy', scale=1, color='tab:blue', label='标准基')
plt.quiver(0, 0, 0, 1, angles='xy', scale_units='xy', scale=1, color='tab:blue')

# 新基 (旋转 45°)
plt.quiver(0, 0, B_new[0,0], B_new[1,0], angles='xy', scale_units='xy', scale=1, color='tab:orange', label='旋转基')
plt.quiver(0, 0, B_new[0,1], B_new[1,1], angles='xy', scale_units='xy', scale=1, color='tab:orange')

# 同一个几何向量
plt.quiver(0, 0, v[0], v[1], angles='xy', scale_units='xy', scale=1, color='tab:green', label='向量 v', linewidth=2)

plt.xlim(-2, 4)
plt.ylim(-2, 4)
plt.gca().set_aspect('equal')
plt.title('同一根箭头，两套尺子量的数字不同')
plt.legend()
plt.show()
```

**你应该观察到：**
- 绿色箭头（向量）没动
- 蓝色坐标轴和橙色坐标轴的朝向不同
- **但用各自的尺子量，得到的数字完全不同！**

### 📌 本章小结卡片：从第一性原理出发

| 概念 | 核心直觉 | 关键公式/性质 | 典型场景 |
|------|---------|--------------|----------|
| **基** | "参考箭头集合" | $\mathbf{x} = B\mathbf{c}$ | 定义坐标系 |
| **坐标** | "在某个基下的展开系数" | $\mathbf{c} = B^{-1}\mathbf{x}$ | 向量表示 |
| **线性无关** | "没有冗余的尺子" | $\sum c_i \mathbf{v}_i = \mathbf{0} \Rightarrow c_i=0$ | 判断基是否合格 |
| **主动变换** | "真的转动量" | $\mathbf{y} = R\mathbf{x}$ | 旋转物体 |
| **被动变换** | "换参考系不动量" | $\mathbf{c}_{new} = R^T \mathbf{c}_{old}$ | 坐标系转换 |
| **3DGS 空间链** | "局部→世界→相机→屏幕" | 多矩阵串联 | 渲染流水线 |

> **问题：** 3DGS 中高斯的旋转参数定义了一个 $3\times3$ 的旋转矩阵 $R$。如果这个矩阵的列向量不是线性无关的（比如秩<3），会发生什么？

> **提示：** 从"基必须独立"和"体积缩放因子$\det(R)$"两个角度思考。

**答案：**
- 列向量不独立 → 三个主轴方向中有冗余
- $\det(R) = 0$ → 高斯被压扁成一个平面或一条线
- **失去三维意义！** → 渲染会出错
