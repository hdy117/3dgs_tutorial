# Ch02 - 如果你要重新发明 3DGS，会被迫发明哪些数学概念？

> **本章目标**：从"优化 Gaussian 参数以最小化 loss"这个单一目标出发，反向推导为什么需要导数、梯度、链式法则。  
> **核心理念**：不是"微积分很厉害所以用它"，而是"问题逼你必须用这些工具"。🔥

---

## 🎯 起点：你只有一个目标

假设你是 2023 年的研究者，面对这样的任务：

```python
# 给定
images = [img_1, img_2, ..., img_N]  # N 张带相机参数的照片
target = "生成一个 3D 表示，能渲染出 Novel View"

# 你选择用 Gaussian Splatting
class Gaussian:
    def __init__(self):
        self.mu = [x, y, z]          # 中心位置（9 自由度 per Gaussian）
        self.covariance = 3x3 matrix # 形状/旋转（6 自由度，对称正定）
        self.opacity = alpha         # 透明度（1 自由度）
        self.color = SH coeffs       # 颜色（4-64 自由度，SH 阶数决定）

# 问题：怎么从"随机初始化的 Gaussian"变成"能渲染出正确图像的 Gaussian"？
```

---

## 🔥 被迫发明 #1: "变化率" → **导数**

### 问题场景

你现在有 100,000 个 Gaussian，每个有 ~20-50 个参数。总共 **2M - 5M 自由度**。

**关键矛盾**：
> 你总不能对每个参数都"试一下往左调、再试一下往右调"吧？那要试多久？

### 被迫的洞察

假设某个参数 θ（比如第 k 个 Gaussian 的 x 坐标），loss 是它的函数 `L(θ)`：

```text
        L(θ)
          │
    高 ───┤      ╱╲
          │     ╱  ╲
    低 ───┼────╱────╰──→ θ
          │   A      B
```

**直觉**：在点 A，曲线"往上走"（θ增大 → loss 增大）。那应该往哪调？

### 导数的诞生

定义：**瞬时变化率** = `极限情况下的平均变化率`

$$\frac{dL}{dθ} = \lim_{h→0} \frac{L(θ+h) - L(θ)}{h}$$

**工程翻译**：
- `dL/dθ > 0` → θ增大会让 loss 变大 → **应该减小 θ**（往 `-gradient` 方向）
- `dL/dθ < 0` → θ增大会让 loss 变小 → **应该增加 θ**
- `dL/dθ = 0` → 可能是极值点（最优解？或者鞍点？）

### ✅ 第一个被迫结论

> **要优化参数，必须知道"每个参数的变化率"**  
> → 这就是为什么你需要**导数**。

---

## 🔥 被迫发明 #2: "多变量同时调" → **梯度向量**

### 问题场景

单个 Gaussian 的参数不是独立的！改变 μ（位置）会影响它投射到哪些像素；改变 α（透明度）会影响它在 alpha blending 中的权重。

**关键矛盾**：
> 如果只调一个参数，其他不动，效率太低。能不能"一次性知道所有参数的最优调整方向"？

### 被迫的洞察

定义：**偏导数** = "固定其他变量，只看这一个的变化率"

$$\frac{∂L}{∂μ_x} = \text{"当 μ_x 变化时 L 怎么变（假设 μ_y, μ_z, α, c...都固定）"}$$

把所有偏导数拼成一个向量：
$$∇_θ L(θ) = \begin{bmatrix} \frac{∂L}{∂μ_x} \\ \frac{∂L}{∂μ_y} \\ \frac{∂L}{∂μ_z} \\ \frac{∂L}{∂α} \\ \vdots \\ \frac{∂L}{∂c_{SH}} \end{bmatrix}$$

**关键性质**：梯度向量指向"上升最快的方向"，所以 `-∇L` 指向"下降最快的方向"。

### ✅ 第二个被迫结论

> **多参数联合优化需要知道每个参数的偏导数**  
> → 这就是为什么你需要**梯度向量**。

---

## 🔥 被迫发明 #3: "穿透渲染管线" → **链式法则**

### 问题场景

这是最关键的！render() 函数是黑盒：

```python
def render(gaussians, camera):
    # Step 1: 投影到 2D
    projected = [project(g, camera) for g in gaussians]
    
    # Step 2: 深度排序
    sorted_gaussians = sort_by_depth(projected)
    
    # Step 3: Alpha blending（逐像素）
    for pixel in image:
        color = (0, 0, 0)
        alpha_accum = 0
        for g in gaussians_contributing_to_this_pixel:
            t = (1 - alpha_accum) * g.opacity_2d
            color += t * g.color
            alpha_accum += t
        pixel.rgb = color
    
    return image
```

**关键矛盾**：
> 你想知道 `∂Loss/∂gaussian.mu`，但 Loss 和 mu 之间隔了投影 → 排序 → blending 三层函数！  
> **怎么办？**

### 被迫的洞察

假设最简单的嵌套：`L(f(g(θ)))`，想求 `dL/dθ`。

**链式法则**（一维版本）：
$$\frac{dL}{dθ} = \frac{dL}{df} · \frac{df}{dg} · \frac{dg}{dθ}$$

**工程翻译**：从 Loss 往回传，每一层"乘以这层的导数"。

扩展到多变量（矩阵形式）：
$$\frac{∂L}{∂θ_i} = \sum_{j,k,...} \frac{∂L}{∂o_j} · \frac{∂o_j}{∂h_k} · ... · \frac{∂u_m}{∂θ_i}$$

### 3DGS 中的链式法则路径

```text
Loss = MSE(rendered_img, ground_truth)
         ↓ ∂L/∂pixel_color (简单：就是误差本身)
pixel_color = alpha_blending(gaussians_2d)
         ↓ ∂color/∂(α_i, c_i) (alpha blending 的导数，解析可求！)
gaussian_2d = project_3D_to_2D(gaussian_3D)
         ↓ ∂(α_2d, c_2d)/∂(μ_3D, Σ_3D) (投影公式的导数，Jacobian 矩阵)
Loss 对 Gaussian 参数的梯度 ← DONE!
```

### ✅ 第三个被迫结论

> **渲染管线是嵌套函数 → 需要链式法则穿透它**  
> → 这就是为什么你需要**反向传播（backpropagation）**。

## 🔥 被迫发明 #4: "累积效应" → **积分**

### 问题场景

Volume Rendering Equation：
$$C(r) = \int_{t_n}^{t_f} T(t) · α(t) · c(t) dt, \quad T(t) = e^{-\int_0^t σ(s) ds}$$

**关键矛盾**：
> 光线穿过场景时，每个点的颜色都要"累积"。但积分是连续的，代码里只能离散求和。  
> **怎么从 `∫` 变成 `Σ`？**

### 被迫的洞察

数值积分的核心思想：**小区间上近似为常数**

$$\int_a^b f(t) dt ≈ \sum_{i=1}^N f(t_i^*) · Δt, \quad Δt = \frac{b-a}{N}$$

在 3DGS 中，每个 Gaussian 相当于一个"离散采样点"：
- `T_i` = 到第 i 个 Gaussian 的透射率（前面所有高斯的透明度累积）
- `(1 - e^{-σ_i δ_i})` ≈ α_i（这个高斯的不透明度）
- `c_i` = 这个高斯的颜色

离散版本：
$$C(r) ≈ \sum_{i=1}^N T_i · α_i · c_i, \quad T_i = \prod_{j<i} (1 - α_j)$$

### ✅ 第四个被迫结论

> **光线穿过场景是"累积效应"** → 自然导出积分形式  
> **但代码里只能用离散求和近似** → 这就是为什么你需要理解**数值积分**。

## 📊 总结：从问题到数学的映射表

| 工程问题 | 被迫发明的数学 | 3DGS 中的角色 |
|----------|----------------|--------------|
| "参数该往哪调？" | **导数** `df/dθ` | 单个参数的更新方向 |
| "多参数一起怎么调？" | **梯度向量** `∇f(θ)` | 所有参数的联合最速下降方向 |
| "渲染管线是黑盒，gradient 怎么穿过去？" | **链式法则** | 反向传播的核心 |
| "光线穿过场景的颜色怎么累积？" | **积分** → 离散求和 | Volume Rendering Equation |
