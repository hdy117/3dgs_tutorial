# Ch05 - 偏导数与梯度：多变量函数的"变化率"

> **本章目标**：理解 `∂f/∂θ_i`（固定其他参数，只看这一个的变化）→ `∇f(θ)`（所有参数的联合更新方向）。  
> **核心洞察**：3DGS 有百万级参数，梯度向量就是它们的"导航地图"。🔥

---

## 🎯 起点：多变量优化问题

在 Ch03，你学了单变量导数 `df/dx`。但现在面对的是：

$$L(θ) = \text{Loss}(\underbrace{μ_x, μ_y, μ_z, α, c_r, c_g, c_b, ...}_{\text{每个 Gaussian 都有这么多参数}})$$

**问题**：有上百万个参数，怎么知道该往哪个方向调？

### 被迫的洞察：**偏导数** = "固定其他变量，只看这一个"

定义 `∂L/∂μ_x` = "当 μ_x 变化时 L 怎么变（假设 μ_y, μ_z, α...都固定不动）"。

---

## 🔥 Part 1: 从几何直观理解偏导数

### 例子 5.1: 二维函数 `f(x,y) = x² + y²`

想象一个碗状曲面：
```text
      z = f(x,y)
       │    ╱╲
       │   ╱  ╲
       │  ╱    ╲────→ 等高线（同心圆）
       │ ╱  ●   ╲
       └─┴───┴───→ y
         x
```

**偏导数 `∂f/∂x`** = "沿着 x 轴方向切一刀，这条曲线的斜率"  
数学上：固定 y，把 f(x,y) 看成单变量函数 g(x) = f(x, y₀)，然后求 g'(x)。

对 `f(x,y) = x² + y²`:
- `∂f/∂x = 2x`（y 当常数，导数就是 0）
- `∂f/∂y = 2y`（同理）

在点 (1, 2):
- `∂f/∂x |_(1,2) = 2·1 = 2` → x 增大会让 f 增大，应该减小 x
- `∂f/∂y |_(1,2) = 2·2 = 4` → y 增大会让 f 增大（而且变化更快！）

---

## 🔥 Part 2: 梯度向量 —— "最陡下降方向"

### 定义：梯度 = 所有偏导数拼成的向量

$$∇f(θ) = \begin{bmatrix} \frac{∂f}{∂θ_1} \\ \frac{∂f}{∂θ_2} \\ ⋮ \\ \frac{∂f}{∂θ_n} \end{bmatrix}$$

对 `f(x,y) = x² + y²`：
$$∇f(x,y) = \begin{bmatrix} 2x \\ 2y \end{bmatrix}, \quad ∇f(1,2) = \begin{bmatrix} 2 \\ 4 \end{bmatrix}$$

### ✅ 关键定理（再次强调）

**定理**: `∇f` 指向"上升最快的方向"，所以 `-∇f` 是下降最快的方向。

**为什么？**（直观理解，不用严格证明）：
- 考虑一步变化 `Δθ = [Δx, Δy]`，函数值变化近似为：
  $$Δf ≈ ∂f/∂x · Δx + ∂f/∂y · Δy = ∇f · Δθ$$（点积！）

- **点积公式**：`a·b = ‖a‖·‖b‖·cos(φ)`
  - φ=0 (同方向) → Δf 最大（上升最快）
  - φ=180° (反方向) → Δf 最小（下降最快！这就是梯度下降）

## 🔥 Part 3: 3DGS 中的实战例子

### 例子 5.2: Gaussian 参数的联合更新

考虑一个简化的 3DGS 场景：loss = `(rendered_pixel - target)²`。

某个 Gaussian 的参数 `θ = [μ_x, μ_y, α]`（只考虑这 3 个），backward() 算出来：
$$∇_θ L(θ) = \begin{bmatrix} ∂L/∂μ_x \\ ∂L/∂μ_y \\ ∂L/∂α \end{bmatrix} = \begin{bmatrix} +0.5 \\ -0.3 \\ +0.1 \end{bmatrix}$$

**解读每个偏导数**：
| 参数 | `∂L/∂θ_i` | 符号 | 直觉含义 | 应该往哪调？ |
|------|-----------|------|----------|-------------|
| μ_x | +0.5 | > 0 | x 增大会让 loss ↑ **快**（斜率陡） | **减小 x**，而且步子要大 |
| μ_y | -0.3 | < 0 | y 增大会让 loss ↓（但变化慢些） | **增加 y** |
| α | +0.1 | > 0 | alpha 增大会让 loss ↑（影响较小） | **减小透明度**，步子可以小 |

**梯度下降更新** (η = 1e-3):
```python
theta_new = theta_old - 1e-3 * np.array([0.5, -0.3, 0.1])
# μ_x: new = old - 0.0005  ← 减得最多（因为梯度最大）
# μ_y: new = old + 0.0003  ← 往反方向调
# α:   new = old - 0.0001  ← 微调
```

### ✅ 关键洞察

> **梯度的大小决定步长，符号决定方向**  
> `|∂L/∂μ_x| > |∂L/∂α|` → μ_x 对 loss 的影响更大 → 调整 μ_x 时步子可以更大。

## 🔥 Part 4: 梯度范数与学习率自适应

### 问题：不同参数的"尺度"不一样

在 3DGS 中：
- `μ ∈ [0, 10]`（米为单位的位置）
- `α ∈ [0, 1]`（透明度，归一化到 [0,1]）
- `c ∈ [-1, +1]`（SH 系数，可能更大范围）

**矛盾**：用同一个学习率 η 更新所有参数合理吗？

### 答案：需要**梯度归一化**或**自适应优化器**

#### Option 1: Gradient Clipping
```python
# 官方代码中的做法
torch.nn.utils.clip_grad_norm_(gaussians.parameters(), max_norm=0.01)

# 原理：限制梯度的 L2 范数，防止某一步更新太大
grad_norm = torch.norm(all_gradients, p=2)
if grad_norm > 0.01:
    all_gradients *= (0.01 / grad_norm)  # scale down 所有梯度
```

#### Option 2: Adam Optimizer（更智能）
```python
# PyTorch 的默认优化器之一
optimizer = torch.optim.Adam(gaussians.parameters(), lr=1e-3)

# Adam 的原理：给每个参数分配不同的学习率
# - 对"经常变化大"的参数，自动减小 η_i
# - 对"梯度稀疏"的参数，适当增大 η_i（momentum + adaptive LR）
```

**5.2 梯度下降模拟（二维版本）**  

```python
import numpy as np

def f(theta): return theta[0]**2 + 4*theta[1]**2  # 椭圆形状的 loss landscape
def grad_f(theta): return np.array([2*theta[0], 8*theta[1]])

theta = np.array([5.0, 3.0])  # 初始点
eta = 0.05

for step in range(20):
    g = grad_f(theta)
    print(f"Step {step:2d}: theta={theta}, f={f(theta):.4f}, grad_norm={np.linalg.norm(g):.4f}")
    
    if np.linalg.norm(g) < 1e-6:
        print("Converged!")
        break
    
    theta = theta - eta * g

print(f"\n最优解：theta=[0,0], f(θ)=0")
```

**问题**: 为什么 y 方向收敛得比 x 快？（提示：看 `∂f/∂y` vs `∂f/∂x` 的系数）
